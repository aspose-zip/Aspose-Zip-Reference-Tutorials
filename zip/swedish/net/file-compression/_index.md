## Vanliga frågor

**Q: Kan jag använda Aspose.Zip för att skapa ett zip archive .net på .NET Core?**  
A: Absolut. Aspose.Zip stödjer fullt ut .NET Core 3.1 och senare, samt .NET 5/6/7.

**Q: Hur modifierar jag ett befintligt arkiv utan att först extrahera det?**  
A: Använd “modify zip file c#”‑API:erna – du kan lägga till, ta bort eller byta namn på poster direkt i arkivströmmen.

**Q: Vilken komprimeringsmetod ger bäst komprimeringsgrad för textfiler?**  
A: Generellt ger Bzip2 eller LZMA högre komprimering för text, men de är långsammare än Deflate. Testa båda för att hitta rätt balans.

**Q: Finns det ett sätt att kryptera filer när zip‑arkivet skapas?**  
A: Ja, Aspose.Zip stödjer AES‑kryptering. Du kan ange ett lösenord på ZipArchive‑objektet innan du sparar.

**Q: Måste jag oroa mig för filnamn‑kodning?**  
A: Aspose.Zip hanterar automatiskt Unicode‑filnamn, men du kan explicit ange kodning om du behöver kompatibilitet med äldre system.

**Q: Kan jag lägga till filer i ett befintligt zip utan att återskapa det?**  
A: Ja, API:et erbjuder en **add files zip .net**‑metod som låter dig lägga till nya poster direkt.

**Q: Stöder Aspose.Zip streaming av stora filer?**  
A: Ja. Du kan arbeta med strömmar för att undvika att ladda hela filer i minnet, vilket är idealiskt för storskaliga scenarier.
