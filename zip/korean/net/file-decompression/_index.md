# Aspose.Zip for .NET을 사용한 파일 압축 해제 방법

## 소개

.NET 환경에서 **파일 압축 해제 방법**을 빠르고 안정적으로 수행해야 할 때, Aspose.Zip for .NET은 수동 추출의 번거로움을 없애는 깔끔하고 고성능 API를 제공합니다. 단일 아카이브, 로그 파일 배치, 또는 비밀번호로 보호된 ZIP을 다루는 경우에도, 이 가이드는 몇 줄의 C# 코드만으로 ZIP 폴더 내용을 추출하고 암호화된 아카이브를 처리하는 방법을 정확히 보여줍니다.

## 빠른 답변
- **Aspose.Zip for .NET은 무엇을 하나요?** C#에서 ZIP, TAR, GZIP 및 기타 아카이브 형식을 생성, 읽기 및 추출할 수 있는 간단한 API를 제공합니다.  
- **한 번에 여러 파일을 압축 해제할 수 있나요?** 예, 라이브러리를 사용하면 한 번의 호출로 모든 엔트리를 추출하거나 개별적으로 순회할 수 있습니다.  
- **비밀번호 보호된 추출을 지원하나요?** 물론입니다 – 암호를 제공하여 암호화된 아카이브를 해제할 수 있습니다.  
- **지원되는 .NET 버전은 무엇인가요?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7 및 이후 버전.  
- **개발에 라이선스가 필요합니까?** 평가용 무료 체험이 가능하지만, 프로덕션 사용에는 상용 라이선스가 필요합니다.  

## Aspose.Zip for .NET을 사용한 파일 압축 해제 방법
Aspose.Zip을 사용한 파일 압축 해제는 매우 간단합니다. 라이브러리는 `Extract` 메서드를 제공하며, 이 메서드는 디렉터리, 특정 파일 또는 메모리 스트림을 대상으로 할 수 있습니다. 아래는 일반적인 워크플로우를 정리한 것입니다:

1. **Create an `Archive` instance** pointing to your zip file.  
2. **Call `Extract`** with the destination folder.  
3. **Optionally pass a password** if the archive is encrypted (`extract password protected zip`).  

이 접근 방식은 전통적인 ZIP, ZIP 폴더 구조 및 XAR, WIM과 같은 최신 형식을 포함한 Aspose.Zip이 지원하는 모든 아카이브 유형에 적용됩니다.

## “다중 파일 압축 해제”란 무엇인가요?
다중 파일 압축 해제는 아카이브(ZIP, TAR 등) 내부에 저장된 모든 엔트리를 추출하고, 필요에 따라 각 파일을 대상 디렉터리에 기록하는 작업을 의미합니다. 번들된 데이터(로그 파일, 이미지, 설정 파일 등)를 받았을 때 이를 처리하기 전에 풀어야 하는 경우에 흔히 사용됩니다.

## 다중 파일 압축 해제를 위해 Aspose.Zip for .NET을 사용하는 이유
- **Performance‑optimized** 추출로 대용량 아카이브도 과도한 메모리 사용 없이 처리합니다.  
- **Built‑in support** 클래식 ZIP, 최신 형식(XAR, WIM) 및 암호화된 아카이브를 기본 지원합니다.  
- **Straightforward C# syntax** – 스트림이나 서드파티 유틸리티(`aspose zip extract`)를 다룰 필요가 없습니다.  
- **Cross‑platform** 호환성으로 Windows, Linux, macOS 어디서든 동일한 코드를 실행할 수 있습니다.

## Zip 폴더 및 비밀번호 보호 아카이브 압축 해제
**zip 폴더** 내용을 압축 해제하거나 **비밀번호 보호된 zip** 아카이브를 다뤄야 할 경우, Aspose.Zip은 두 시나리오를 원활히 처리합니다. 대상 경로와 필요 시 비밀번호 문자열을 추출 메서드에 전달하기만 하면 됩니다. 외부 도구가 필요 없으며 코드베이스를 깔끔하게 유지할 수 있습니다.

## 일반적인 사용 사례
- **Batch processing** 원격 서버에서 받은 로그 아카이브를 일괄 처리합니다.  
- **Automated deployment** 스크립트가 설치 전에 리소스 번들을 풀어냅니다.  
- **Data migration** 레거시 zip 파일을 읽어 내용물을 데이터베이스에 저장합니다.  

## 팁 및 모범 사례
- **Use streaming** 매우 큰 파일을 추출할 때 메모리 사용량을 낮게 유지합니다.  
- **Validate file paths** 추출 후 경로를 검증하여 디렉터리 트래버설 취약점을 방지합니다.  
- **Handle exceptions** `InvalidPasswordException`과 같은 예외를 처리해 사용자에게 명확한 피드백을 제공합니다.  

## 자주 묻는 질문

**Q: zip 아카이브를 메모리 스트림으로 직접 추출할 수 있나요?**  
A: 예, Aspose.Zip을 사용하면 `MemoryStream`에 엔트리를 읽어 디스크에 쓰지 않고도 처리할 수 있습니다 (`extract zip archive c#`).  

**Q: 특정 폴더 구조로 추출을 지원하나요?**  
A: 물론입니다. 출력 디렉터리를 지정하면 API가 아카이브 내부 폴더 계층을 재현합니다.  

**Q: C#에서 비밀번호 보호된 zip 파일을 어떻게 추출하나요?**  
A: `Extract` 메서드에 비밀번호를 전달하면 됩니다(예: `archive.Extract(outputPath, password)`).  

**Q: 압축을 풀지 않고 아카이브 내용을 나열할 수 있나요?**  
A: 예, `archive.Entries`를 순회하면 파일 이름, 크기, 타임스탬프 등을 확인할 수 있습니다.  

**Q: 아카이브에 중복 파일 이름이 있으면 어떻게 되나요?**  
A: 기본적으로 기존 파일을 덮어쓰며, `OverwriteMode` 옵션으로 동작을 변경할 수 있습니다.  

**Q: zip 폴더에서 선택된 엔트리만 추출할 수 있나요?**  
A: 예, 이름이나 확장자로 `archive.Entries`를 필터링한 뒤 선택된 엔트리에 `Extract`를 호출하면 됩니다.  

**Q: 메모리가 제한된 장치에서 큰 zip 파일을 어떻게 처리하나요?**  
A: 라이브러리는 지연 로딩과 스트리밍을 사용해 현재 엔트리만 메모리에 로드합니다.
