# 비밀번호 보호 및 암호화

## 소개

.NET 애플리케이션에서 파일 보안이 걱정되시나요? 이 가이드에서는 Aspose.Zip for .NET을 사용하여 **zip 비밀번호 보호** 아카이브를 만드는 방법을 알아봅니다. *zip에 비밀번호 추가*가 필요하든, AES 암호화를 사용하든, 혹은 단순히 디렉터리를 보호하든 상관없습니다. 전통적인 비밀번호부터 최신 AES 기반 보호까지 모든 일반적인 시나리오를 다루는 단계별 튜토리얼을 제공하므로 데이터를 기밀하게 유지하고 규정을 준수할 수 있습니다.

## 빠른 답변
- **“password protect zip”이 의미하는 바는 무엇인가요?** 비밀번호 또는 암호화를 ZIP 아카이브에 적용하여 인증 없이 내용물을 열 수 없게 만드는 것을 의미합니다.  
- **가장 강력한 암호화 알고리즘은 무엇인가요?** AES‑256이 Aspose.Zip에서 제공하는 가장 안전한 옵션입니다.  
- **파일마다 다른 비밀번호로 보호할 수 있나요?** 네, Aspose.Zip은 각 엔트리에 고유한 비밀번호를 지정할 수 있도록 지원합니다.  
- **프로덕션 사용에 라이선스가 필요합니까?** 비시험 배포에는 상용 라이선스가 필요합니다.  
- **API가 .NET 6+와 호환되나요?** 물론입니다 – Aspose.Zip은 .NET Framework, .NET Core 및 .NET 5/6을 모두 지원합니다.

## 자주 묻는 질문

**Q: Aspose.Zip을 사용해 zip 파일에 비밀번호를 어떻게 추가하나요?**  
**A:** `ZipArchive` 클래스를 사용하고 `Password` 속성을 설정한 뒤 AES‑256과 같은 암호화 방식을 선택합니다.

**Q: 압축 없이 디렉터리를 비밀번호로 보호할 수 있나요?**  
**A:** 네, Aspose.Zip은 폴더 구조를 포함하는 아카이브를 생성하고 전체 아카이브에 비밀번호를 적용할 수 있습니다.

**Q: “encrypt zip with aes”와 전통적인 비밀번호 보호의 차이점은 무엇인가요?**  
**A:** AES 암호화는 강력한 암호 보안(128/256비트 키)을 제공하는 반면, 전통적인 ZIP 비밀번호는 약한 ZipCrypto를 사용합니다.

**Q: .NET에서 AES 암호화 zip 아카이브를 어떻게 압축 해제하나요?**  
**A:** `ZipArchive.ExtractAll`(또는 `ExtractEntry`)를 호출하고 아카이브 생성 시 사용한 동일한 비밀번호를 제공하면 됩니다.

**Q: 디스크에 쓰지 않고 AES 암호화 파일 스트림을 압축 해제할 수 있나요?**  
**A:** 네, Aspose.Zip은 스트림을 직접 다루는 인‑메모리 압축 해제를 지원합니다.
