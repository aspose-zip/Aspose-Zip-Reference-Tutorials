# Разархивирование сохранённого файла с помощью Aspose.Zip для .NET

## Введение

В современных приложениях .NET **создать zip без сжатия** — полезный метод, когда требуется быстрое архивирование без больших затрат на сжатие данных. Aspose.Zip для .NET используется для создания таких архивов, а также для **извлечения нескольких zip-файлов**. В этом руководстве вы увидите, как открыть zip-файл, прочитать данные zip-записи и выполнить операцию **Извлечение zip-файла C#** шаг за шагом.

## Быстрые ответы
- **Что такое «создать zip без сжатия»?** Это добавление файлов в ZIP‑архив методом «store», при котором данные остаются постоянными.
- **Какая библиотека обрабатывает это в .NET?** Aspose.Zip для .NET.
- **Нужна ли мне лицензия для запуска примера?** Бесплатная пробная версия подходит для разработки; для продакшна требуется коммерческая лицензия.
- **Могу ли я извлечь несколько файлов одновременно?** Да — в руководстве показано, как **извлечь несколько zip-файлов** в цикле.
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET5/6/7.

## Что такое «создать zip без сжатия»?

При создании ZIP‑архива методом **store** каждый файл добавляется в архив в исходном виде. Это приводит к большему размеру архива по сравнению со сжатыми ZIP‑файлами, но операция выполняется гораздо быстрее, а оригинальные байты файлов остаются нетронутыми — идеально для того, чтобы увидеть, что важнее скорость или целостность данных, чем размер.

## Понимание хранилища методов сжатия zip

**хранилище метода сжатия zip** (также называемое «хранилище») указывает формат ZIP, продающий любой шаг по сжатию данных. Aspose.Zip обеспечивает этот метод через перечисление CompressionMethod.Store, поэтому он явно выбирает его для каждой записи. Использование метода хранения особенно удобно, когда вы работаете с уже сжатыми медиафайлами (например, JPEG, MP3), где дополнительное сжатие не дает преимуществ.

## Как открыть ZIP и извлечь несколько файлов

Теперь, когда у нас есть сохранённый ZIP, посмотрим, **how to open zip** и извлечём файлы.

### Шаг 2.1: Открытие ZIP-файла

```csharp
string dataDir = "Your Document Directory";

using (FileStream zipFile = File.Open(dataDir + "StoreMultipleFilesWithoutCompression_out.zip", FileMode.Open))
{
    using (Archive archive = new Archive(zipFile))
    {
```

Объект `Archive` представляет открытый ZIP и даёт доступ к каждой записи через коллекцию `Entries`.

### Шаг 2.2. Создание извлеченных файлов


```csharp
        using (var extracted = File.Create(dataDir + "alice_extracted_store_out.txt"))
        {
            using (var decompressed = archive.Entries[0].Open())
            {
                byte[] buffer = new byte[8192];
                int bytesRead;

                // Reading from decompressed stream to extracting file.
                while (0 < (bytesRead = decompressed.Read(buffer, 0, buffer.Length)))
                {
                    extracted.Write(buffer, 0, bytesRead);
                }
            }
        }
```

Здесь мы **read zip entry** 0, копируем её байты в новый файл и автоматически закрываем потоки благодаря оператору `using`.

### Шаг 2.3. Повторение процесса для другого файла

```csharp
        using (var extracted = File.Create(dataDir + "asyoulik_extracted_store_out.txt"))
        {
            using (var decompressed = archive.Entries[1].Open())
            {
                byte[] buffer = new byte[8192];
                int bytesRead;

                // Reading from decompressed stream to extracting file.
                while (0 < (bytesRead = decompressed.Read(buffer, 0, buffer.Length)))
                {
                    extracted.Write(buffer, 0, bytesRead);
                }
            }
        }
    }
}
```

Итерируя `archive.Entries`, вы можете **extract multiple zip files** (или несколько записей) всего несколькими строками кода.

## Распространенные проблемы и решения

| Выпуск | Причина | Исправить |
|-------|-------|-----|
| `FileNotFoundException` при открытии ZIP | Неправильный путь `dataDir` | Убедитесь, что `dataDir` заканчивается слешем, или воспользуйтесь `Path.Combine`. |
| Извлеченный файл пуст | Буфер не был сброшен | Блок `using` автоматически сбрасывает; Убедитесь, что поток читается до тех пор, пока `bytesRead` не станет 0 (как показано). |
| Исключение из лицензии | Запуск без действующей лицензии | Перед развертыванием замените пробную или постоянную лицензию. |
