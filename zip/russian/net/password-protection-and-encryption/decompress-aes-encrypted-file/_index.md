Независимо от того, работаете ли вы со стандартной защитой ZIP или с AES‑зашифрованными архивами, ниже приведённые шаги проведут вас через весь процесс на C#. К концу руководства вы сможете распаковывать zip‑файлы, защищённые паролем, распаковывать AES‑zip файлы и интегрировать решение в свои приложения.

## Как извлечь пароль ZIP‑файла (распаковать ZIP, защищённый паролем)

### Шаг 1: Настройка проекта

Создайте новый консольный или настольный проект C# в Visual Studio. Добавьте ссылку на скачанный DLL Aspose.Zip и скопируйте ваш зашифрованный ZIP‑файл в папку вывода проекта (например, `bin\Debug\net6.0`).

### Шаг 2: Инициализация переменных

Определите каталог, в котором находятся ваши тестовые файлы. Это делает код чистым и переиспользуемым:

```csharp
string dataDir = "YourDocumentDirectory";
```

### Шаг 3: Распаковка AES‑зашифрованного файла

Теперь переходим к основной логике, которая действительно **извлекает пароль zip‑файла** и читает зашифрованную запись. Приведённый ниже фрагмент открывает архив, передаёт пароль (`p@s$` в этом примере) и записывает извлечённое содержимое в новый файл.

```csharp
//ExStart: DecompressAESEncryptedFile
using (FileStream fs = File.OpenRead(dataDir + "PasswordProtectWithAES256_out.zip"))
{
    using (var extracted = File.Create(dataDir + "alice_aesextracted_out.txt"))
    {
        using (Archive archive = new Archive(fs))
        {
            using (var decompressed = archive.Entries[0].Open("p@s$"))
            {
                byte[] b = new byte[8192];
                int bytesRead;
                while (0 < (bytesRead = decompressed.Read(b, 0, b.Length)))
                {
                    extracted.Write(b, 0, bytesRead);
                }
            }
        }
    }
}
//ExEnd: DecompressAESEncryptedFile
```

> **Полезный совет:** Если нужно извлечь несколько записей, пройдитесь в цикле по `archive.Entries` и вызовите `Open(password)` для каждой.

## Как распаковать AES‑ZIP файлы

Тот же класс `Archive` работает с любым AES‑зашифрованным архивом, независимо от длины ключа (128, 192 или 256 бит). Просто укажите правильный пароль, и библиотека выполнит дешифрование внутри.

## Распространённые проблемы и их устранение

| Симптом | Вероятная причина | Решение |
|---------|-------------------|---------|
| **“Invalid password” exception** | Неправильный пароль или повреждённый архив | Проверьте строку пароля, убедитесь, что она соответствует регистру и специальным символам. |
| **Zero‑byte output file** | Поток не был сброшен | Вызовите `extracted.Flush()` после цикла записи (обычно это делает `using`). |
| **Performance slowdown on large files** | Малый размер буфера | Увеличьте размер буфера (например, `byte[] b = new byte[65536];`). |
