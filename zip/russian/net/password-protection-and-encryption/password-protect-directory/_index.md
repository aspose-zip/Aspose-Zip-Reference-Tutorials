# Создание zip‑архива с паролем для каталогов .NET – руководство Aspose.Zip

В этом руководстве вы **создадите zip‑архивы с паролем** для целых каталогов, используя библиотеку Aspose.Zip для .NET. Независимо от того, нужно ли вам **зашифровать папку**, защитить файлы резервных копий или просто ограничить доступ к конфиденциальным данным, этот пошаговый учебник покажет, как сделать это с чистым кодом C#.

## Быстрые ответы
- **Какая библиотека рекомендуется?** Aspose.Zip for .NET  
- **Могу ли я зашифровать всю папку?** Да – просто укажите API на папку, которую хотите заархивировать.  
- **Поддерживается ли изменение пароля zip‑архива?** Абсолютно, используйте `TraditionalEncryptionSettings`.  
- **Нужна ли лицензия для продакшна?** Для коммерческого использования требуется действующая лицензия Aspose.Zip.  
- **Работает с .NET Core/5/6?** Да, API полностью совместим с современными .NET‑runtime.

## Что означает «создать zip‑архив с паролем»?
Создание zip‑архива с паролем означает сжатие файлов или каталогов в ZIP‑архив с одновременным применением шифрования, так что архив можно открыть только с правильным паролем. Это защищает содержимое от неавторизованного доступа.

## Как создать zip‑архив с паролем для каталога
Ниже вы найдёте полное, человекочитаемое пошаговое руководство, охватывающее всё от настройки проекта до изменения пароля позже.

## Почему стоит использовать Aspose.Zip для защиты паролем каталогов в .NET?
Aspose.Zip предлагает простой, высокопроизводительный API, поддерживающий **c# zip password protection**, традиционное шифрование ZipCrypto и AES‑шифрование. Он эффективно обрабатывает большие каталоги и без проблем интегрируется в любой .NET‑проект.

## Шаг 1: Установите путь к каталогу ресурсов
Определите путь, указывающий на каталог, который вы собираетесь заархивировать и защитить.

```csharp
string dataDir = "Your Document Directory";
```

## Шаг 2: Защитите каталог паролем
Используйте `TraditionalEncryptionSettings` для указания пароля и создания зашифрованного архива. Это ядро **c# zip password protection**.

```csharp
//ExStart: PasswordProtectDirectory
using (FileStream zipFile = File.Open(dataDir + "all_corpus_encrypted_out.zip", FileMode.Create))
{
    DirectoryInfo corpus = new DirectoryInfo(".\\CanterburyCorpus");
    using (var archive = new Archive(new ArchiveEntrySettings(null, new TraditionalEncryptionSettings("p@s$"))))
    {
        archive.CreateEntries(corpus);
        archive.Save(zipFile);
        //ExEnd: PasswordProtectDirectory
    }
}
```

## Шаг 3: Объяснение кода
- **Создание выходного файла:** `File.Open(..., FileMode.Create)` открывает (или создаёт) ZIP‑файл, который будет хранить зашифрованные данные.  
- **Выбор исходной папки:** `new DirectoryInfo(".\\CanterburyCorpus")` указывает Aspose.Zip, какой каталог сжимать.  
- **Применение пароля:** `new TraditionalEncryptionSettings("p@s$")` задаёт пароль, защищающий архив.  
- **Добавление записей и сохранение:** `archive.CreateEntries(corpus)` добавляет каждый файл из папки, а `archive.Save(zipFile)` записывает зашифрованный ZIP на диск.  

## Как изменить пароль zip‑архива позже?
Если нужно **изменить пароль zip**, просто пересоздайте архив с новым экземпляром `TraditionalEncryptionSettings`, содержащим новый пароль, и сохраните его снова. Такой подход также работает, когда вы хотите **создать зашифрованный zip‑архив** из существующей папки с другим паролем.

## Советы по созданию надёжного пароля для zip‑папки
- Используйте сочетание заглавных, строчных букв, цифр и символов.  
- Стремитесь к минимуму 12 символов; более длинные пароли экспоненциально сложнее взломать.  
- Избегайте общеупотребительных слов или шаблонов; рассмотрите возможность использования парольной фразы.

## Распространённые проблемы и советы
- **Большие папки:** Aspose.Zip передаёт данные потоками, поэтому потребление памяти остаётся низким даже для огромных каталогов.  
- **Сложность пароля:** Используйте надёжный пароль (смесь букв, цифр, символов) для повышения безопасности.  
- **Ошибки лицензии:** Убедитесь, что применили действующий файл лицензии; иначе библиотека работает в режиме оценки с ограничениями.  
- **Пароль zip‑папки не распознаётся:** Проверьте, что при открытии архива вы используете тот же метод шифрования (`TraditionalEncryptionSettings`).

## Быстрый FAQ (AI‑friendly)

**Q: Как зашифровать папку с помощью zip, используя Aspose.Zip?**  
A: Используйте `TraditionalEncryptionSettings` при создании объекта `Archive`, затем вызовите `CreateEntries` для целевой папки.

**Q: Можно ли установить пароль zip‑папки после создания архива?**  
A: Нет, пароль должен быть задан во время создания; чтобы изменить его, пересоздайте архив с новым паролем.

**Q: Поддерживает ли Aspose.Zip AES‑шифрование для более высокой безопасности?**  
A: Да, можно переключиться на `AesEncryptionSettings` для AES‑256 вместо традиционного ZipCrypto.

**Q: Совместима ли библиотека с .NET 6 и .NET 7?**  
A: Абсолютно – текущий релиз работает со всеми современными .NET‑runtime.

**Q: Что происходит, если попытаться открыть zip‑архив с паролем без пароля?**  
A: Aspose.Zip выбросит `PasswordRequiredException`, предлагая ввести правильный пароль.
