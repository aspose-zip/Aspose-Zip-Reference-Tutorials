# Как создавать 7z‑файлы с безопасным архивированием в .NET с помощью Aspose.Zip

## Введение

Если вам нужно **создавать 7z**‑архивы, которые одновременно компактны и защищены, вы попали по адресу. В современных .NET‑приложениях безопасное архивирование необходимо для защиты интеллектуальной собственности, соблюдения требований конфиденциальности данных и снижения расходов на хранение. Aspose.Zip для .NET предоставляет простой API для генерации **Seven Zip**‑архивов, применения **AES‑шифрования** и даже **защиты 7z‑файлов паролем** — всё это без выхода из среды C#.

В этом руководстве мы пройдем весь процесс создания 7z‑архива, шифрования записи zip и сохранения результата на диск. К концу вы поймёте, как **шифровать zip**‑файлы, использовать **seven zip compression** и интегрировать безопасное архивирование в любой .NET‑проект.

## Что такое 7z‑архив и зачем нужен AES‑шифр?
Файл **7z** — это архивный формат с высоким уровнем сжатия, созданный утилитой 7‑Zip. Он поддерживает продвинутые алгоритмы, такие как LZMA/LZMA2, и сильное **AES‑256**‑шифрование, что делает его идеальным для **secure archiving .net**‑сценариев, где важны как размер, так и конфиденциальность.

## Шаг 2: Создайте Seven Zip‑файл с AES‑шифрованием

```csharp
//ExStart: ArchiveWithEncryptedEntry
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("entry1.bin", 
            new MemoryStream(new byte[] { 0x00, 0xFF }), 
            new SevenZipEntrySettings(new SevenZipLZMACompressionSettings(), new SevenZipAESEncryptionSettings("test1")));
        archive.Save(sevenZipFile);
    }
}
//ExEnd: ArchiveWithEncryptedEntry
Console.WriteLine("Successfully Created a Seven Zip File with AES Encryption Settings");
```

**Пояснение:**  
На этом шаге мы открываем (или создаём) файл **archive.7z**, добавляем одну запись **entry1.bin** и применяем **AES‑шифрование** с паролем `"test1"`. Объект `SevenZipLZMACompressionSettings` управляет алгоритмом сжатия, а `SevenZipAESEncryptionSettings` — шифрованием. При необходимости можно повторять вызов `CreateEntry`, добавляя новые файлы, каждый со своей конфигурацией шифрования.

### Как шифровать отдельные zip‑записи
Если требуется **шифровать zip‑записи** разными паролями, просто создавайте новый `SevenZipAESEncryptionSettings` для каждого вызова `CreateEntry`.

### Как защитить 7z‑файлы паролем
Пароль, указанный в `SevenZipAESEncryptionSettings`, выступает в роли **защиты паролем** для всего архива. При открытии архива необходимо предоставить тот же пароль.

## Часто задаваемые вопросы

**В: Как добавить несколько зашифрованных записей в один 7z‑архив?**  
О: Вызывайте `archive.CreateEntry` многократно, передавая отдельный `SevenZipAESEncryptionSettings` для каждой записи, если нужны разные пароли.

**В: Поддерживает ли Aspose.Zip потоковую работу с большими файлами без полной загрузки в память?**  
О: Да, вы можете передать `FileStream` или любой другой поток в `CreateEntry`, что позволяет эффективно работать с большими файлами.

**В: Можно ли расшифровать существующий 7z‑архив с помощью Aspose.Zip?**  
О: Используйте конструктор `SevenZipArchive`, принимающий поток, и укажите правильный пароль через `SevenZipAESEncryptionSettings`.

**В: Можно ли задать уровень сжатия для каждой записи отдельно?**  
О: Да, настройте `SevenZipLZMACompressionSettings` для каждой записи перед вызовом `CreateEntry`.
