# Как хранить файлы без сжатия с помощью Aspose.Zip для .NET

В современном .NET‑разработке, **как хранить файлы** эффективно, может существенно влиять на производительность и затраты на хранение. Когда необходимо сохранять файлы точно в оригинальном виде — без какого‑либо сжатия — Aspose.Zip для .NET предоставляет чистый, простой API. В этом руководстве мы пройдем шаги по созданию архива ZIP без сжатия, сохранению файлов в ZIP и интеграции решения в ваше приложение.

## Быстрые ответы
- **Что означает «uncompressed zip»?** Это ZIP‑архив, в котором каждая запись сохраняется методом «store», оставляя оригинальные байты файла нетронутыми.  
- **Зачем избегать сжатия?** Для ускорения архивирования, сохранения оригинальных размеров файлов для последующей обработки или соблюдения нормативных требований, запрещающих изменение данных.  
- **Какой класс Aspose.Zip отвечает за это?** `ArchiveEntrySettings` в сочетании с `StoreCompressionSettings`.  
- **Нужна ли лицензия?** Бесплатная trial‑версия подходит для тестирования; для продакшна требуется коммерческая лицензия.  
- **Поддерживаемые версии .NET?** .NET Framework, .NET Core, .NET 5/6/7 и более новые.

## Что значит хранить несколько файлов без сжатия?
Хранение нескольких файлов без сжатия означает добавление каждого файла в контейнер ZIP с использованием метода сжатия *store*. Файлы остаются байт‑в‑байт идентичными оригиналам, что идеально, когда требуется быстрое архивирование или необходимо сохранить данные без изменений.

## Почему использовать Aspose.Zip для архивов без сжатия?
- **Скорость:** Не запускаются процессорозатратные алгоритмы сжатия.  
- **Предсказуемый размер:** Размер архива равен сумме оригинальных файлов плюс минимальные накладные расходы ZIP.  
- **Совместимость:** Полученный ZIP работает с любой стандартной утилитой распаковки.  
- **Гибкость:** При необходимости можно смешивать сжатые и несжатые записи в одном архиве.

## Предварительные требования
- **Aspose.Zip for .NET** – интегрирован в ваш проект. Смотрите официальную [documentation](https://reference.aspose.com/zip/net/) для шагов установки.  
- **.NET Development Environment** – Visual Studio, VS Code или любой предпочитаемый IDE.  
- **Document Directory** – папка на вашем компьютере, содержащая файлы, которые вы хотите заархивировать (например, «Your Document Directory»).

## Импорт пространств имён
Перед написанием кода импортируйте необходимые пространства имён, чтобы компилятор знал, где находятся классы Aspose.

```csharp
using Aspose.Zip;
using Aspose.Zip.Saving;
using System.IO;
using System.Text;
```

## Шаг 1: Установить Document Directory
Определите путь, где находятся ваши исходные файлы. Замените заполнитель реальной папкой в вашей системе.

```csharp
string dataDir = "Your Document Directory";
```

## Шаг 2: Создать ZIP‑архив без сжатия
Суть руководства – мы создаём экземпляр `Archive`, настроенный с `StoreCompressionSettings`. Это указывает Aspose.Zip *store* (т.е. не сжимать) каждую запись.

```csharp
using (FileStream zipFile = File.Open(dataDir + "StoreMultipleFilesWithoutCompression_out.zip", FileMode.Create))
{
    FileInfo fi1 = new FileInfo(dataDir + "alice29.txt");
    FileInfo fi2 = new FileInfo(dataDir + "lcet10.txt");

    using (Archive archive = new Archive(new ArchiveEntrySettings(new StoreCompressionSettings())))
    {
        archive.CreateEntry("alice29.txt", fi1);
        archive.CreateEntry("lcet10.txt", fi2);
        archive.Save(zipFile, new ArchiveSaveOptions() { Encoding = Encoding.ASCII });
    }
}
```

> **Pro tip:** Если вам нужно **save files to zip** одновременно сжатие некоторых и оставление других без сжатия, создайте отдельные экземпляры `ArchiveEntrySettings` для каждого файла и добавьте их в один `Archive`.

## Распространённые проблемы и решения

| Проблема | Почему происходит | Решение |
|----------|-------------------|---------|
| **Файл не найден** | Неправильный путь `dataDir` или отсутствует расширение файла. | Проверьте путь и убедитесь, что файлы существуют. Используйте `Path.Combine` для более безопасного объединения. |
| **Отказ в доступе** | Процесс не имеет прав на чтение исходных файлов или запись ZIP‑архива. | Запустите приложение с необходимыми правами или выберите папку с правом записи. |
| **Неожиданный размер файла в ZIP** | Архив был создан с использованием сжатия по умолчанию. | Убедитесь, что `new StoreCompressionSettings()` передаётся в `ArchiveEntrySettings`. |

## Часто задаваемые вопросы

**Q: Могу ли я сжимать определённые типы файлов, а другие хранить без сжатия?**  
A: Да, вы можете создавать разные `ArchiveEntrySettings` для каждого файла и смешивать сжатые и несжатые записи в одном архиве.

**Q: Совместим ли Aspose.Zip for .NET с .NET Core и .NET 5/6?**  
A: Абсолютно. Библиотека поддерживает .NET Framework, .NET Core, .NET Standard и последние версии .NET.

**Q: Как следует обрабатывать исключения во время процесса архивирования?**  
A: Оберните код архивирования в блок `try‑catch` и запишите детали исключения. Это обеспечивает корректный сбой и упрощает отладку.

**Q: Поддерживает ли Aspose.Zip многопоточное архивирование?**  
A: Да, вы можете обрабатывать несколько файлов параллельно и добавлять их в архив, но помните, что объект `Archive` сам по себе не является потокобезопасным; синхронизируйте доступ при добавлении записей.

**Q: Могу ли я интегрировать Aspose.Zip в существующий проект без значительных изменений кода?**  
A: Определённо. API разработан для простого «drop‑in» использования. Обратитесь к официальной [documentation](https://reference.aspose.com/zip/net/) для рекомендаций по миграции.
