# Создание не сжатого zip .NET с Aspose.Zip для .NET

В современной разработке на .NET **создание не сжатого zip .NET** может значительно ускорить архивирование и сделать размеры файлов предсказуемыми. Когда вам нужно **упаковывать файлы без сжатия** — например, чтобы соответствовать нормативным требованиям или ускорить последующую обработку — Aspose.Zip для .NET предлагает чистый, простой API. В этом руководстве мы пошагово покажем, как создать не сжатый ZIP‑архив, добавить файлы и интегрировать решение в ваше приложение.

## Быстрые ответы
- **Что означает «не сжатый zip»?** Это ZIP‑архив, в котором каждый элемент хранится методом «store», оставляя оригинальные байты файла нетронутыми.  
- **Зачем избегать сжатия?** Чтобы ускорить архивирование, сохранить оригинальные размеры файлов для последующей обработки или соответствовать нормативным требованиям, запрещающим изменение данных.  
- **Какой класс Aspose.Zip отвечает за это?** `ArchiveEntrySettings` в сочетании с `StoreCompressionSettings`.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для тестирования; для продакшн‑использования требуется коммерческая лицензия.  
- **Поддерживаемые версии .NET?** .NET Framework, .NET Core, .NET 5/6/7 и более новые.  

## Что такое создание не сжатого zip .NET?
Создание не сжатого ZIP означает добавление каждого файла в контейнер ZIP с использованием метода сжатия *store*. Файлы остаются байт‑за‑байтом идентичными оригиналам, что идеально подходит, когда требуется быстрое архивирование или необходимо сохранить данные без изменений.

## Почему использовать Aspose.Zip для файлов zip без сжатия?
- **Скорость:** Не запускаются процессоёмкие алгоритмы сжатия.  
- **Предсказуемый размер:** Размер архива равен сумме оригинальных файлов плюс минимальные накладные расходы ZIP.  
- **Совместимость:** Полученный ZIP работает с любой стандартной утилитой распаковки.  
- **Гибкость:** При необходимости можно смешивать сжатые и несжатые записи в одном архиве.  

## Предварительные требования
- **Aspose.Zip for .NET** — интегрирован в ваш проект. См. официальную [документацию](https://reference.aspose.com/zip/net/) для шагов установки.  
- **Среда разработки .NET** — Visual Studio, VS Code или любая предпочитаемая IDE.  
- **Каталог документов** — папка на вашем компьютере, содержащая файлы, которые вы хотите заархивировать (например, «Your Document Directory»).  

## Импорт пространств имён
Перед написанием кода импортируйте необходимые пространства имён, чтобы компилятор знал, где находятся классы Aspose.

```csharp
using Aspose.Zip;
using Aspose.Zip.Saving;
using System.IO;
using System.Text;
```

## Шаг 1: Установить каталог документов
Определите путь, где находятся исходные файлы. Замените заполнитель реальным каталогом на вашей системе.

```csharp
string dataDir = "Your Document Directory";
```

## Шаг 2: Создать ZIP‑архив без сжатия
Суть руководства — мы создаём экземпляр `Archive`, настроенный с `StoreCompressionSettings`. Это указывает Aspose.Zip *хранить* (т.е. не сжимать) каждую запись.

```csharp
using (FileStream zipFile = File.Open(dataDir + "StoreMultipleFilesWithoutCompression_out.zip", FileMode.Create))
{
    FileInfo fi1 = new FileInfo(dataDir + "alice29.txt");
    FileInfo fi2 = new FileInfo(dataDir + "lcet10.txt");

    using (Archive archive = new Archive(new ArchiveEntrySettings(new StoreCompressionSettings())))
    {
        archive.CreateEntry("alice29.txt", fi1);
        archive.CreateEntry("lcet10.txt", fi2);
        archive.Save(zipFile, new ArchiveSaveOptions() { Encoding = Encoding.ASCII });
    }
}
```

> **Совет:** Если вам нужно **сохранять файлы в zip**, одновременно сжимая некоторые и оставляя другие без сжатия, создайте отдельные экземпляры `ArchiveEntrySettings` для каждого файла и добавьте их в один `Archive`.

## Распространённые проблемы и решения
| Issue | Why It Happens | Fix |
|-------|----------------|-----|
| **Файл не найден** | Неправильный путь `dataDir` или отсутствует расширение файла. | Проверьте путь и убедитесь, что файлы существуют. Используйте `Path.Combine` для более безопасного объединения. |
| **Доступ запрещён** | Процесс не имеет прав на чтение исходных файлов или запись ZIP‑архива. | Запустите приложение с необходимыми правами или выберите папку с правом записи. |
| **Неожиданный размер файла в ZIP** | Архив был создан с использованием сжатия по умолчанию. | Убедитесь, что в `ArchiveEntrySettings` передаётся `new StoreCompressionSettings()`. |

## Часто задаваемые вопросы

**Q: Могу ли я сжимать определённые типы файлов, оставляя другие без сжатия?**  
A: Да, вы можете создавать разные `ArchiveEntrySettings` для каждого файла и смешивать сжатые и несжатые записи в одном архиве.

**Q: Совместим ли Aspose.Zip for .NET с .NET Core и .NET 5/6?**  
A: Абсолютно. Библиотека поддерживает .NET Framework, .NET Core, .NET Standard и последние версии .NET.

**Q: Как следует обрабатывать исключения во время процесса архивирования?**  
A: Оберните код архивирования в блок `try‑catch` и запишите детали исключения в журнал. Это обеспечивает корректное завершение и упрощает отладку.

**Q: Поддерживает ли Aspose.Zip многопоточное архивирование?**  
A: Да, вы можете обрабатывать несколько файлов параллельно и добавлять их в архив, но помните, что объект `Archive` не является потокобезопасным; синхронизируйте доступ при добавлении записей.

**Q: Могу ли я интегрировать Aspose.Zip в существующий проект без значительных изменений кода?**  
A: Определённо. API разработан для простого внедрения. Обратитесь к официальной [документации](https://reference.aspose.com/zip/net/) для рекомендаций по миграции.
