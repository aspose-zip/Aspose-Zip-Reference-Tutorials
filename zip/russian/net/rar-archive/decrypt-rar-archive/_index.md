# Извлечение RAR в папку с помощью Aspose.Zip для .NET

## Введение

Если вам нужно **извлечь RAR в папку** и работать с архивами, защищёнными паролем, Aspose.Zip для .NET делает эту задачу простой. В этом руководстве мы пройдём по точным шагам чтения зашифрованного RAR‑файла, указания пароля RAR и извлечения содержимого архива в целевой каталог. Независимо от того, создаёте ли вы настольную утилиту или серверный сервис, вы увидите, как быстро и надёжно интегрировать логику расшифровки.

## Быстрые ответы
- **Что означает «извлечь RAR в папку»?** Это означает открытие RAR‑архива и запись каждого элемента в указанный каталог на диске.  
- **Какая библиотека обрабатывает расшифровку?** Aspose.Zip для .NET предоставляет встроенную поддержку зашифрованных RAR‑архивов.  
- **Нужна ли лицензия для тестирования?** Доступна временная лицензия для оценки; полная лицензия требуется для продакшна.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+ и .NET 5/6+.  
- **Сколько времени занимает реализация?** Обычно менее 10 минут для базового сценария извлечения.

## Что такое «извлечь RAR в папку»?
Извлечение RAR‑архива в папку означает распаковку каждого файла, хранящегося в архиве, и размещение их в выбранном вами каталоге. Когда архив зашифрован, необходимо также предоставить правильный пароль перед началом извлечения.

## Почему использовать Aspose.Zip для извлечения зашифрованного RAR?
Aspose.Zip абстрагирует сложности формата RAR, обрабатывая как стандартные, так и зашифрованные архивы без внешних зависимостей. Он предоставляет чистый объектно‑ориентированный API, высокую производительность и отличную обработку ошибок — идеальное решение для .NET‑разработчиков, которым нужен надёжный способ **как расшифровать RAR** файлы.

## Предварительные требования

Прежде чем приступить к руководству, убедитесь, что у вас есть следующие предварительные требования:

1. Библиотека Aspose.Zip для .NET: Убедитесь, что библиотека Aspose.Zip установлена в вашем .NET‑проекте. Вы можете скачать её из [документации Aspose.Zip](https://reference.aspose.com/zip/net/).
2. Каталог документов: Создайте каталог, где находится ваш зашифрованный RAR‑архив. Замените «Your Document Directory» в примере кода фактическим путём к этому каталогу.

## Импорт пространств имён

Начнём с импорта необходимых пространств имён для эффективного использования библиотеки Aspose.Zip. Добавьте следующие строки в начало вашего .NET‑файла:

```csharp
//ExStart: ImportNamespaces
using Aspose.Zip;
using System.IO;
//ExEnd: ImportNamespaces
```

## Шаг 1 – Открытие зашифрованного RAR‑архива

Сначала откройте поток только для чтения зашифрованного RAR‑файла. Это подготовит файл к расшифровке и извлечению.

```csharp
//ExStart: DecryptRarArchive_Step1
using (FileStream fs = File.OpenRead(dataDir + "encrypted.rar"))
{
    //ExEnd: DecryptRarArchive_Step1
    // Continue to the next step here...
}
```

## Шаг 2 – Указание пароля RAR (как расшифровать RAR)

Теперь создайте экземпляр `RarArchive` и укажите Aspose.Zip пароль, защищающий архив. Замените `"p@s$"` реальным паролем, который вы использовали при создании зашифрованного RAR.

```csharp
//ExStart: DecryptRarArchive_Step2
using (RarArchive archive = new RarArchive(fs, new RarArchiveLoadOptions() { DecryptionPassword = "p@s$" }))
{
    //ExEnd: DecryptRarArchive_Step2
    // Continue to the next step here...
}
```

## Шаг 3 – Извлечение содержимого в папку (извлечение зашифрованного RAR)

Наконец, извлеките каждый элемент в выбранную вами папку. Это завершает операцию **извлечения RAR в папку**.

```csharp
//ExStart: DecryptRarArchive_Step3
archive.ExtractToDirectory(dataDir + "DecompressRar_out");
//ExEnd: DecryptRarArchive_Step3
```

Повторите эти шаги для каждого RAR‑архива, который нужно расшифровать, обеспечивая бесшовную интеграцию Aspose.Zip для .NET в ваш проект.

## Распространённые подводные камни и советы

- **Неправильный пароль** – Если пароль неверен, Aspose.Zip бросает `WrongPasswordException`. Проверьте строку, которую вы передаёте в `DecryptionPassword`.
- **Большие архивы** – Для очень больших RAR‑файлов рассмотрите возможность сначала извлечь их во временную папку, а затем переместить файлы в окончательное место, чтобы избежать нехватки места на диске.
- **Безопасность путей** – Всегда проверяйте `dataDir` и пути вывода, чтобы предотвратить уязвимости типа обхода каталогов.
