### Как сжать папку

1. **Создайте объект `ZipPackage`** — он представляет архив, который вы собираетесь создать.  
2. **Добавьте целевой каталог** с помощью метода `AddFolder`, который автоматически включает вложенные подпапки и файлы.  
3. **Сохраните архив** в нужное место с расширением `.zip`.

> *Примечание:* Фактический C#‑код для этих шагов уже предоставлен на связанной странице учебника «Effortless Directory Compression».

## Как защитить zip‑архив паролем в .NET

Чтобы добавить пароль к вашему zip‑файлу, просто задайте свойство `Password` у объекта `ZipPackage` перед вызовом `Save`. Aspose.Zip поддерживает надёжное шифрование AES‑256, гарантируя, что только пользователи с правильным паролем смогут извлечь содержимое. Это самый простой способ **password protect zip archive** файлов при сохранении высокой скорости сжатия.

## Распространённые подводные камни и советы

- **Большие файлы:** При работе с файлами более 2 ГБ включайте режим **Zip64**, чтобы избежать ограничений по размеру.  
- **Проблемы с длиной пути:** Используйте свойство `UseLongFileNames`, если иерархия папок превышает лимит Windows в 260 символов.  
- **Совет по производительности:** Установите `CompressionLevel` в `Fast` для быстрых сборок или в `Maximum`, когда нужен минимальный размер архива.  
- **Защита паролем:** Не забывайте хранить пароли безопасно; рассмотрите использование Azure Key Vault или аналогичных сервисов управления секретами.

## Часто задаваемые вопросы

**Q: Можно ли создать zip‑архив, защищённый паролем, используя Aspose.Zip?**  
A: Да. При сохранении архива вы можете указать `ZipPassword` и выбрать алгоритм шифрования, например AES‑256.

**Q: Поддерживает ли Aspose.Zip потоковую работу с большими файлами без полной загрузки их в память?**  
A: Абсолютно. Вы можете работать с объектами `FileStream`, позволяя эффективно сжимать или извлекать файлы любого размера.

**Q: Что делать, если нужно разбить большой архив на несколько частей?**  
A: Используйте метод `SplitArchive`, задав максимальный размер части; Aspose.Zip автоматически создаст последовательные файлы‑части.

**Q: Можно ли добавить файлы в уже существующий zip‑архив?**  
A: Да. Откройте архив в режиме `Update` и вызовите `AddFile` или `AddFolder` для добавления нового содержимого.
