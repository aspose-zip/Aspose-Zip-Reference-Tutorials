# Сжатие каталога в Zip и распаковка – Aspose.Zip for .NET

Если вам нужно **сжать каталог в zip** и затем извлечь этот архив в приложении .NET, вы попали по адресу. В этом руководстве мы пройдем весь процесс — от создания zip‑архива до чистой пошаговой распаковки с помощью Aspose.Zip for .NET. К концу вы получите переиспользуемый шаблон для .NET‑проектов и полное понимание того, как распаковывать в стиле .NET.

## Быстрые ответы
- **Что значит «compress directory to zip»?** Это превращение содержимого папки в один .zip‑файл.  
- **Как извлечь zip в каталог?** Используйте метод `Archive.ExtractToDirectory`, как показано в руководстве.  
- **Какие версии .NET поддерживаются?** Все современные .NET Framework, .NET Core и .NET 5/6+ версии.  
- **Нужна ли лицензия для продакшна?** Да, для использования без ограничений требуется коммерческая лицензия Aspose.Zip.  
- **Можно ли автоматизировать процесс в CI/CD?** Конечно — добавьте тот же код в скрипты сборки.

## Что такое «compress directory to zip»?
Сжатие каталога в zip объединяет каждый файл и подпапку в один сжатый архив. Это уменьшает размер хранилища, упрощает передачу и является стандартным способом упаковки ресурсов для развертывания.

## Почему стоит использовать Aspose.Zip for .NET?
Aspose.Zip предоставляет **чисто управляемый** API, который работает без нативных зависимостей, поддерживает большие файлы и обеспечивает высокопроизводительное zip‑сжатие .NET. Он также автоматически обрабатывает такие случаи, как архивы с паролем и имена файлов в Unicode.

## Предварительные требования
- Библиотека **Aspose.Zip for .NET** установлена (скачайте её [здесь](https://releases.aspose.com/zip/net/)).  
- Папка на диске, которую вы хотите заархивировать — укажите её путь в переменной `dataDir`.  
- Среда разработки .NET (Visual Studio, VS Code или любой другой IDE).

## Импорт пространств имён
Сначала подключим необходимые пространства имён:

```csharp
using Aspose.Zip;
using System.IO;
```

## Шаг 1: Сжатие каталога в Zip
Мы создадим zip‑файл из каталога, который позже будете распаковывать. Вспомогательный метод `CompressDirectory.Run()` выполнит всю работу.

```csharp
string dataDir = "Your Document Directory";
CompressDirectory.Run();
```

> **Совет:** Пример `CompressDirectory` упаковывает каждый файл из `dataDir` в `CompressDirectory_out.zip`. При желании переименуйте выходной файл согласно вашим конвенциям.

## Шаг 2: Распаковка папки (Как unzip .NET)

### Шаг 2.1: Открытие zip‑файла
Откройте сгенерированный архив с помощью `FileStream`. Это подготовит файл к чтению.

```csharp
using (FileStream zipFile = File.Open(dataDir + "CompressDirectory_out.zip", FileMode.Open))
{
```

### Шаг 2.2: Создание экземпляра Archive
Создайте объект `Archive`, который представляет контейнер zip.

```csharp
    using (var archive = new Archive(zipFile))
    {
```

### Шаг 2.3: Извлечение в каталог
Наконец, извлеките содержимое в новую папку. Это шаг **extract zip to directory**.

```csharp
        archive.ExtractToDirectory(dataDir + "DecompressFolder_out");
    }
}
```

Поздравляем! Вы успешно **сжали каталог в zip** и затем **извлекли zip в каталог** с помощью Aspose.Zip for .NET. Такой подход гарантирует целостность данных, оставаясь при этом лаконичным и читаемым.

## Распространённые проблемы и решения
| Симптом | Возможная причина | Решение |
|---------|-------------------|---------|
| `UnauthorizedAccessException` при извлечении | Целевой каталог только для чтения или используется | Убедитесь, что путь назначения доступен для записи и не заблокирован |
| Пустой каталог после извлечения | Неправильный путь к исходному zip‑файлу | Проверьте, что `dataDir + "CompressDirectory_out.zip"` указывает на правильный файл |
| Большие файлы вызывают OutOfMemoryException | Используется буфер по умолчанию для очень больших архивов | Примените `ArchiveOptions` для увеличения размера буфера или считывайте файлы частями |
