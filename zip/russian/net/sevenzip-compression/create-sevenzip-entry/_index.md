# Создание зашифрованного архива 7z с помощью Aspose.Zip для .NET

## Введение

В этом руководстве вы узнаете **как создавать зашифрованные 7z**‑файлы с помощью библиотеки Aspose.Zip для .NET. Независимо от того, нужно ли вам защитить конфиденциальные данные, соответствовать политикам безопасности или просто эффективно сжать файлы, это руководство проведёт вас через каждый шаг — от настройки проекта до подтверждения успешного создания архива. Давайте посмотрим, как легко добавить файл в 7z‑архив с шифрованием AES.

## Быстрые ответы
- **Что означает «создать зашифрованный 7z»?** Это создание 7‑zip‑архива, защищённого шифрованием AES.
- **Какая библиотека используется?** Aspose.Zip для .NET.
- **Нужна ли лицензия?** Для тестирования достаточно временной лицензии; для продакшн‑использования требуется полная лицензия.
- **Можно ли добавить несколько файлов?** Да, можно вызывать `CreateEntry` последовательно для каждого файла.
- **Поддерживается ли шифрование AES?** Да, Aspose.Zip поддерживает шифрование AES‑256 для 7z‑архивов.

## Что такое зашифрованный 7z‑архив?
7z‑архив — это контейнерный формат с высоким уровнем сжатия. При **создании зашифрованных 7z**‑архивов содержимое перемешивается с помощью шифрования AES, делая его нечитаемым без правильного пароля. Это идеальное решение для безопасной передачи или хранения конфиденциальных файлов.

## Почему стоит использовать Aspose.Zip для зашифрованных 7z‑файлов?
- **Полная интеграция с .NET** — работает с .NET Framework, .NET Core и .NET 5/6.
- **Встроенная поддержка AES‑256** — не требуется сторонних инструментов.
- **Простой API** — однострочные вызовы для добавления файлов и сохранения архива.
- **Кроссплатформенность** — работает на Windows, Linux и macOS.

## Предварительные требования

Прежде чем начать, убедитесь, что у вас есть следующее:

- **Библиотека Aspose.Zip для .NET** — скачайте её [здесь](https://releases.aspose.com/zip/net/).
- **Папка с правом записи** на вашем компьютере, куда будет сохранён архив.
- **Исходный файл** (например, `file.dat`), который вы хотите сжать и зашифровать.

## Импорт пространств имён

Добавьте необходимое пространство имён в начало вашего C#‑файла:

```csharp
using Aspose.Zip.SevenZip;
```

## Пошаговое руководство

### Шаг 1: Определите рабочий каталог

Укажите путь к папке, содержащей исходный файл, который нужно сжать.

```csharp
string dataDir = "Your Document Directory";
```

Замените `"Your Document Directory"` на реальный путь на вашем компьютере.

### Шаг 2: Создайте зашифрованный 7z‑элемент

Суть руководства — открываем новый файловый поток, создаём `SevenZipArchive`, добавляем элемент и сохраняем архив. В примере добавляется один файл (`file.dat`) как `data.bin` внутри архива.

```csharp
//ExStart: CreateSevenZipEntry
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("data.bin", "file.dat");
        archive.Save(sevenZipFile);
    }
}
//ExEnd: CreateSevenZipEntry
```

> **Полезный совет:** Чтобы включить шифрование AES, установите свойство `Encryption` у `SevenZipArchive` перед вызовом `Save`. (Свойство опущено здесь для краткости примера.)

### Шаг 3: Подтвердите успех

Выведите дружелюбное сообщение, чтобы убедиться, что операция завершилась без ошибок.

```csharp
Console.WriteLine("Successfully Created a Seven Zip File with AES Encryption Settings");
```

### Шаг 4: Проверьте архив (по желанию)

После выполнения программы перейдите в папку, где находится `archive.7z`, и откройте её клиентом 7‑zip. Если вы включили шифрование в Шаге 2, вас попросят ввести пароль.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|----------|----------|
| **Файл не найден** | Неправильный `dataDir` или имя исходного файла | Проверьте путь и убедитесь, что `file.dat` существует. |
| **Отказ в доступе** | Недостаточно прав на запись | Запустите приложение с повышенными правами или выберите папку с правом записи. |
| **Шифрование не применено** | Отсутствуют настройки шифрования у архива | Установите `archive.Encryption = EncryptionAlgorithm.Aes256;` перед `Save`. |
