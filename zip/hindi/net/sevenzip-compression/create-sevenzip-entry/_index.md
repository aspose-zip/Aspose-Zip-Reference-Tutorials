# Aspose.Zip for .NET के साथ एन्क्रिप्टेड 7z आर्काइव बनाएं

## परिचय

इस ट्यूटोरियल में आप Aspose.Zip लाइब्रेरी for .NET का उपयोग करके **एन्क्रिप्टेड 7z** फ़ाइलें कैसे बनाते हैं, सीखेंगे। चाहे आपको संवेदनशील डेटा की सुरक्षा करनी हो, सुरक्षा नीतियों का पालन करना हो, या बस फ़ाइलों को कुशलता से संकुचित करना हो, यह गाइड आपको हर कदम के माध्यम से ले जाएगा—परियोजना सेटअप से लेकर यह पुष्टि करने तक कि आर्काइव सफलतापूर्वक बनाया गया है। चलिए देखते हैं कि AES एन्क्रिप्शन के साथ 7z आर्काइव में फ़ाइल जोड़ना कितना आसान है।

## त्वरित उत्तर
- **“create encrypted 7z” का क्या मतलब है?** इसका अर्थ है एक 7‑zip आर्काइव बनाना जो AES एन्क्रिप्शन से सुरक्षित हो।
- **कौन सी लाइब्रेरी उपयोग की जाती है?** Aspose.Zip for .NET.
- **क्या मुझे लाइसेंस चाहिए?** परीक्षण के लिए एक अस्थायी लाइसेंस पर्याप्त है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।
- **क्या मैं कई फ़ाइलें जोड़ सकता हूँ?** हाँ, आप प्रत्येक फ़ाइल के लिए `CreateEntry` को बार‑बार कॉल कर सकते हैं।
- **क्या AES एन्क्रिप्शन समर्थित है?** हाँ, Aspose.Zip 7z आर्काइव के लिए AES‑256 एन्क्रिप्शन का समर्थन करता है।

## एन्क्रिप्टेड 7z आर्काइव क्या है?

7z आर्काइव एक उच्च‑संपीड़न कंटेनर फ़ॉर्मेट है। जब आप **एन्क्रिप्टेड 7z** आर्काइव बनाते हैं, तो सामग्री को AES एन्क्रिप्शन द्वारा बेतरतीब किया जाता है, जिससे सही पासवर्ड के बिना इसे पढ़ा नहीं जा सकता। यह गोपनीय फ़ाइलों को सुरक्षित रूप से ट्रांसमिट या स्टोर करने के लिए आदर्श है।

## एन्क्रिप्टेड 7z फ़ाइलों के लिए Aspose.Zip क्यों उपयोग करें?

- **पूर्ण .NET एकीकरण** – .NET Framework, .NET Core, और .NET 5/6 के साथ काम करता है।
- **इन‑बिल्ट AES‑256 समर्थन** – बाहरी टूल्स की आवश्यकता नहीं।
- **सरल API** – फ़ाइलें जोड़ने और आर्काइव सहेजने के लिए एक‑लाइन कॉल्स।
- **क्रॉस‑प्लेटफ़ॉर्म** – Windows, Linux, और macOS पर चलता है।

- **Aspose.Zip for .NET लाइब्रेरी** – इसे [यहाँ](https://releases.aspose.com/zip/net/) डाउनलोड करें।

- **एक स्रोत फ़ाइल** (जैसे `file.dat`) जिसे आप संकुचित और एन्क्रिप्ट करना चाहते हैं।

using Aspose.Zip.SevenZip;

## चरण‑दर‑चरण गाइड

### चरण 1: कार्य निर्देशिका निर्धारित करें

उस फ़ोल्डर का पथ सेट करें जिसमें वह स्रोत फ़ाइल है जिसे आप संकुचित करना चाहते हैं।

```csharp
string dataDir = "Your Document Directory";
```

`"Your Document Directory"` को अपने मशीन पर वास्तविक पथ से बदलें।

### चरण 2: एन्क्रिप्टेड 7z एंट्री बनाएं

ट्यूटोरियल का मुख्य भाग – हम एक नया फ़ाइल स्ट्रीम खोलते हैं, एक `SevenZipArchive` बनाते हैं, एक एंट्री जोड़ते हैं, और आर्काइव सहेजते हैं। यह उदाहरण एक फ़ाइल (`file.dat`) को आर्काइव के भीतर `data.bin` के रूप में जोड़ता है।

```csharp
//ExStart: CreateSevenZipEntry
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("data.bin", "file.dat");
        archive.Save(sevenZipFile);
    }
}
//ExEnd: CreateSevenZipEntry
```

> **प्रो टिप:** AES एन्क्रिप्शन सक्षम करने के लिए, `Save` कॉल करने से पहले `SevenZipArchive` पर `Encryption` प्रॉपर्टी सेट करें। (उदाहरण को संक्षिप्त रखने के लिए यहाँ प्रॉपर्टी को छोड़ दिया गया है।)

### चरण 3: सफलता की पुष्टि करें

एक मैत्रीपूर्ण संदेश प्रिंट करें ताकि आपको पता चले कि ऑपरेशन बिना त्रुटियों के पूरा हो गया।

```csharp
Console.WriteLine("Successfully Created a Seven Zip File with AES Encryption Settings");
```

### चरण 4: आर्काइव सत्यापित करें (वैकल्पिक)

प्रोग्राम चलने के बाद, `archive.7z` वाले फ़ोल्डर में जाएँ और इसे 7‑zip क्लाइंट से खोलने का प्रयास करें। यदि आपने चरण 2 में एन्क्रिप्शन जोड़ा है तो आपको पासवर्ड के लिए प्रॉम्प्ट किया जाएगा।

## अक्सर पूछे जाने वाले प्रश्न

### Q: क्या मैं Aspose.Zip for .NET को अन्य आर्काइव फ़ॉर्मेट्स के साथ उपयोग कर सकता हूँ?
Aspose.Zip मुख्य रूप से ZIP और 7z फ़ॉर्मेट पर केंद्रित है। अन्य फ़ॉर्मेट को संभालने के लिए, उन फ़ॉर्मेट के लिए विशेष रूप से तैयार लाइब्रेरीज़ देखें।

### Q: क्या Aspose.Zip बड़े‑पैमाने के अनुप्रयोगों के लिए उपयुक्त है?
