# Aspose.Zip for .NET के साथ 7z आर्काइव को एन्क्रिप्ट कैसे करें

## परिचय

इस ट्यूटोरियल में आप **7z** फ़ाइलों को Aspose.Zip लाइब्रेरी for .NET का उपयोग करके **कैसे एन्क्रिप्ट करें** सीखेंगे। चाहे आपको संवेदनशील डेटा की सुरक्षा करनी हो, सुरक्षा नीतियों का पालन करना हो, या सिर्फ फ़ाइलों को कुशलतापूर्वक संकुचित करना हो, यह गाइड प्रोजेक्ट सेटअप से लेकर यह पुष्टि करने तक के हर चरण को कवर करता है कि आर्काइव सफलतापूर्वक बनाया गया है। चलिए देखते हैं कि **AES एन्क्रिप्शन के साथ 7z में फ़ाइल जोड़ना** कितना आसान है और विश्वसनीय 7z आर्काइव कैसे जनरेट करें।

## त्वरित उत्तर
- **“एन्क्रिप्टेड 7z बनाना” का क्या अर्थ है?** इसका मतलब है AES एन्क्रिप्शन के साथ संरक्षित 7‑zip आर्काइव बनाना।  
- **कौन सी लाइब्रेरी उपयोग की जाती है?** Aspose.Zip for .NET।  
- **क्या मुझे लाइसेंस चाहिए?** परीक्षण के लिए एक अस्थायी लाइसेंस पर्याप्त है; उत्पादन के लिए पूर्ण लाइसेंस आवश्यक है।  
- **क्या मैं कई फ़ाइलें जोड़ सकता हूँ?** हाँ—`CreateEntry` को बार‑बार कॉल करके **7z में कई फ़ाइलें जोड़ें**।  
- **क्या AES एन्क्रिप्शन समर्थित है?** हाँ, Aspose.Zip **AES‑256 सेट करने** के लिए समर्थन प्रदान करता है।  

## एन्क्रिप्टेड 7z आर्काइव क्या है?
7z आर्काइव एक हाई‑कम्प्रेशन कंटेनर फॉर्मेट है। जब आप **एन्क्रिप्टेड 7z** बनाते हैं, तो सामग्री को AES एन्क्रिप्शन द्वारा स्क्रैम्बल किया जाता है, जिससे सही पासवर्ड के बिना इसे पढ़ा नहीं जा सकता। यह गोपनीय फ़ाइलों को सुरक्षित रूप से ट्रांसमिट या स्टोर करने के लिए आदर्श है।

## Aspose.Zip को एन्क्रिप्टेड 7z फ़ाइलों के लिए क्यों चुनें?
- **पूर्ण .NET इंटीग्रेशन** – .NET Framework, .NET Core, और .NET 5/6 के साथ काम करता है।  
- **इन‑बिल्ट AES‑256 समर्थन** – बाहरी टूल की आवश्यकता नहीं; आप आसानी से **AES सेट करना** सीख सकते हैं।  
- **सरल API** – **7z में फ़ाइल जोड़ें** और आर्काइव सहेजने के लिए एक‑लाइन कॉल्स।  
- **क्रॉस‑प्लेटफ़ॉर्म** – Windows, Linux, और macOS पर चलता है।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

- **Aspose.Zip for .NET लाइब्रेरी** – इसे [यहाँ](https://releases.aspose.com/zip/net/) से डाउनलोड करें।  
- **एक लिखने योग्य फ़ोल्डर** जहाँ आर्काइव सहेजा जाएगा।  
- **एक स्रोत फ़ाइल** (जैसे, `file.dat`) जिसे आप संकुचित और एन्क्रिप्ट करना चाहते हैं।

## नेमस्पेस आयात करें

अपने C# फ़ाइल के शीर्ष पर आवश्यक नेमस्पेस जोड़ें:

```csharp
using Aspose.Zip.SevenZip;
```

## चरण‑दर‑चरण गाइड

### चरण 1: कार्य निर्देशिका निर्धारित करें

उस फ़ोल्डर का पथ सेट करें जिसमें वह स्रोत फ़ाइल है जिसे आप संकुचित करना चाहते हैं।

```csharp
string dataDir = "Your Document Directory";
```

`"Your Document Directory"` को अपने मशीन पर वास्तविक पथ से बदलें।

### चरण 2: एन्क्रिप्टेड 7z एंट्री बनाएं

ट्यूटोरियल का मुख्य भाग – हम नई फ़ाइल स्ट्रीम खोलते हैं, `SevenZipArchive` बनाते हैं, एक एंट्री जोड़ते हैं, और आर्काइव सहेजते हैं। यह उदाहरण एक फ़ाइल (`file.dat`) को आर्काइव के अंदर `data.bin` के रूप में जोड़ता है।

```csharp
//ExStart: CreateSevenZipEntry
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("data.bin", "file.dat");
        archive.Save(sevenZipFile);
    }
}
//ExEnd: CreateSevenZipEntry
```

> **प्रो टिप:** AES एन्क्रिप्शन सक्षम करने के लिए `SevenZipArchive` पर `Encryption` प्रॉपर्टी सेट करें, फिर `Save` कॉल करें। (उदाहरण को संक्षिप्त रखने के लिए यहाँ प्रॉपर्टी छोड़ी गई है।)

### चरण 3: सफलता की पुष्टि करें

एक मैत्रीपूर्ण संदेश प्रिंट करें ताकि आपको पता चले कि ऑपरेशन बिना त्रुटियों के पूरा हो गया।

```csharp
Console.WriteLine("Successfully Created a Seven Zip File with AES Encryption Settings");
```

### चरण 4: आर्काइव सत्यापित करें (वैकल्पिक)

प्रोग्राम चलने के बाद, `archive.7z` वाले फ़ोल्डर में जाएँ और उसे 7‑zip क्लाइंट से खोलने की कोशिश करें। यदि आपने चरण 2 में एन्क्रिप्शन जोड़ा है, तो आपको पासवर्ड के लिए प्रॉम्प्ट मिलेगा। यह चरण आपको **7z पासवर्ड सत्यापित** करने में भी मदद करता है।

## सामान्य समस्याएँ और समाधान

| समस्या | कारण | समाधान |
|-------|-------|-----|
| **फ़ाइल नहीं मिली** | गलत `dataDir` या स्रोत फ़ाइल का नाम | पथ को दोबारा जांचें और सुनिश्चित करें कि `file.dat` मौजूद है। |
| **पहुँच अस्वीकृत** | अपर्याप्त लिखने की अनुमति | एप्लिकेशन को उन्नत अधिकारों के साथ चलाएँ या लिखने योग्य फ़ोल्डर चुनें। |
| **एन्क्रिप्शन लागू नहीं हुआ** | आर्काइव पर एन्क्रिप्शन सेटिंग्स नहीं दी गईं | `Save` से पहले `archive.Encryption = EncryptionAlgorithm.Aes256;` सेट करें। |

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं एक ही 7z आर्काइव में एक से अधिक फ़ाइलें जोड़ सकता हूँ?**  
उत्तर: बिल्कुल। प्रत्येक फ़ाइल के लिए `archive.CreateEntry` कॉल करें जिससे आप **7z में फ़ाइल जोड़ें** या **7z में कई फ़ाइलें जोड़ें**।

**प्रश्न: AES एन्क्रिप्शन के लिए पासवर्ड कैसे निर्दिष्ट करें?**  
उत्तर: `SevenZipArchive` पर `Password` प्रॉपर्टी सेट करें, उदाहरण के लिए `archive.Password = "YourStrongPassword";`। इससे बाद में **7z पासवर्ड सत्यापित** करने में मदद मिलती है।

**प्रश्न: क्या Aspose.Zip अन्य आर्काइव फॉर्मेट्स को सपोर्ट करता है?**  
उत्तर: Aspose.Zip मुख्यतः ZIP और 7z फॉर्मेट्स पर केंद्रित है। अन्य फॉर्मेट्स के लिए विशेष लाइब्रेरी पर विचार करें।
