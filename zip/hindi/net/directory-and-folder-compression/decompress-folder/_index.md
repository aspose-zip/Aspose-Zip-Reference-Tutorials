# डायरेक्टरी को ज़िप में संकुचित करें और डिकम्प्रेस करें – Aspose.Zip for .NET

यदि आपको **डायरेक्टरी को ज़िप में संकुचित** करना है और फिर उस आर्काइव को .NET एप्लिकेशन में निकालना है, तो आप सही जगह पर आए हैं। इस ट्यूटोरियल में हम पूरे वर्कफ़्लो को चरण‑बद्ध तरीके से देखेंगे—पहले ज़िप आर्काइव बनाना, फिर Aspose.Zip for .NET का उपयोग करके साफ़‑सुथरा अनज़िप करना। अंत तक आपके पास .NET प्रोजेक्ट्स के लिए ज़िप कॉम्प्रेशन का पुन: उपयोग योग्य पैटर्न और .NET शैली में अनज़िप करने की ठोस समझ होगी।

## त्वरित उत्तर
- **“डायरेक्टरी को ज़िप में संकुचित” का क्या मतलब है?** इसका अर्थ है फ़ोल्डर की सभी सामग्री को एकल .zip फ़ाइल में बदलना।  
- **मैं ज़िप को डायरेक्टरी में कैसे निकालूँ?** गाइड में दिखाए अनुसार `Archive.ExtractToDirectory` मेथड का उपयोग करें।  
- **कौन से .NET संस्करण समर्थित हैं?** सभी आधुनिक .NET Framework, .NET Core, और .NET 5/6+ संस्करण।  
- **प्रोडक्शन के लिए लाइसेंस आवश्यक है?** हाँ, गैर‑ट्रायल उपयोग के लिए एक व्यावसायिक Aspose.Zip लाइसेंस आवश्यक है।  
- **क्या इसे CI/CD पाइपलाइन में ऑटोमेट किया जा सकता है?** बिल्कुल—सिर्फ वही कोड अपने बिल्ड स्क्रिप्ट्स में जोड़ें।

## “डायरेक्टरी को ज़िप में संकुचित” क्या है?
डायरेक्टरी को ज़िप में संकुचित करने से प्रत्येक फ़ाइल और सब‑फ़ोल्डर एकल संकुचित आर्काइव में बंडल हो जाते हैं। इससे स्टोरेज आकार घटता है, ट्रांसफ़र आसान होता है, और डिप्लॉयमेंट के लिए संसाधनों को पैकेज करने का मानक तरीका मिलता है।

## Aspose.Zip for .NET क्यों उपयोग करें?
Aspose.Zip एक **pure‑managed** API प्रदान करता है जो नेटिव डिपेंडेंसीज़ के बिना काम करता है, बड़े फ़ाइलों को सपोर्ट करता है, और उच्च‑प्रदर्शन ज़िप कॉम्प्रेशन .NET क्षमताएँ देता है। यह पासवर्ड‑प्रोटेक्टेड आर्काइव और Unicode फ़ाइल नामों जैसे एज केस भी बॉक्स से बाहर संभालता है।

## पूर्वापेक्षाएँ
- **Aspose.Zip for .NET** लाइब्रेरी स्थापित हो (इसे [यहाँ](https://releases.aspose.com/zip/net/) से डाउनलोड करें)।  
- डिस्क पर वह फ़ोल्डर जो आप आर्काइव करना चाहते हैं – इसका पाथ `dataDir` वेरिएबल में सेट करें।  
- .NET विकास पर्यावरण (Visual Studio, VS Code, या कोई भी पसंदीदा IDE)।

## नेमस्पेस इम्पोर्ट करें
सबसे पहले, आवश्यक नेमस्पेस को स्कोप में लाएँ:

```csharp
using Aspose.Zip;
using System.IO;
```

## चरण 1: डायरेक्टरी को ज़िप में संकुचित करें
हम उस डायरेक्टरी से ज़िप फ़ाइल बनाएँगे जिसे आप बाद में डिकम्प्रेस करेंगे। `CompressDirectory.Run()` हेल्पर यह काम संभालता है।

```csharp
string dataDir = "Your Document Directory";
CompressDirectory.Run();
```

> **प्रो टिप:** `CompressDirectory` सैंपल `dataDir` की हर फ़ाइल को `CompressDirectory_out.zip` में पैक करता है। आउटपुट फ़ाइल का नाम अपनी नेमिंग कन्वेंशन के अनुसार बदल सकते हैं।

## चरण 2: फ़ोल्डर को डिकम्प्रेस करें (कैसे अनज़िप .NET)

### चरण 2.1: ज़िप फ़ाइल खोलें
जनरेटेड आर्काइव को `FileStream` के साथ खोलें। यह फ़ाइल को पढ़ने के लिए तैयार करता है।

```csharp
using (FileStream zipFile = File.Open(dataDir + "CompressDirectory_out.zip", FileMode.Open))
{
```

### चरण 2.2: आर्काइव इंस्टेंस बनाएं
`Archive` ऑब्जेक्ट को इंस्टैंशिएट करें, जो ज़िप कंटेनर का प्रतिनिधित्व करता है।

```csharp
    using (var archive = new Archive(zipFile))
    {
```

### चरण 2.3: डायरेक्टरी में एक्सट्रैक्ट करें
अंत में, सामग्री को नई फ़ोल्डर में एक्सट्रैक्ट करें। यही **ज़िप को डायरेक्टरी में एक्सट्रैक्ट** करने का चरण है।

```csharp
        archive.ExtractToDirectory(dataDir + "DecompressFolder_out");
    }
}
```

बधाई हो! आपने सफलतापूर्वक **डायरेक्टरी को ज़िप में संकुचित** किया और फिर Aspose.Zip for .NET का उपयोग करके **ज़िप को डायरेक्टरी में एक्सट्रैक्ट** किया। यह तरीका डेटा इंटीग्रिटी सुनिश्चित करता है जबकि कोड को संक्षिप्त और पठनीय रखता है।

## सामान्य समस्याएँ और समाधान
| लक्षण | संभावित कारण | समाधान |
|---------|--------------|-----|
| `UnauthorizedAccessException` एक्सट्रैक्शन के दौरान | टार्गेट फ़ोल्डर रीड‑ओनली है या उपयोग में है | सुनिश्चित करें कि गंतव्य पाथ लिखने योग्य है और लॉक नहीं है |
| एक्सट्रैक्शन के बाद आउटपुट फ़ोल्डर खाली | गलत स्रोत ज़िप पाथ | दोबारा जांचें कि `dataDir + "CompressDirectory_out.zip"` सही फ़ाइल की ओर इशारा कर रहा है |
| बड़े फ़ाइलों पर OutOfMemoryException | बहुत बड़े आर्काइव पर डिफ़ॉल्ट बफ़र साइज का उपयोग | `ArchiveOptions` से बफ़र साइज बढ़ाएँ या फ़ाइलों को चंक्स में स्ट्रीम करें |
