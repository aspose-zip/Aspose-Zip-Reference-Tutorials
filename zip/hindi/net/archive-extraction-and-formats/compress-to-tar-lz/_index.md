# Aspose.Zip for .NET के साथ कई फ़ाइलों को tar के रूप में संपीड़ित कैसे करें

आधुनिक .NET विकास में, फ़ाइलों को कुशलतापूर्वक पैकेज करना डिप्लॉयमेंट आकार और नेटवर्क ट्रांसफ़र समय को काफी सुधार सकता है। **Compress multiple files tar** तब अक्सर आवश्यक होता है जब आपको बैकअप, वितरण, या क्लाउड अपलोड के लिए हल्का, LZ‑संपीड़ित TAR आर्काइव चाहिए। इस ट्यूटोरियल में हम Aspose.Zip लाइब्रेरी का उपयोग करके एक स्पष्ट, चरण‑दर‑चरण **tar.lz compression example** दिखाएंगे, ताकि आप जल्दी से अपने अनुप्रयोगों में **tar.lz archive** बना सकें।

## त्वरित उत्तर
- **मैं कौन सी लाइब्रेरी उपयोग करूँ?** Aspose.Zip for .NET.  
- **इम्प्लीमेंटेशन में कितना समय लगेगा?** बेसिक उदाहरण के लिए लगभग 5‑10 मिनट।  
- **क्या मुझे लाइसेंस चाहिए?** परीक्षण के लिए फ्री ट्रायल काम करता है; प्रोडक्शन के लिए एक कमर्शियल लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.  
- **क्या मैं एक साथ कई फ़ाइलें संपीड़ित कर सकता हूँ?** हाँ – सहेजने से पहले अधिक एंट्री जोड़ें।

## कई फ़ाइलों को tar के रूप में संपीड़ित कैसे करें
यह अनुभाग सीधे मुख्य कीवर्ड को संबोधित करता है और आपको Aspose.Zip का उपयोग करके **compress multiple files tar** करने के सटीक चरण दिखाता है। यह तरीका किसी भी संख्या में फ़ाइलों के लिए काम करता है, और आप इसे अपनी फ़ोल्डर संरचना के अनुसार आसानी से अनुकूलित कर सकते हैं।

## tar.lz संपीड़न क्या है?
`tar.lz` एक TAR आर्काइव है जिसे LZMA एल्गोरिदम (अक्सर केवल **LZ** कहा जाता है) से संपीड़ित किया गया है। यह TAR की सरलता को LZ के उच्च संपीड़न अनुपात के साथ जोड़ता है, जिससे यह बैकअप फ़ाइलों, पैकेज वितरण, या किसी भी ऐसे परिदृश्य के लिए आदर्श बनता है जहाँ बैंडविड्थ महत्वपूर्ण है।

## .NET के लिए Aspose.Zip क्यों उपयोग करें?
* **कोई बाहरी निर्भरताएँ नहीं** – शुद्ध .NET इम्प्लीमेंटेशन।  
* **क्रॉस‑प्लेटफ़ॉर्म समर्थन** – Windows, Linux, और macOS पर काम करता है।  
* **इन‑बिल्ट LZ संपीड़न** – अतिरिक्त नेटिव टूल्स स्थापित करने की आवश्यकता नहीं।  
* **मज़बूत त्रुटि संभाल** – एक्सेप्शन वर्णनात्मक होते हैं, जिससे डिबगिंग आसान हो जाता है।

## नेमस्पेस आयात करें
आवश्यक नेमस्पेस जोड़ें ताकि कंपाइलर को पता चले कि हम जिन क्लासों का उपयोग करेंगे वे कहाँ स्थित हैं।

```csharp
using System;
using Aspose.Zip.Tar;
```

## tar.lz आर्काइव कैसे बनाएं – चरण‑दर‑चरण गाइड

### चरण 1: एक फ़ाइल को संपीड़ित करें
पहला उदाहरण सबसे बुनियादी स्थिति दिखाता है – एक फ़ाइल को TAR आर्काइव में जोड़ना और फिर इसे **tar.lz** फ़ाइल के रूप में सहेजना।

```csharp
//ExStart: CompressSingleFile
using (TarArchive archive = new TarArchive())
{
    archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
    archive.SaveLzipped(dataDir + "archive.tar.lz");
}
```

**व्याख्या**

- `new TarArchive()` एक खाली TAR कंटेनर बनाता है।  
- `CreateEntry` आपके `dataDir` से फ़ाइल `alice29.txt` जोड़ता है।  
- `SaveLzipped` आर्काइव को डिस्क पर लिखता है और LZ संपीड़न लागू करता है, जिससे `archive.tar.lz` बनता है।

### चरण 2: एक ही आर्काइव में कई फ़ाइलें संपीड़ित करें
अक्सर आपको कई फ़ाइलों को एक साथ बंडल करने की आवश्यकता होती है। सहेजने से पहले प्रत्येक फ़ाइल के लिए `CreateEntry` कॉल करें। यह **add files to tar lz** और प्रभावी रूप से **compress multiple files tar** को दर्शाता है।

```csharp
//ExStart: CompressMultipleFiles
using (TarArchive archive = new TarArchive())
{
    archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
    archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
    archive.SaveLzipped(dataDir + "archive.tar.lz");
}
```

**व्याख्या**

- कोड Step 1 के समान पैटर्न का अनुसरण करता है, लेकिन एक दूसरा एंट्री (`lcet10.txt`) जोड़ता है।  
- आप आवश्यकता अनुसार `CreateEntry` को कई बार दोहरा सकते हैं; लाइब्रेरी आंतरिक TAR संरचना को स्वचालित रूप से संभालती है।

### चरण 3: अपना दस्तावेज़ डायरेक्टरी निर्दिष्ट करें
प्लेसहोल्डर को वास्तविक पाथ से बदलें जहाँ आपकी स्रोत फ़ाइलें स्थित हैं। यह पाथ ऊपर के उदाहरणों द्वारा उपयोग किया जाता है।

```csharp
string dataDir = "Your Document Directory";
```

**व्याख्या**

- `dataDir` को एक पूर्ण‑योग्य फ़ोल्डर पाथ पर सेट करें, उदाहरण के लिए `@"C:\MyFiles\"`।  
- डायरेक्टरी को वैरिएबल में रखने से कोड पुन: उपयोग योग्य और रखरखाव में आसान बनता है।
