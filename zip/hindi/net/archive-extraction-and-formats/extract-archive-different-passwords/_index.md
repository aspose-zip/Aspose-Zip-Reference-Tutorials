# Aspose.Zip for .NET का उपयोग करके पासवर्ड के साथ ज़िप निकालना कैसे करें

आधुनिक .NET अनुप्रयोगों में, ZIP आर्काइव के भीतर संवेदनशील डेटा की सुरक्षा एक सामान्य आवश्यकता है। यह ट्यूटोरियल **पासवर्ड के साथ ज़िप निकालने का तरीका** दिखाता है जब प्रत्येक एंट्री अलग पासवर्ड का उपयोग करती है, जिससे आपको सुरक्षा पर सूक्ष्म नियंत्रण मिलता है जबकि निष्कर्षण प्रक्रिया सरल रहती है। इस Aspose.Zip उदाहरण का पालन करके आप देखेंगे कि व्यक्तिगत एंट्रीज़ के लिए पासवर्ड‑सुरक्षित ज़िप निष्कर्षण कैसे किया जाता है।

## त्वरित उत्तर
- **मैं कौनसी लाइब्रेरी उपयोग करूँ?** Aspose.Zip for .NET.  
- **क्या मैं विभिन्न पासवर्ड वाले एंट्रीज़ को निकाल सकता हूँ?** हाँ—प्रत्येक एंट्री को उसके अपने पासवर्ड से खोला जा सकता है।  
- **क्या उत्पादन के लिए लाइसेंस चाहिए?** एक व्यावसायिक लाइसेंस आवश्यक है; एक मुफ्त ट्रायल उपलब्ध है।

## “how to extract zip” क्या है?
ZIP आर्काइव को निकालना मतलब संकुचित कंटेनर को पढ़ना और उसकी सामग्री को फ़ाइल सिस्टम पर लिखना है। जब आर्काइव पासवर्ड‑सुरक्षित हो, तो डेटा को डिकम्प्रेस करने से पहले प्रत्येक एंट्री के लिए सही पासवर्ड प्रदान करना आवश्यक होता है।

## पासवर्ड‑सुरक्षित निष्कर्षण के लिए Aspose.Zip क्यों उपयोग करें?
- **सूक्ष्म सुरक्षा:** प्रत्येक फ़ाइल का अपना पासवर्ड हो सकता है, जिससे यदि एक पासवर्ड समझौता हो जाए तो जोखिम कम हो जाता है।  
- **लचीलापन:** आप व्यापार तर्क (जैसे, उपयोगकर्ता भूमिकाएँ) के आधार पर प्रोग्रामेटिक रूप से तय कर सकते हैं कि कौन सा पासवर्ड लागू करना है।  
- **प्रदर्शन:** Aspose.Zip एंट्रीज़ को मेमोरी में प्रोसेस करता है, जिससे पूरे आर्काइव को पहले अनज़िप करने की आवश्यकता नहीं रहती।  
- **क्रॉस‑प्लेटफ़ॉर्म समर्थन:** .NET 5/6+ के साथ Windows, Linux, और macOS पर काम करता है।

## नेमस्पेस आयात करें

पहले, आर्काइव के साथ काम करने के लिए आवश्यक नेमस्पेस आयात करें:

```csharp
using Aspose.Zip;
using System.IO;
```

ये दो `using` स्टेटमेंट आपको `Archive` क्लास और मानक I/O यूटिलिटीज़ तक पहुंच देते हैं।

## चरण 1: कार्य निर्देशिका निर्धारित करें

फ़ोल्डर सेट करें जहाँ ZIP फ़ाइल स्थित है और जहाँ निकाली गई फ़ाइलें लिखी जाएँगी:

```csharp
string dataDir = "Your Document Directory";
```

> **Pro tip:** यदि आपको Linux/macOS का समर्थन करना है तो क्रॉस‑प्लेटफ़ॉर्म पाथ निर्माण के लिए `Path.Combine` का उपयोग करें।

## Aspose.Zip का उपयोग करके पासवर्ड के साथ ज़िप निकालना कैसे करें

नीचे हम ठीक‑ठीक चरणों के माध्यम से दिखाते हैं कि कैसे आर्काइव को खोलें और प्रत्येक एंट्री को उसके अपने पासवर्ड का उपयोग करके निकालें। यह सेक्शन **पासवर्ड के साथ ज़िप निकालना** प्रत्येक एंट्री के लिए दर्शाता है, जो “how to extract zip” प्रक्रिया का मूल है।

### चरण 2: विभिन्न पासवर्ड के साथ आर्काइव एंट्रीज़ निकालें

#### चरण 2.1: ज़िप फ़ाइल खोलें

```csharp
using (FileStream zipFile = File.Open(dataDir + "\\different_password.zip", FileMode.Open))
{
    using (Archive archive = new Archive(zipFile))
    {
        // Continue to the next steps...
    }
}
```

`Archive` ऑब्जेक्ट ZIP कंटेनर का प्रतिनिधित्व करता है। `FileStream` और `Archive` को `using` ब्लॉक्स के भीतर रखना सुनिश्चित करता है कि सभी संसाधन तुरंत मुक्त हो जाएँ।

#### चरण 2.2: प्रथम एंट्री निकालें (पासवर्ड = “first_pass”)

```csharp
archive.Entries[0].Extract(dataDir + "alice29_extracted_pass_out.txt", "first_pass");
```

यहाँ हम `Entries` कलेक्शन के माध्यम से **कई ज़िप एंट्रीज़ निकालते** हैं। पहली एंट्री को पासवर्ड `"first_pass"` से डिक्रिप्ट किया जाता है।

#### चरण 2.3: द्वितीय एंट्री निकालें (पासवर्ड = “second_pass”)

```csharp
archive.Entries[1].Extract(dataDir + "asyoulik_extracted_pass_out.txt", "second_pass");
```

दूसरी एंट्री अलग पासवर्ड का उपयोग करती है, जिससे प्रत्येक व्यक्तिगत फ़ाइल के लिए **ज़िप एंट्री पासवर्ड निकालने** का प्रदर्शन होता है।

#### चरण 2.4: (वैकल्पिक) सभी एंट्रीज़ पर लूप करें

यदि आपको **कई ज़िप एंट्रीज़ निकालनी** हैं बिना इंडेक्स हार्ड‑कोड किए, तो आप `archive.Entries` पर इटररेट कर सकते हैं और अपनी स्वयं की लुक‑अप लॉजिक के आधार पर प्रत्येक एंट्री के लिए उपयुक्त पासवर्ड प्रदान कर सकते हैं। यह पैटर्न बड़े आर्काइव्स को संभालते समय अच्छी तरह स्केल करता है।
