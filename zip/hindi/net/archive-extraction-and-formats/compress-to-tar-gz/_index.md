# Aspose.Zip for .NET के साथ फ़ाइलों को tar में जोड़ें और TarGz बनाएं

## परिचय

आधुनिक .NET अनुप्रयोगों में, **add files to tar** को तेज़ और भरोसेमंद तरीके से करना एक सामान्य आवश्यकता है—चाहे आप लॉग्स को पैकेज कर रहे हों, क्लाउड स्टोरेज के लिए डेटा तैयार कर रहे हों, या डिप्लॉयमेंट बंडल बना रहे हों। Aspose.Zip for .NET आपको एक साफ़, हाई‑परफ़ॉर्मेंस API देता है जिससे आप **add files to tar** कर सकते हैं, फिर इस आर्काइव को व्यापक रूप से उपयोग किए जाने वाले **tar.gz** फ़ॉर्मेट में कंप्रेस कर सकते हैं। इस ट्यूटोरियल में आप सीखेंगे कि कैसे **create tar.gz archive** को शुरू से, चरण‑दर‑चरण, Aspose.Zip .NET लाइब्रेरी का उपयोग करके बनाया जाता है।

## त्वरित उत्तर
- **मैं कौन सी लाइब्रेरी उपयोग करूँ?** Aspose.Zip for .NET  
- **मैं फ़ाइलों को tar में कैसे जोड़ूँ?** प्रत्येक फ़ाइल के लिए `TarArchive.CreateEntry` का उपयोग करें।  
- **क्या मैं सीधे tar.gz में कंप्रेस कर सकता हूँ?** हाँ—`SaveGzipped` को कॉल करें।  
- **उत्पादन के लिए लाइसेंस चाहिए?** गैर‑ट्रायल उपयोग के लिए एक वैध Aspose लाइसेंस आवश्यक है।  
- **समर्थित .NET संस्करण?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7।

## “add files to tar” क्या है?
फ़ाइलों को tar आर्काइव में जोड़ना मतलब कई फ़ाइलों को एक एकल, अनकम्प्रेस्ड कंटेनर में बंडल करना। tar फ़ॉर्मेट डायरेक्टरी संरचनाओं और फ़ाइल मेटाडेटा को संरक्षित करता है, जिससे यह वैकल्पिक कंप्रेशन (जैसे gzip) से पहले आर्काइविंग के लिए आदर्श बनता है, जिससे **create tar.gz archive** संभव हो पाता है।

## Aspose.Zip का उपयोग करके फ़ाइलों को tar.gz में कंप्रेस क्यों करें?
- **कोई बाहरी टूल नहीं** – सब कुछ आपके .NET कोड के भीतर चलता है।  
- **उच्च प्रदर्शन** – स्ट्रीम‑आधारित API बड़े फ़ाइलों को कुशलता से संभालता है।  
- **क्रॉस‑प्लेटफ़ॉर्म** – Windows, Linux, और macOS पर काम करता है।  
- **समृद्ध फीचर सेट** – एन्क्रिप्शन, पासवर्ड प्रोटेक्शन, और कस्टम एंट्री एट्रिब्यूट्स को सपोर्ट करता है।

## नेमस्पेस इम्पोर्ट करें

अपने .NET प्रोजेक्ट में उन नेमस्पेस को इम्पोर्ट करें जो tar‑संबंधित क्लासेज़ को एक्सपोज़ करते हैं:

```csharp
using System;
using Aspose.Zip.Tar;
```

## Aspose.Zip for .NET के साथ tar.gz आर्काइव कैसे बनाएं

### चरण 1: अपना डॉक्यूमेंट डायरेक्टरी सेट करें

पहले, कोड को उस फ़ोल्डर की ओर इंगित करें जिसमें वे फ़ाइलें हैं जिन्हें आप आर्काइव करना चाहते हैं।

```csharp
string dataDir = "Your Document Directory";
```

> **प्रो टिप:** पाथ बनाते समय `Path.Combine` का उपयोग करें ताकि प्लेटफ़ॉर्म‑स्पेसिफिक सेपरेटर समस्याओं से बचा जा सके।

### चरण 2: TarArchive को इनिशियलाइज़ करें

एक `TarArchive` इंस्टेंस बनाएं जो एंट्रीज़ को रखेगा।

```csharp
using (TarArchive archive = new TarArchive())
{
    // Entries will be added inside this block.
}
```

### चरण 3: फ़ाइलें जोड़ें – “add files to tar” का मूल भाग

`using` ब्लॉक के अंदर, प्रत्येक फ़ाइल को आर्काइव में शामिल करने के लिए जोड़ें।

```csharp
archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
```

प्रत्येक `CreateEntry` कॉल **एंट्री नाम** (फ़ाइल tar में कैसे दिखाई देगी) और डिस्क पर **सोर्स फ़ाइल पाथ** लेती है।

### चरण 4: Gzipped Tar के रूप में सेव करें (फ़ाइलों को tar.gz में कंप्रेस कैसे करें)

अंत में, tar कंटेंट को gzip स्ट्रीम में लिखें ताकि एक कॉम्पैक्ट `archive.tar.gz` बन सके।

```csharp
archive.SaveGzipped(dataDir + "archive.tar.gz");
```

`SaveGzipped` एक ही फ्लुएंट कॉल में tar पैकेजिंग और gzip कंप्रेशन दोनों को संभालता है।

## सामान्य उपयोग केस

| परिदृश्य | “add files to tar” क्यों मददगार है |
|----------|-----------------------------------|
| **लॉग एग्रीगेशन** | क्लाउड स्टोरेज पर अपलोड करने से पहले दैनिक लॉग्स को एकल आर्काइव में बंडल करें। |
| **डिप्लॉयमेंट पैकेज** | Windows बिल्ड पाइपलाइन से Linux सर्वरों के लिए पोर्टेबल tar.gz बंडल बनाएं। |
| **डेटा बैकअप** | फ़ोल्डर हायरार्की और मेटाडेटा को संरक्षित रखें जबकि बैकअप आकार को कम रखें। |

## सामान्य समस्याएँ और समाधान

- **फ़ाइल नहीं मिली त्रुटि** – सुनिश्चित करें कि `dataDir` उचित पाथ सेपरेटर के साथ समाप्त हो या `Path.Combine` का उपयोग करें।  
- **बड़ी फ़ाइलें मेमोरी प्रेशर बनाती हैं** – पूरी फ़ाइल को मेमोरी में लोड करने से बचने के लिए स्ट्रीम‑आधारित ओवरलोड (`CreateEntry` के साथ `Stream`) का उपयोग करें।  
- **Gzip आउटपुट करप्ट है** – `SaveGzipped` कॉल करने से पहले आर्काइव को बंद (`using` ब्लॉक) कर लें।
