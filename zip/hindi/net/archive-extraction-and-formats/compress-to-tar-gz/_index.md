# Aspose.Zip for .NET के साथ tar आर्काइव बनाएं और फ़ाइलें जोड़ें

## परिचय

आधुनिक .NET अनुप्रयोगों में **tar आर्काइव बनाना** और **फ़ाइलें tar में जोड़ना** तेज़ और विश्वसनीय रूप से करना एक सामान्य आवश्यकता है—चाहे आप लॉग्स को पैकेज कर रहे हों, क्लाउड स्टोरेज के लिए डेटा तैयार कर रहे हों, या डिप्लॉयमेंट बंडल बना रहे हों। Aspose.Zip for .NET आपको एक साफ़, उच्च‑प्रदर्शन API प्रदान करता है जिससे आप **फ़ाइलें tar में जोड़ सकते हैं**, फिर व्यापक रूप से उपयोग किए जाने वाले **tar.gz** फ़ॉर्मेट में आर्काइव को संकुचित कर सकते हैं। इस गाइड में हम पूरी प्रक्रिया को कवर करेंगे, प्रोजेक्ट सेटअप से लेकर तैयार‑to‑ship `archive.tar.gz` बनाने तक।

## त्वरित उत्तर
- **कौन सी लाइब्रेरी उपयोग करनी चाहिए?** Aspose.Zip for .NET  
- **फ़ाइलें tar में कैसे जोड़ें?** प्रत्येक फ़ाइल के लिए `TarArchive.CreateEntry` का उपयोग करें।  
- **क्या सीधे tar.gz में संकुचित कर सकते हैं?** हाँ—`SaveGzipped` को कॉल करें।  
- **उत्पादन के लिए लाइसेंस चाहिए?** गैर‑ट्रायल उपयोग के लिए वैध Aspose लाइसेंस आवश्यक है।  
- **समर्थित .NET संस्करण?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7।

## “फ़ाइलें tar में जोड़ना” क्या है?
फ़ाइलों को tar आर्काइव में जोड़ना का अर्थ है कई फ़ाइलों को एकल, अनकम्प्रेस्ड कंटेनर में बंडल करना। tar फ़ॉर्मेट डायरेक्टरी संरचनाओं और फ़ाइल मेटाडेटा को संरक्षित करता है, जिससे यह वैकल्पिक संकुचन (जैसे gzip) से पहले आर्काइविंग के लिए आदर्श बनता है **tar.gz आर्काइव बनाने** के लिए।

## Aspose.Zip से फ़ाइलों को tar.gz में संकुचित करने के लाभ
- **कोई बाहरी टूल नहीं** – सब कुछ आपके .NET कोड के भीतर चलता है।  
- **उच्च प्रदर्शन** – स्ट्रीम‑आधारित API बड़े फ़ाइलों को कुशलता से संभालता है।  
- **क्रॉस‑प्लेटफ़ॉर्म tar** – Windows, Linux, और macOS पर बिना बदलाव के काम करता है।  
- **समृद्ध फीचर सेट** – एन्क्रिप्शन, पासवर्ड प्रोटेक्शन, और कस्टम एंट्री एट्रिब्यूट्स को सपोर्ट करता है।

## नेमस्पेस आयात करें

अपने .NET प्रोजेक्ट में उन नेमस्पेस को आयात करें जो tar‑संबंधित क्लासेज़ को उजागर करते हैं:

```csharp
using System;
using Aspose.Zip.Tar;
```

## Aspose.Zip for .NET का उपयोग करके फ़ाइलें tar में कैसे जोड़ें

### चरण 1: अपना डॉक्यूमेंट डायरेक्टरी सेट करें

पहले, कोड को उस फ़ोल्डर की ओर इंगित करें जिसमें वे फ़ाइलें हैं जिन्हें आप आर्काइव करना चाहते हैं।

```csharp
string dataDir = "Your Document Directory";
```

> **प्रो टिप:** पाथ बनाते समय `Path.Combine` का उपयोग करें ताकि प्लेटफ़ॉर्म‑विशिष्ट सेपरेटर समस्याओं से बचा जा सके।

### चरण 2: एक TarGz आर्काइव बनाएं

अब हम tar आर्काइव बनाएँगे, एंट्रीज़ जोड़ेंगे, और एक ही फ़्लुएंट फ्लो में इसे संकुचित करेंगे।

#### 2.1 TarArchive को इनिशियलाइज़ करें

```csharp
using (TarArchive archive = new TarArchive())
{
    // Entries will be added inside this block.
}
```

#### 2.2 फ़ाइलें जोड़ें – “फ़ाइलें tar में जोड़ना” का मुख्य भाग

```csharp
archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
```

प्रत्येक `CreateEntry` कॉल **एंट्री नाम** (फ़ाइल tar के अंदर कैसे दिखाई देगी) और डिस्क पर **सोर्स फ़ाइल पाथ** लेती है। आप कई फ़ाइलें एक ही आर्काइव में **फ़ाइलें tar में जोड़ने** के लिए `CreateEntry` को बार‑बार कॉल कर सकते हैं।

#### 2.3 Gzipped Tar के रूप में सेव करें (tar.gz कैसे संकुचित करें)

```csharp
archive.SaveGzipped(dataDir + "archive.tar.gz");
```

`SaveGzipped` tar सामग्री को gzip स्ट्रीम में लिखता है, जिससे आपको एक कॉम्पैक्ट `archive.tar.gz` फ़ाइल मिलती है जो वितरण के लिए तैयार है।

## सामान्य उपयोग केस

| परिदृश्य | “फ़ाइलें tar में जोड़ना” क्यों मददगार है |
|----------|----------------------------------------|
| **लॉग एग्रीगेशन** | क्लाउड स्टोरेज पर अपलोड करने से पहले दैनिक लॉग्स को एकल आर्काइव में बंडल करें। |
| **डिप्लॉयमेंट पैकेज** | Windows बिल्ड पाइपलाइन से Linux सर्वरों के लिए पोर्टेबल tar.gz बंडल बनाएं। |
| **डेटा बैकअप** | फ़ोल्डर हाइरार्की और मेटाडेटा को संरक्षित रखें जबकि बैकअप आकार कम रखें। |

## सामान्य समस्याएँ और समाधान

- **फ़ाइल नहीं मिली त्रुटि** – सुनिश्चित करें कि `dataDir` उचित पाथ सेपरेटर के साथ समाप्त हो या `Path.Combine` का उपयोग करें।  
- **बड़ी फ़ाइलें मेमोरी प्रेशर बनाती हैं** – स्ट्रीम‑आधारित ओवरलोड (`CreateEntry` के साथ `Stream`) का उपयोग करें ताकि पूरी फ़ाइल मेमोरी में लोड न हो।  
- **Gzip आउटपुट करप्ट है** – `SaveGzipped` कॉल करने से पहले आर्काइव को बंद (`using` ब्लॉक) करना सुनिश्चित करें।
