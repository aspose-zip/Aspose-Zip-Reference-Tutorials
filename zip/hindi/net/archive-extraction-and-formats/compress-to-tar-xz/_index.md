# कैसे फ़ाइलों को tar में जोड़ें, .NET के साथ Aspose.Zip से tarxz आर्काइव बनाएं

## परिचय

यदि आपको **फ़ाइलों को tar में जोड़ना** है और फिर उन्हें .NET का उपयोग करके tarxz आर्काइव में संकुचित करना है, तो Aspose.Zip for .NET प्रक्रिया को सरल और विश्वसनीय बनाता है। चाहे आप लॉग, कॉन्फ़िगरेशन फ़ाइलें, या किसी भी अन्य एसेट को संग्रहण या ट्रांसमिशन के लिए पैकेज कर रहे हों, TarXz फ़ॉर्मेट में संकुचन उच्च संपीड़न अनुपात प्रदान करता है जबकि परिचित tar संरचना को बनाए रखता है। इस ट्यूटोरियल में हम सटीक चरणों—कोड स्निपेट्स सहित—के माध्यम से चलेंगे, ताकि आप भरोसे के साथ अपने .NET एप्लिकेशन में tarxz निर्माण को एकीकृत कर सकें।

## त्वरित उत्तर
- **मुख्य क्लास कौन सी है?** `TarArchive` from `Aspose.Zip.Tar`
- **tarxz कैसे संकुचित करें?** एंट्रीज़ जोड़ने के बाद `SaveXzCompressed` का उपयोग करें
- **समर्थित .NET संस्करण?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6
- **लाइसेंस आवश्यक है?** हाँ, उत्पादन के लिए वैध Aspose.Zip लाइसेंस
- **सामान्य कार्यान्वयन समय?** ~5‑10 मिनट

## TarXz आर्काइव क्या है?

एक **TarXz आर्काइव** पारंपरिक Unix `tar` कंटेनर को XZ संपीड़न के साथ जोड़ता है। tar भाग कई फ़ाइलों को एक ही स्ट्रीम में बंडल करता है, जबकि XZ मजबूत, लॉसलेस संपीड़न प्रदान करता है। यह फ़ॉर्मेट स्रोत कोड, बैकअप और बड़े डेटा सेट वितरित करने के लिए लोकप्रिय है क्योंकि यह डायरेक्टरी संरचनाओं को बनाए रखता है और साधारण tar या zip की तुलना में छोटे फ़ाइल आकार प्राप्त करता है।

## Aspose.Zip के साथ फ़ाइलों को tar में जोड़ना और tarxz में संकुचित करना क्यों?

- **उच्च संपीड़न अनुपात** – XZ अक्सर gzip से 30‑50 % छोटा संकुचित करता है।  
- **क्रॉस‑प्लेटफ़ॉर्म संगतता** – TarXz फ़ाइलें Linux, macOS और Windows पर खोली जा सकती हैं।  
- **सरल API** – Aspose.Zip लो‑लेवल विवरणों को एब्स्ट्रैक्ट करता है, जिससे आप अपने बिज़नेस लॉजिक पर ध्यान केंद्रित कर सकते हैं।  
- **कोई बाहरी टूल नहीं** – सब कुछ आपके .NET प्रोसेस के भीतर चलता है, क्लाउड या CI पाइपलाइन के लिए आदर्श।  

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास है:

- **Aspose.Zip for .NET** स्थापित (आधिकारिक [Aspose.Zip documentation](https://reference.aspose.com/zip/net/) से डाउनलोड करें)।  
- उन फ़ाइलों वाला फ़ोल्डर जिसे आप आर्काइव करना चाहते हैं। नीचे के उदाहरणों में इस फ़ोल्डर को `dataDir` वेरिएबल द्वारा संदर्भित किया गया है।  
- वैध Aspose.Zip लाइसेंस (मूल्यांकन के लिए वैकल्पिक, उत्पादन के लिए आवश्यक)।

## नेमस्पेस आयात करें

पहले, उन नेमस्पेस को आयात करें जो TarXz कार्यक्षमता को उजागर करते हैं।

```csharp
using System;
using Aspose.Zip.Tar;
```

## Aspose.Zip का उपयोग करके फ़ाइलों को tar में जोड़ना

### चरण 1: `TarArchive` को इनिशियलाइज़ करें

एक नया `TarArchive` इंस्टेंस बनाएं जो उन फ़ाइलों को रखेगा जिन्हें आप संकुचित करना चाहते हैं।

```csharp
using (TarArchive archive = new TarArchive())
{
```

> **प्रो टिप:** `using` स्टेटमेंट सुनिश्चित करता है कि आर्काइव सही ढंग से डिस्पोज़ हो, जिससे किसी भी अनमैनेज्ड रिसोर्स को मुक्त किया जा सके।

### चरण 2: फ़ाइलों को आर्काइव में जोड़ें

हर फ़ाइल को जोड़ें जिसे आप शामिल करना चाहते हैं। इस उदाहरण में हम दो टेक्स्ट फ़ाइलें जोड़ते हैं, लेकिन आप जितनी भी एंट्रीज़ चाहें जोड़ सकते हैं।

```csharp
    archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
    archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
```

> **यह क्यों महत्वपूर्ण है:** संकुचन से पहले एंट्रीज़ जोड़ने से Aspose.Zip पहले tar कंटेनर बनाता है, फिर एक ही चरण में XZ संकुचन लागू करता है।

### चरण 3: XZ संकुचन के साथ आर्काइव सहेजें

अंत में, XZ संकुचन का उपयोग करके tar आर्काइव को डिस्क पर लिखें। परिणामी फ़ाइल का एक्सटेंशन `.tar.xz` होगा।

```csharp
    archive.SaveXzCompressed(dataDir + "archive.tar.xz");
}
```

> **परिणाम:** अब आपके पास एक पूरी तरह से संकुचित `archive.tar.xz` फ़ाइल है जिसे किसी भी प्लेटफ़ॉर्म पर ट्रांसफ़र, स्टोर या अनपैक किया जा सकता है जो TarXz का समर्थन करता है।

## सामान्य समस्याएँ और समाधान

| समस्या | कारण | समाधान |
|-------|-------|-----|
| **“File not found” अपवाद** | गलत `dataDir` पाथ | सुनिश्चित करें कि डायरेक्टरी पाथ बैकस्लैश (`\`) पर समाप्त हो या `Path.Combine` का उपयोग करें। |
| **बड़ी मेमोरी उपयोग** | बहुत बड़ी फ़ाइलें मेमोरी में संकुचित हो रही हैं | `TarArchive` को स्ट्रीमिंग मोड में उपयोग करें (`SaveXzCompressed` ओवरलोड जो `Stream` स्वीकार करता है)। |
| **लाइसेंस लागू नहीं हुआ** | लाइसेंस फ़ाइल अनुपलब्ध | एप्लिकेशन स्टार्ट पर लाइसेंस लोड करें: `new Aspose.Zip.License().SetLicense("Aspose.Zip.lic");` |
