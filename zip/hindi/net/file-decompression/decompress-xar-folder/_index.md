# Aspose.Zip for .NET का उपयोग करके Xar आर्काइव को फ़ोल्डर में निकालने का तरीका

## परिचय

यदि आप एक .NET डेवलपर हैं जिन्हें **xar आर्काइव** फ़ाइलों को तेज़ी और विश्वसनीयता के साथ निकालना है, तो Aspose.Zip for .NET एक साफ़, उच्च‑प्रदर्शन API प्रदान करता है जो पूरी प्रक्रिया को बाहरी टूल्स के बिना संभालता है। इस ट्यूटोरियल में हम Xar आर्काइव को फ़ोल्डर में डीकंप्रेस करने के लिए आवश्यक प्रत्येक चरण को समझेंगे, यह बताएँगे कि यह विधि आपका समय कैसे बचाती है, और तैयार‑चलाने‑योग्य कोड देंगे। अंत तक, आप समझेंगे कि कब इस दृष्टिकोण का उपयोग करना है, इसे अपने प्रोजेक्ट में कैसे एकीकृत करना है, और सामान्य समस्याओं से कैसे बचा जाए।

## त्वरित उत्तर
- **लाइब्रेरी क्या करती है?** यह Xar आर्काइव को पढ़ती और निकालती है बिना बाहरी टूल्स के।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.6+, .NET Core 3.1+, .NET 5/6+।  
- **क्या लाइसेंस चाहिए?** विकास के लिए एक मुफ्त ट्रायल काम करता है; उत्पादन के लिए एक व्यावसायिक लाइसेंस आवश्यक है।  
- **इम्प्लीमेंटेशन में कितना समय लगेगा?** आमतौर पर 10 मिनट से कम।  
- **क्या मैं कस्टम फ़ोल्डर में निकाल सकता हूँ?** हाँ—सिर्फ `ExtractToDirectory` में लक्ष्य पथ निर्दिष्ट करें।

## “how to extract xar” क्या है?

Xar आर्काइव को निकालना मतलब संकुचित पैकेज को पढ़ना और उसकी आंतरिक फ़ाइलों को डिस्क पर किसी डायरेक्टरी में लिखना है। यह तब उपयोगी होता है जब आप macOS इंस्टालर्स, बैकअप यूटिलिटीज़, या थर्ड‑पार्टी टूल्स से XAR पैकेज प्राप्त करते हैं और उन्हें .NET एप्लिकेशन में प्रोसेस करना चाहते हैं।

## इस कार्य के लिए Aspose.Zip क्यों उपयोग करें?

- **कोई बाहरी निर्भरताएँ नहीं** – शुद्ध .NET, कोई नेटिव बाइनरी नहीं।  
- **स्ट्रीम‑आधारित API** – फ़ाइलों, मेमोरी स्ट्रीम्स, या नेटवर्क स्ट्रीम्स के साथ काम करता है।  
- **मज़बूत त्रुटि संभाल** – विस्तृत एक्सेप्शन आपको भ्रष्ट आर्काइव को ट्रबलशूट करने में मदद करते हैं।  
- **पूर्ण .NET संगतता** – Windows, Linux, और macOS रनटाइम्स पर काम करता है।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

- **Aspose.Zip for .NET** – आपके प्रोजेक्ट में इंटीग्रेटेड। आप इसे [यहाँ](https://releases.aspose.com/zip/net/) से डाउनलोड कर सकते हैं।  
- **डॉक्यूमेंट डायरेक्टरी** – आपके सॉल्यूशन में एक फ़ोल्डर जहाँ नमूना `.xar` फ़ाइल और निकाली गई आउटपुट रखी जाएगी।

## नेमस्पेस इम्पोर्ट करें

अपने .NET प्रोजेक्ट में Aspose.Zip कार्यक्षमता तक पहुँचने के लिए आवश्यक नेमस्पेस शामिल करें:

```csharp
using System.IO;
using Aspose.Zip.Xar;
```

## चरण 1: अपना डॉक्यूमेंट डायरेक्टरी परिभाषित करें

```csharp
string dataDir = "Your Document Directory";
```

`"Your Document Directory"` को उस पूर्ण या सापेक्ष पथ से बदलें जिसमें `sample.xar` मौजूद है और जहाँ आप आउटपुट फ़ोल्डर बनाना चाहते हैं। बाद में `Path.Combine` का उपयोग करने से विभिन्न ऑपरेटिंग सिस्टम्स में पाथ‑सेपरेटर समस्याओं से बचा जा सकता है।

## चरण 2: Xar आर्काइव को डीकंप्रेस करें

```csharp
//ExStart: DecompressXarArchive
using (FileStream fs = File.OpenRead(dataDir + "sample.xar"))
{
    using (XarArchive archive = new XarArchive(fs))
    {
        archive.ExtractToDirectory(dataDir + "DecompressXar_out");
    }
}
```

यह स्निपेट Xar फ़ाइल को खोलता है, एक `XarArchive` इंस्टेंस बनाता है, और **पूरे डीकंप्रेस xar आर्काइव** को `DecompressXar_out` में निकालता है। ऑपरेशन पूरी तरह से स्ट्रीम‑आधारित है, इसलिए बड़े पैकेजों के साथ भी यह कुशलता से काम करता है।

## चरण 3: कोड चलाएँ

अपना एप्लिकेशन बिल्ड और रन करें। निष्पादन के बाद, आपको अपने डॉक्यूमेंट डायरेक्टरी के अंदर `DecompressXar_out` नाम का नया फ़ोल्डर मिलेगा, जिसमें मूल `.xar` आर्काइव में पैकेज की गई सभी फ़ाइलें होंगी।

## सामान्य समस्याएँ एवं सुझाव

- **फ़ाइल नहीं मिली** – सुनिश्चित करें कि `File.OpenRead` में दिया गया पथ सही ढंग से `sample.xar` की ओर इशारा कर रहा है। सुरक्षित पाथ हैंडलिंग के लिए `Path.Combine` का उपयोग करें।  
- **एक्सेस डिनाइड** – फ़ाइल‑सिस्टम अनुमतियों को पर्याप्त रखें, विशेषकर जब संरक्षित डायरेक्टरी में लिख रहे हों।  
- **भ्रष्ट आर्काइव** – Aspose.Zip `InvalidDataException` थ्रो करता है; स्रोत `.xar` फ़ाइल की अखंडता जांचें।
