# Aspose.Zip .NET के साथ पासवर्ड‑सुरक्षित Zip आर्काइव

## परिचय

इस चरण‑दर‑चरण ट्यूटोरियल में आपका स्वागत है जहाँ हम **कैसे पासवर्ड‑प्रोटेक्टेड zip आर्काइव** फ़ाइलों को पारंपरिक एन्क्रिप्शन के साथ Aspose.Zip for .NET का उपयोग करके बनाते हैं। Aspose.Zip एक शक्तिशाली लाइब्रेरी है जो डेवलपर्स को zip आर्काइव को आसानी से बनाना, पढ़ना और संशोधित करना संभव बनाती है। इस गाइड में हम कई फ़ाइलों को संकुचित करने, उन्हें zip में जोड़ने, और पासवर्ड के साथ आर्काइव को सुरक्षित करने की प्रक्रिया को स्पष्ट और रखरखाव‑योग्य कोड के साथ दिखाएंगे।

## त्वरित उत्तर
- **“पासवर्ड‑प्रोटेक्टेड zip आर्काइव” का क्या अर्थ है?** इसका मतलब है zip फ़ाइल को एन्क्रिप्ट करना ताकि उसकी सामग्री केवल सही पासवर्ड प्रदान करने पर ही खोली जा सके।  
- **.NET में यह कार्य कौन सी लाइब्रेरी संभालती है?** Aspose.Zip for .NET पारंपरिक एन्क्रिप्शन के लिए बिल्ट‑इन सपोर्ट प्रदान करती है।  
- **क्या उत्पादन के लिए लाइसेंस चाहिए?** हाँ, उत्पादन उपयोग के लिए एक व्यावसायिक लाइसेंस आवश्यक है; एक मुफ्त ट्रायल उपलब्ध है।  
- **क्या इसे Linux पर चलाया जा सकता है?** बिल्कुल—Aspose.Zip क्रॉस‑प्लेटफ़ॉर्म है और Windows, Linux, तथा macOS पर काम करता है।  
- **मैं कितनी फ़ाइलें जोड़ सकता हूँ?** आप कोई भी संख्या में फ़ाइलें जोड़ सकते हैं; उदाहरण में तीन फ़ाइलें दिखायी गई हैं, लेकिन यह तरीका स्केलेबल है।

## “पासवर्ड‑प्रोटेक्टेड zip आर्काइव” क्या है?

एक पासवर्ड‑प्रोटेक्टेड zip आर्काइव एन्क्रिप्शन (इस मामले में पारंपरिक PKZIP एन्क्रिप्शन) का उपयोग करके आर्काइव के अंदर फ़ाइल डेटा को स्क्रैम्बल करता है। जब उपयोगकर्ता आर्काइव को एक्सट्रैक्ट करने की कोशिश करता है, तो सामग्री को डिक्रिप्ट करने के लिए सही पासवर्ड प्रदान करना आवश्यक होता है।

## इस कार्य के लिए Aspose.Zip क्यों उपयोग करें?

- **सरल API** – एन्क्रिप्शन को सक्षम करने के लिए केवल एक लाइन कोड।  
- **कोई बाहरी निर्भरताएँ नहीं** – शुद्ध .NET, कोई नेटिव DLL नहीं।  
- **क्रॉस‑प्लेटफ़ॉर्म** – .NET Framework, .NET Core, और .NET 5/6+ के साथ काम करता है।  
- **प्रदर्शन‑उन्नत** – बड़े फ़ाइलों को कुशलता से संभालता है।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित हैं:

- **Aspose.Zip for .NET** – इसे आधिकारिक साइट से [यहाँ](https://releases.aspose.com/zip/net/) डाउनलोड करें।  
- **फ़ाइलों वाला फ़ोल्डर** – नमूना कोड में `"Your Document Directory"` को उस वास्तविक पथ से बदलें जहाँ आप ज़िप करना चाहते हैं।

## नेमस्पेस आयात करें

Aspose.Zip क्लासेज़ के साथ काम करने के लिए आवश्यक नेमस्पेस आयात करें:

```csharp
using Aspose.Zip;
using Aspose.Zip.Saving;
using System.IO;
```

## पारंपरिक एन्क्रिप्शन के साथ zip एन्क्रिप्ट करना

### चरण 1: Zip फ़ाइल सेट अप करें (पासवर्ड‑प्रोटेक्टेड Zip आर्काइव)

Zip फ़ाइल बनाएं और पारंपरिक एन्क्रिप्शन सेटिंग्स कॉन्फ़िगर करें। पासवर्ड `"p@s$"` केवल एक उदाहरण है—वास्तविक प्रोजेक्ट्स के लिए इसे एक मजबूत पासवर्ड से बदलें।

```csharp
//ExStart: CompressMultipleFilesWithTraditionalEncryption
using (FileStream zipFile = File.Open(".\\CompressMultipleFilesWithTraditionalEncryption_out.zip", FileMode.Create))
{
    // Create archive with traditional encryption settings
    using (var archive = new Archive(new ArchiveEntrySettings(null, new TraditionalEncryptionSettings("p@s$"))))
    {
        // Continue to the next step...
    }
}
//ExEnd: CompressMultipleFilesWithTraditionalEncryption
```

### Zip आर्काइव में फ़ाइलें जोड़ें

अब प्रत्येक फ़ाइल को जोड़ें जिसे आप शामिल करना चाहते हैं। `CreateEntry` मेथड zip के अंदर फ़ाइल नाम और एक स्ट्रीम (`source1`, `source2`, `source3`) लेता है जो वास्तविक फ़ाइल डेटा की ओर इशारा करता है।

```csharp
archive.CreateEntry("alice29.txt", source1);
archive.CreateEntry("asyoulik.txt", source2);
archive.CreateEntry("fields.c", source3);
```

### Zip फ़ाइल सहेजें (पासवर्ड के साथ फ़ाइलें संकुचित करें)

अंत में, आर्काइव को डिस्क पर स्थायी रूप से लिखें। यह चरण एन्क्रिप्टेड zip फ़ाइल को पहले निर्दिष्ट स्थान पर लिखता है।

```csharp
archive.Save(zipFile);
```

> **प्रो टिप:** हमेशा स्ट्रीम्स को `using` स्टेटमेंट्स के साथ बंद करें ताकि फ़ाइल हैंडल तुरंत रिलीज़ हो जाएँ, विशेषकर बड़े फ़ाइलों के साथ काम करते समय।

## सामान्य समस्याएँ और समाधान

| समस्या | कारण | समाधान |
|-------|--------|-----|
| **गलत पासवर्ड त्रुटि** | `TraditionalEncryptionSettings` में पासवर्ड मेल नहीं खाता या टाइपो है | पासवर्ड स्ट्रिंग की जाँच करें और एक्सट्रैक्ट करते समय वही पासवर्ड उपयोग करें। |
| **फ़ाइल नहीं जोड़ी गई** | स्रोत स्ट्रीम (`sourceX`) null है या डिस्पोज़ हो गया है | `File.OpenRead` के साथ स्रोत स्ट्रीम खोलें और फिर `CreateEntry` को पास करें। |
| **बड़ी फ़ाइलों पर प्रदर्शन धीमा** | कई बड़ी एंट्रीज़ के साथ डिफ़ॉल्ट कॉम्प्रेशन लेवल उपयोग किया गया | तेज़ प्रोसेसिंग के लिए `ArchiveEntrySettings` में `CompressionLevel` सेट करने पर विचार करें। |
