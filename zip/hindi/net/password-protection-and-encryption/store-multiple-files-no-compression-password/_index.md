# Aspose.Zip for .NET - पासवर्ड प्रोटेक्ट ज़िप आर्काइव ट्यूटोरियल

आधुनिक .NET विकास में, फ़ाइलों को सुरक्षित रूप से आर्काइव करना एक सामान्य आवश्यकता है। **Aspose.Zip for .NET** के साथ, आप **पासवर्ड प्रोटेक्ट ज़िप आर्काइव** फ़ाइलें बना सकते हैं, कई आइटम बिना किसी संपीड़न के स्टोर कर सकते हैं, और मजबूत AES एन्क्रिप्शन लागू कर सकते हैं—सिर्फ कुछ ही लाइनों के C# कोड में। यह ट्यूटोरियल आपको ठीक‑ठीक चरण‑दर‑चरण दिखाता है कि कैसे एक ज़िप बनाएं जिसमें कई फ़ाइलें हों, *store* (कोई‑संपीड़न) मोड उपयोग हो, और पासवर्ड से लॉक हो।

## त्वरित उत्तर
- **“पासवर्ड प्रोटेक्ट ज़िप आर्काइव” का क्या मतलब है?** यह ज़िप की सामग्री को एन्क्रिप्ट करता है ताकि केवल सही पासवर्ड से ही इसे खोला जा सके।  
- **कौन सा एन्क्रिप्शन एल्गोरिद्म उपयोग किया जाता है?** `AesEcryptionSettings` के माध्यम से AES‑256।  
- **क्या मैं एक से अधिक फ़ाइलें जोड़ सकता हूँ?** हाँ – प्रत्येक स्रोत फ़ाइल के लिए `CreateEntry` कॉल को दोहराएँ।  
- **उत्पादन के लिए लाइसेंस चाहिए?** एक वाणिज्यिक लाइसेंस आवश्यक है; एक मुफ्त ट्रायल उपलब्ध है।  
- **क्या यह .NET 6/7 पर समर्थित है?** बिल्कुल – Aspose.Zip .NET Framework, .NET Core, और .NET 5/6/7 के साथ काम करता है।

## पासवर्ड प्रोटेक्ट ज़िप आर्काइव क्या है?
एक *पासवर्ड प्रोटेक्ट ज़िप आर्काइव* वह ZIP फ़ाइल है जिसकी एंट्रीज़ उपयोगकर्ता‑परिभाषित पासवर्ड से एन्क्रिप्ट की गई होती हैं। जब आर्काइव खोला जाता है, तो पासवर्ड प्रदान करना आवश्यक होता है; अन्यथा सामग्री पढ़ी नहीं जा सकती। Aspose.Zip इसे AES‑256 एन्क्रिप्शन के माध्यम से लागू करता है, जिससे संवेदनशील डेटा के लिए मजबूत सुरक्षा मिलती है।

## Aspose.Zip के साथ ज़िप फ़ाइल पासवर्ड प्रोटेक्शन क्यों उपयोग करें?
- **कोई‑संपीड़न स्टोरेज** – `StoreCompressionSettings` मूल फ़ाइल आकार को बरकरार रखता है, पहले से संपीड़ित मीडिया के लिए आदर्श।  
- **मजबूत एन्क्रिप्शन** – AES‑256 डेटा को ब्रूट‑फ़ोर्स हमलों से बचाता है।  
- **पूर्ण .NET एकीकरण** – .NET Framework, .NET Core, और .NET 5/6/7 के साथ बिना किसी नेटिव डिपेंडेंसी के काम करता है।  
- **सरल API** – एक आर्काइव बनाएं, पासवर्ड सेट करें, एंट्रीज़ जोड़ें, और सहेजें – सभी कुछ ही लाइनों में।

## पूर्वापेक्षाएँ

कोड में जाने से पहले सुनिश्चित करें कि आपके पास:

- **Aspose.Zip for .NET** स्थापित हो। आप इसे [यहाँ](https://releases.aspose.com/zip/net/) डाउनलोड कर सकते हैं।  
- एक फ़ोल्डर हो जिसमें वह फ़ाइलें हों जिन्हें आप आर्काइव करना चाहते हैं। नीचे के उदाहरणों में, वेरिएबल `dataDir` उस फ़ोल्डर की ओर इशारा करता है।

## नेमस्पेस इम्पोर्ट करें

पहले, आवश्यक नेमस्पेस को स्कोप में लाएँ:

```csharp
// The path to the resource directory.
string dataDir = "Your Document Directory"

// Import Aspose.Zip namespaces
using Aspose.Zip;
using Aspose.Zip.Settings.Compression;
using Aspose.Zip.Settings.Encryption;
```

## पासवर्ड प्रोटेक्ट ज़िप आर्काइव बनाना और कई फ़ाइलें बिना संपीड़न के स्टोर करना

नीचे चरण‑दर‑चरण गाइड दिया गया है। प्रत्येक चरण में एक छोटा स्पष्टीकरण और मूल कोड ब्लॉक (बिना परिवर्तन) शामिल है।

### चरण 1: ज़िप फ़ाइल खोलें

हम एक नया `FileStream` बनाते हैं जो परिणामी आर्काइव को रखेगा।

```csharp
//ExStart: StoreMutlipleFilesWithoutCompressionWithPassword
using (FileStream zipFile = File.Open(dataDir + "StoreMutlipleFilesWithoutCompressionWithPassword_out.zip", FileMode.Create))
{
```

### चरण 2: स्रोत फ़ाइल खोलें

पहली फ़ाइल खोलें जिसे आप आर्काइव में जोड़ना चाहते हैं। अतिरिक्त फ़ाइलों के लिए आप इस ब्लॉक को दोहरा सकते हैं।

```csharp
using (FileStream source1 = File.Open(dataDir + "alice29.txt", FileMode.Open, FileAccess.Read))
{
```

### चरण 3: स्टोर कॉम्प्रेशन और AES एन्क्रिप्शन के साथ आर्काइव बनाएं

यहाँ हम आर्काइव को **स्टोर** (कोई‑संपीड़न नहीं) करने और **ज़िप फ़ाइल पासवर्ड प्रोटेक्शन** को AES‑256 के साथ लागू करने के लिए कॉन्फ़िगर करते हैं।

```csharp
using (var archive = new Archive(new ArchiveEntrySettings(new StoreCompressionSettings(), new AesEcryptionSettings("p@s$", EncryptionMethod.AES256))))
{
```

### चरण 4: आर्काइव एंट्री बनाएं और सहेजें – *create archive entry c#*

अब हम फ़ाइल को आर्काइव में जोड़ते हैं और एन्क्रिप्टेड ज़िप को डिस्क पर लिखते हैं।

```csharp
archive.CreateEntry("alice29.txt", source1);
archive.Save(zipFile);
```

> **प्रो टिप:** अधिक फ़ाइलें जोड़ने के लिए, `archive.CreateEntry("anotherFile.txt", anotherStream);` को `archive.Save(zipFile);` से पहले कॉल करें।

## सामान्य समस्याएँ और समाधान

| समस्या | कारण | समाधान |
|-------|--------|-----|
| **“Invalid password” अपवाद** | पासवर्ड गलत है या एन्क्रिप्शन विधि मेल नहीं खाती। | सुनिश्चित करें कि `AesEcryptionSettings` में पासवर्ड स्ट्रिंग वही है जो आप ज़िप खोलने के लिए उपयोग करेंगे, और `EncryptionMethod.AES256` का उपयोग कर रहे हैं। |
| **फ़ाइल आकार अपेक्षा से बड़ा** | अनजाने में संपीड़न उपयोग हो रहा है। | पुष्टि करें कि आप `StoreCompressionSettings` (कोई‑संपीड़न) का उपयोग कर रहे हैं, न कि `DeflateCompressionSettings` का। |
| **स्ट्रीम बंद नहीं हुई** | `using` स्टेटमेंट्स के लिए बंद करने वाला ब्रेसेस गायब है। | प्रत्येक `using` ब्लॉक को सही ढंग से बंद करें; नमूना कोड सही नेस्टिंग दिखाता है। |
