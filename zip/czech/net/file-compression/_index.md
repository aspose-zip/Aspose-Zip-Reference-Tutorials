## Často kladené otázky

**Q: Mohu použít Aspose.Zip k vytvoření zip archive .net na .NET Core?**  
A: Rozhodně. Aspose.Zip plně podporuje .NET Core 3.1 a novější, stejně jako .NET 5/6/7.

**Q: Jak mohu upravit existující archiv, aniž bych jej nejprve rozbalil?**  
A: Použijte API “modify zip file c#” – můžete přidávat, mazat nebo přejmenovávat položky přímo ve streamu archivu.

**Q: Která kompresní metoda poskytuje nejlepší poměr pro textové soubory?**  
A: Obecně Bzip2 nebo LZMA poskytují vyšší kompresi pro text, ale jsou pomalejší než Deflate. Otestujte obě, abyste našli správnou rovnováhu.

**Q: Existuje způsob, jak šifrovat soubory při vytváření zip archivu?**  
A: Ano, Aspose.Zip podporuje AES šifrování. Můžete nastavit heslo na objekt ZipArchive před uložením.

**Q: Musím se starat o kódování názvů souborů?**  
A: Aspose.Zip automaticky zpracovává Unicode názvy souborů, ale můžete explicitně nastavit kódování, pokud potřebujete kompatibilitu se staršími systémy.

**Q: Mohu přidat soubory do existujícího zipu, aniž bych ho znovu vytvářel?**  
A: Ano, API poskytuje metodu **add files zip .net**, která vám umožní přímo přidávat nové položky.

**Q: Podporuje Aspose.Zip streamování velkých souborů?**  
A: Ano. Můžete pracovat se streamy, abyste se vyhnuli načítání celých souborů do paměti, což je ideální pro scénáře ve velkém měřítku.
