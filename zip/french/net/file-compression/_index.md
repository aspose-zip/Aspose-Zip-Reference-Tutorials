## Questions fréquentes

**Q : Puis-je utiliser Aspose.Zip pour créer une archive zip .net sur .NET Core ?**  
R : Absolument. Aspose.Zip prend pleinement en charge .NET Core 3.1 et versions ultérieures, ainsi que .NET 5/6/7.

**Q : Comment modifier une archive existante sans l'extraire au préalable ?**  
R : Utilisez les API “modify zip file c#” – vous pouvez ajouter, supprimer ou renommer des entrées directement sur le flux de l'archive.

**Q : Quelle méthode de compression offre le meilleur taux pour les fichiers texte ?**  
R : Généralement, Bzip2 ou LZMA offrent une compression plus élevée pour le texte, mais elles sont plus lentes que Deflate. Testez les deux pour trouver le bon compromis.

**Q : Existe-t-il un moyen de chiffrer les fichiers lors de la création de l'archive zip ?**  
R : Oui, Aspose.Zip prend en charge le chiffrement AES. Vous pouvez définir un mot de passe sur l'objet ZipArchive avant l'enregistrement, activant **zip file encryption c#**.

**Q : Dois-je me préoccuper de l'encodage des noms de fichiers ?**  
R : Aspose.Zip gère automatiquement les noms de fichiers Unicode, mais vous pouvez définir explicitement l'encodage si vous avez besoin de compatibilité avec des systèmes anciens.
