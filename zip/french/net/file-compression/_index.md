## FAQ
**Q : Puis-je utiliser Aspose.Zip pour créer une archive zip .net sur .NET Core ?**  
R : Absolument. Aspose.Zip prend entièrement en charge .NET Core 3.1 et versions ultérieures, ainsi que .NET 5/6/7.

**Q : Comment modifier une archive existante sans l'extraire au préalable ?**  
R : Utilisez les API « modify zip file c# » – vous pouvez ajouter, supprimer ou renommer des entrées directement sur le flux de l'archive.

**Q : Quelle méthode de compression offre le meilleur taux pour les fichiers texte ?**  
R : En général, Bzip2 ou LZMA offrent une compression supérieure pour le texte, mais elles sont plus lentes que Deflate. Testez les deux pour trouver le bon équilibre.

**Q : Existe-t-il un moyen de chiffrer les fichiers lors de la création de l'archive zip ?**  
R : Oui, Aspose.Zip prend en charge le chiffrement AES. Vous pouvez définir un mot de passe sur l'objet ZipArchive avant de l'enregistrer.

**Q : Dois-je me préoccuper de l'encodage des noms de fichiers ?**  
R : Aspose.Zip gère automatiquement les noms de fichiers Unicode, mais vous pouvez définir explicitement l'encodage si vous avez besoin de compatibilité avec des systèmes anciens.

**Q : Puis-je ajouter des fichiers à un zip existant sans le recréer ?**  
R : Oui, l'API fournit une méthode **add files zip .net** qui vous permet d'ajouter de nouvelles entrées directement.

**Q : Aspose.Zip prend‑il en charge le streaming de gros fichiers ?**  
R : Oui. Vous pouvez travailler avec des flux pour éviter de charger des fichiers entiers en mémoire, ce qui est idéal pour les scénarios à grande échelle.
