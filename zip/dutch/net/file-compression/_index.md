## Veelgestelde vragen

**Q: Kan ik Aspose.Zip gebruiken om een zip archive .net te maken op .NET Core?**  
A: Absoluut. Aspose.Zip ondersteunt volledig .NET Core 3.1 en later, evenals .NET 5/6/7.

**Q: Hoe kan ik een bestaand archief aanpassen zonder het eerst te extraheren?**  
A: Gebruik de “modify zip file c#” API’s – je kunt items direct op de archief‑stream toevoegen, verwijderen of hernoemen.

**Q: Welke compressiemethode geeft de beste ratio voor tekstbestanden?**  
A: Over het algemeen bieden Bzip2 of LZMA een hogere compressie voor tekst, maar ze zijn langzamer dan Deflate. Test beide om de juiste balans te vinden.

**Q: Is er een manier om bestanden te versleutelen tijdens het maken van het zip‑archief?**  
A: Ja, Aspose.Zip ondersteunt AES‑encryptie. Je kunt een wachtwoord instellen op het ZipArchive‑object vóór het opslaan.

**Q: Moet ik me zorgen maken over bestandsnaam‑codering?**  
A: Aspose.Zip behandelt automatisch Unicode‑bestandsnamen, maar je kunt de codering expliciet instellen als je compatibiliteit met legacy‑systemen nodig hebt.

**Q: Kan ik bestanden toevoegen aan een bestaande zip zonder deze opnieuw te maken?**  
A: Ja, de API biedt een **add files zip .net** methode die je in staat stelt nieuwe items direct toe te voegen.

**Q: Ondersteunt Aspose.Zip streaming van grote bestanden?**  
A: Ja. Je kunt met streams werken om te voorkomen dat volledige bestanden in het geheugen worden geladen, wat ideaal is voor grootschalige scenario's.
