## 常见问题

**Q: 可以在 .NET Core 上使用 Aspose.Zip 创建 zip archive .net 吗？**  
A: 当然可以。Aspose.Zip 完全支持 .NET Core 3.1 及更高版本，以及 .NET 5/6/7。

**Q: 如何在不先解压的情况下修改已有的归档？**  
A: 使用 “modify zip file c#” API——您可以直接在归档流上添加、删除或重命名条目。

**Q: 哪种压缩方法对文本文件的压缩率最高？**  
A: 通常 Bzip2 或 LZMA 能提供更高的文本压缩率，但速度慢于 Deflate。建议两者都测试，以找到合适的平衡点。

**Q: 创建 zip 存档时可以加密文件吗？**  
A: 可以，Aspose.Zip 支持 AES 加密。您可以在保存前为 ZipArchive 对象设置密码。

**Q: 需要担心文件名编码吗？**  
A: Aspose.Zip 会自动处理 Unicode 文件名，但如果需要兼容旧系统，您可以显式设置编码。

**Q: 能否在不重新创建的情况下向已有 zip 添加文件？**  
A: 可以，API 提供了 **add files zip .net** 方法，直接在现有归档中追加新条目。

**Q: Aspose.Zip 支持流式处理大文件吗？**  
A: 支持。您可以使用流方式避免将整个文件加载到内存中，这在大规模场景下尤为适用。
