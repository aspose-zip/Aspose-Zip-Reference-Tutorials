# 如何使用 Aspose.Zip for .NET 解壓縮檔案

## 介紹

當您需要在 .NET 環境中快速且可靠地 **解壓縮檔案** 時，Aspose.Zip for .NET 提供簡潔且高效能的 API，免除手動解壓的麻煩。無論您面對的是單一壓縮檔、批次日誌檔，或是受密碼保護的 zip，本指南都會示範如何僅透過幾行 C# 程式碼即可解壓 zip 資料夾內容並處理加密的壓縮檔。

## 快速解答
- **Aspose.Zip for .NET 的功能是什麼？** 它提供簡單的 API 以在 C# 中建立、讀取與解壓 ZIP、TAR、GZIP 以及其他壓縮格式。  
- **我可以一次解壓多個檔案嗎？** 可以，函式庫允許您一次呼叫即解壓所有項目，或逐一遍歷。  
- **是否支援密碼保護的解壓？** 當然支援 – 您可以提供密碼以解鎖加密的壓縮檔。  
- **支援哪些 .NET 版本？** .NET Framework 4.5 以上、.NET Core 3.1 以上、.NET 5/6/7 及更高版本。  
- **開發是否需要授權？** 免費試用可用於評估；正式上線需購買商業授權。  

## 使用 Aspose.Zip for .NET 解壓縮檔案的方法
使用 Aspose.Zip 解壓檔案相當簡單。函式庫提供 `Extract` 方法，可將內容解壓至目錄、特定檔案或記憶體串流。以下說明一般的工作流程：

1. **建立指向 zip 檔的 `Archive` 實例**。  
2. **呼叫 `Extract`**，並指定目標資料夾。  
3. **若壓縮檔已加密，則可選擇傳入密碼**（`extract password protected zip`）。  

此方式適用於 Aspose.Zip 支援的所有壓縮類型，包含傳統 zip、zip 資料夾結構，以及 XAR、WIM 等現代格式。

## 什麼是「解壓多個檔案」？
解壓多個檔案指的是將壓縮檔（ZIP、TAR 等）內的每個項目全部抽取出來，並可選擇寫入目標目錄。當您收到捆綁的資料（如日誌檔、影像或設定檔集合）需要在處理前先解壓時，這項操作相當常見。

## 為什麼使用 Aspose.Zip for .NET 來解壓多個檔案？
- **效能最佳化**的解壓，能在不大量佔用記憶體的情況下處理大型壓縮檔。  
- **內建支援**傳統 ZIP、現代格式（XAR、WIM）以及加密壓縮檔。  
- **簡潔的 C# 語法** – 無需自行處理串流或使用第三方工具（`aspose zip extract`）。  
- **跨平台**相容性，讓相同程式碼可在 Windows、Linux 或 macOS 上執行。

## 常見問題

**Q: 我可以直接將 zip 壓縮檔解壓至記憶體串流嗎？**  
A: 可以，Aspose.Zip 允許您將條目讀入 `MemoryStream` 而不寫入磁碟（`extract zip archive c#`）。

**Q: 函式庫是否支援解壓至特定的資料夾結構？**  
A: 當然支援。您可以指定輸出目錄，API 會重建壓縮檔內部的資料夾層級。

**Q: 如何在 C# 中解壓受密碼保護的 zip 檔案？**  
A: 在 `Extract` 方法中提供密碼（例如 `archive.Extract(outputPath, password)`）。

**Q: 有沒有辦法在不解壓的情況下列出壓縮檔內容？**  
A: 有，您可以遍歷 `archive.Entries` 以檢查檔名、大小與時間戳記。

**Q: 若壓縮檔內有重複的檔名該怎麼辦？**  
A: 預設情況下，函式庫會覆寫已存在的檔案；您可以使用 `OverwriteMode` 選項變更此行為。

**Q: 我可以只解壓 zip 資料夾中的特定條目嗎？**  
A: 可以，依名稱或副檔名篩選 `archive.Entries`，然後對選取的條目呼叫 `Extract`。

**Q: Aspose.Zip 在低記憶體裝置上如何處理大型 zip 檔？**  
A: 函式庫採用延遲載入與串流機制，僅將當前條目載入記憶體。
