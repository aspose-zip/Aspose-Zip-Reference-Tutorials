## 常見問題

**Q:** 我可以在 .NET Core 上使用 Aspose.Zip 建立 zip archive .net 嗎？  
**A:** 當然可以。Aspose.Zip 完全支援 .NET Core 3.1 及以上版本，同時亦相容 .NET 5/6/7。

**Q:** 如何在不先解壓的情況下修改既有壓縮檔？  
**A:** 使用 “modify zip file c#” API，即可直接在壓縮檔串流上新增、刪除或重新命名條目。

**Q:** 哪種壓縮方式對文字檔的壓縮率最佳？  
**A:** 通常 Bzip2 或 LZMA 能提供較高的文字壓縮率，但相較於 Deflate 速度較慢。建議兩者皆測試，以找到最適合的平衡點。

**Q:** 建立 zip archive 時能同時加密檔案嗎？  
**A:** 可以，Aspose.Zip 支援 AES 加密。您可在儲存前於 ZipArchive 物件設定密碼。

**Q:** 是否需要特別處理檔名編碼？  
**A:** Aspose.Zip 會自動處理 Unicode 檔名，若需相容舊系統，可自行設定編碼。

**Q:** 能否在不重新建立的情況下向既有 zip 新增檔案？  
**A:** 可以，API 提供 **add files zip .net** 方法，讓您直接在原壓縮檔中追加新條目。

**Q:** Aspose.Zip 是否支援大型檔案的串流處理？  
**A:** 支援。您可以使用串流方式避免一次將整個檔案載入記憶體，特別適合大規模情境。
