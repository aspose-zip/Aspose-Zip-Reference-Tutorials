# كيفية استخراج ملف Zip باستخدام كلمة مرور باستخدام Aspose.Zip لـ .NET

في تطبيقات .NET الحديثة، حماية البيانات الحساسة داخل أرشيفات ZIP هي متطلب شائع. يوضح هذا الدليل **كيفية استخراج zip باستخدام كلمة مرور** عندما يستخدم كل عنصر كلمة مرور مختلفة، مما يمنحك تحكمًا دقيقًا في الأمان مع الحفاظ على بساطة عملية الاستخراج. باتباع مثال Aspose.Zip ستتمكن من رؤية كيفية تنفيذ استخراج zip محمي بكلمة مرور لكل عنصر على حدة.

- **هل أحتاج إلى ترخيص للإنتاج؟** يتطلب ترخيص تجاري؛ يتوفر نسخة تجريبية مجانية.

- **الوقت النموذجي للتنفيذ؟** حوالي 10 دقائق لسيناريو أساسي.

## ما هو “كيفية استخراج zip”؟
استخراج أرشيف ZIP يعني قراءة الحاوية المضغوطة وكتابة محتوياتها إلى نظام الملفات. عندما يكون الأرشيف محميًا بكلمة مرور، يجب أيضًا توفير كلمة المرور الصحيحة لكل عنصر قبل أن يتم فك ضغط البيانات.

## لماذا تستخدم Aspose.Zip لاستخراج محمي بكلمة مرور؟
- **أمان دقيق:** يمكن لكل ملف أن يكون له كلمة مرور خاصة به، مما يقلل المخاطر إذا تم اختراق كلمة مرور واحدة.  
- **المرونة:** يمكنك تحديد كلمة المرور برمجيًا بناءً على منطق الأعمال (مثل أدوار المستخدم).  
- **الأداء:** تقوم Aspose.Zip بمعالجة العناصر في الذاكرة، مما يجنب الحاجة إلى فك ضغط الأرشيف بالكامل أولاً.  
- **دعم متعدد المنصات:** يعمل على Windows وLinux وmacOS مع .NET 5/6+.

## استيراد مساحات الأسماء

أولاً، استورد مساحات الأسماء المطلوبة للعمل مع الأرشيفات:

```csharp
using Aspose.Zip;
using System.IO;
```

هذان بيانان `using` يمنحانك الوصول إلى فئة `Archive` وأدوات الإدخال/الإخراج القياسية.

## الخطوة 1: تعريف دليل العمل

حدد المجلد الذي يوجد فيه ملف ZIP وأين سيتم كتابة الملفات المستخرجة:

```csharp
string dataDir = "Your Document Directory";
```

> **نصيحة احترافية:** استخدم `Path.Combine` لبناء المسارات عبر المنصات إذا كنت بحاجة إلى دعم Linux/macOS.

## كيفية استخراج Zip باستخدام كلمة مرور باستخدام Aspose.Zip

فيما يلي نستعرض الخطوات الدقيقة لفتح الأرشيف واستخراج كل عنصر باستخدام كلمة مروره الخاصة. يوضح هذا القسم **استخراج zip باستخدام كلمة مرور** لكل عنصر، وهو جوهر عملية “كيفية استخراج zip”.

### الخطوة 2: استخراج عناصر الأرشيف بكلمات مرور مختلفة

#### الخطوة 2.1: فتح ملف Zip

```csharp
using (FileStream zipFile = File.Open(dataDir + "\\different_password.zip", FileMode.Open))
{
    using (Archive archive = new Archive(zipFile))
    {
        // Continue to the next steps...
    }
}
```

كائن `Archive` يمثل حاوية ZIP. الحفاظ على `FileStream` و`Archive` داخل كتل `using` يضمن تحرير جميع الموارد على الفور.

#### الخطوة 2.2: استخراج العنصر الأول (كلمة المرور = “first_pass”)

```csharp
archive.Entries[0].Extract(dataDir + "alice29_extracted_pass_out.txt", "first_pass");
```

هنا نقوم **باستخراج عدة عناصر zip** عن طريق الوصول إليها عبر مجموعة `Entries`. يتم فك تشفير العنصر الأول باستخدام كلمة المرور `"first_pass"`.

#### الخطوة 2.3: استخراج العنصر الثاني (كلمة المرور = “second_pass”)

```csharp
archive.Entries[1].Extract(dataDir + "asyoulik_extracted_pass_out.txt", "second_pass");
```

العنصر الثاني يستخدم كلمة مرور مختلفة، مما يوضح معالجة **استخراج كلمة مرور عنصر zip** لكل ملف على حدة.

#### الخطوة 2.4: (اختياري) التكرار عبر جميع العناصر

إذا كنت بحاجة إلى **استخراج عدة عناصر zip** دون ترميز الفهارس يدويًا، يمكنك التكرار عبر `archive.Entries` وتوفير كلمة المرور المناسبة لكل عنصر بناءً على منطق البحث الخاص بك. هذا النمط يتوسع بشكل جيد عند التعامل مع أرشيفات كبيرة.

| المشكلة | السبب | الحل |
|-------|--------|-----|
| *استثناء “Invalid password”* | تم توفير كلمة مرور خاطئة أو العنصر غير مشفر فعليًا. | تحقق من سلسلة كلمة المرور وتأكد من أن العنصر محمي بكلمة مرور. |
| *الملف غير موجود* | مسار `dataDir` غير صحيح. | استخدم `Path.Combine(dataDir, "different_password.zip")` وتحقق مرة أخرى من المجلد. |
