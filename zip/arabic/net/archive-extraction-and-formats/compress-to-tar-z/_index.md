# إضافة ملفات إلى tar وضغطها إلى TarZ باستخدام Aspose.Zip لـ .NET

## مقدمة

إذا كنت بحاجة إلى **إضافة ملفات إلى tar** ثم ضغط الأرشيف إلى تنسيق TarZ، فإن Aspose.Zip لـ .NET يجعل العملية بأكملها سهلة. في هذا الدرس سنستعرض كل خطوة — من إعداد مشروعك إلى إنشاء أرشيف tar، إضافة الملفات، وأخيرًا حفظ ملف .tar.z مضغوط. في النهاية ستحصل على قطعة كود قابلة لإعادة الاستخدام يمكنك إدراجها في أي تطبيق .NET.

## إجابات سريعة
- **ما المكتبة التي تتعامل مع إنشاء tar؟** Aspose.Zip for .NET  
- **كم عدد أسطر الكود؟** حوالي 15 سطرًا (باستثناء التعليقات)  
- **هل أحتاج إلى ترخيص للاختبار؟** نسخة تجريبية مجانية متاحة؛ الترخيص مطلوب للإنتاج.  
- **الإصدارات المدعومة من .NET؟** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+  
- **هل يمكنني ضغط المجلدات وليس الملفات فقط؟** نعم – يمكنك إضافة دلائل كاملة باستخدام حلقة.

## ما هو **إضافة ملفات إلى tar**؟

إضافة ملفات إلى أرشيف tar يجمعها في حاوية واحدة غير مضغوطة تحافظ على بنية الدليل وبيانات تعريف الملفات. Tar هو تنسيق يونكس كلاسيكي ويُعد أساسًا للعديد من سير عمل الضغط، بما في ذلك تنسيق TarZ المستخدم في هذا الدليل.

## لماذا إضافة ملفات إلى tar قبل الضغط إلى TarZ؟

- **قابلية النقل** – يعمل أرشيف tar عبر المنصات دون القلق بشأن معالجة كل ملف على حدة.  
- **السرعة** – إنشاء حاوية tar سريع؛ الضغط اللاحق بنظام Z يركز فقط على تقليل الحجم.  
- **التوافق** – تتوقع العديد من الأدوات القديمة وجود `.tar` قبل تطبيق ضغط بنمط gzip، وهذا ما يقدمه `.tar.z` بالضبط.  

### لماذا هذا مهم لمطوري .NET

استخدام حاوية tar يتيح لك الحفاظ على شفرة .NET بسيطة ومحددة. يمكنك إنشاء الأرشيف في الذاكرة، بثه مباشرةً إلى استجابة، أو تخزينه على القرص دون التعامل مع ملفات zip مؤقتة. هذا النمط مفيد بشكل خاص لخطوط بناء البرمجيات، تجميع السجلات، أو عندما تحتاج إلى إرسال مجموعة من ملفات التكوين إلى خدمة تعمل على Linux.

## استيراد مساحات الأسماء

أضف عبارات `using` المطلوبة في أعلى ملف C# الخاص بك:

```csharp
using System;
using Aspose.Zip.Tar;
```

> **نصيحة احترافية:** استخدم `Path.Combine` إذا كنت بحاجة إلى بناء المسارات بشكل ديناميكي؛ فهو يتجنب فقدان فواصل المسار على أنظمة تشغيل مختلفة.

## دليل خطوة بخطوة

### الخطوة 1: تعريف دليل المستند الخاص بك

```csharp
string dataDir = "Your Document Directory";
```

> **لماذا هذه الخطوة مهمة:** `dataDir` يعمل كموقع أساسي لكل ملف ستضيفه. الاحتفاظ به في متغير واحد يجعل الشفرة سهلة الصيانة وإعادة الاستخدام عبر أرشيفات متعددة.

### الخطوة 2: إنشاء أرشيف Tar وإضافة ملفات

#### 2.1: إنشاء كائن أرشيف Tar

```csharp
using (TarArchive archive = new TarArchive())
{
    // Your code for creating the Tar archive goes here
}
```

> كتلة `using` تضمن أن كائن `TarArchive` يتم تحريره بشكل صحيح، مما يحرر أي مقبض ملف أو مخزن مؤقت للذاكرة.

#### 2.2: إضافة ملفات إلى الأرشيف  

داخل كتلة `using`، أضف كل ملف تريد تضمينه:

```csharp
archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
```

يمكنك تكرار `CreateEntry` لعدد الملفات المطلوبة، أو تنفيذ حلقة عبر دليل لإضافتها برمجيًا. على سبيل المثال، حلقة `foreach (var file in Directory.GetFiles(dataDir))` ستمكنك من معالجة عدد عشوائي من الملفات مع الحفاظ على مساراتها النسبية.

#### 2.3: حفظ ملف TarZ المضغوط  

بعد إضافة جميع الإدخالات، اضغط أرشيف tar إلى تنسيق `.tar.z`:

```csharp
archive.SaveZCompressed(dataDir + "archive.tar.z");
```

سيتم وضع ملف `archive.tar.z` الناتج في نفس المجلد الذي حددته في `dataDir`. يمكنك الآن إرسال هذه الحزمة المضغوطة الوحيدة إلى أي نظام يفهم TarZ.

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|---------|-------|------|
| **الملف غير موجود** | مسار خاطئ أو امتداد ملف مفقود | تحقق من أن `dataDir` ينتهي بفاصل مسار وأن أسماء الملفات صحيحة. |
| **تم رفض الوصول** | أذونات غير كافية على المجلد الهدف | شغّل التطبيق بصلاحيات مناسبة أو اختر دليلًا قابلًا للكتابة. |
| **الملف المضغوط أكبر من المتوقع** | الملفات الأصلية مضغوطة بالفعل (مثل الصور، الفيديوهات) | يعمل TarZ بشكل أفضل على ملفات النص أو السجلات؛ فكر في ترك الملفات المضغوطة مسبقًا كما هي. |

### المخاطر الشائعة التي يجب الانتباه إليها
- **غياب الشرطة المائلة النهائية** – إذا لم ينتهي `dataDir` بـ `\` أو `/`، فإن دمج السلاسل سينتج مسارًا غير صالح.  
- **دليل كبير** – إضافة آلاف الملفات قد تستهلك الذاكرة؛ فكر في تدفق الإدخالات أو استخدام نسخة `TarArchive` التي تكتب مباشرةً إلى تدفق ملف.  
- **مشكلات الترميز** – قد تحتاج أسماء الملفات غير ASCII إلى معالجة ترميز صريحة؛ Aspose.Zip يحترم UTF‑8 بشكل افتراضي، لكن تحقق من ذلك على المنصة المستهدفة.
