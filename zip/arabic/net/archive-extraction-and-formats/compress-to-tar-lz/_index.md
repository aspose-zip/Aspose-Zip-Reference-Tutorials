# كيفية ضغط tar.lz باستخدام Aspose.Zip لـ .NET

في تطوير .NET الحديث، يمكن أن يؤدي تجميع الملفات بكفاءة إلى تحسين كبير في حجم النشر وأوقات نقل البيانات عبر الشبكة. **كيفية ضغط tar.lz** هو طلب شائع عندما تحتاج إلى أرشيف TAR خفيف الوزن ومضغوط بـ LZ. في هذا الدرس سنستعرض مثالًا واضحًا خطوة بخطوة **لضغط tar.lz** باستخدام مكتبة Aspose.Zip، حتى تتمكن من إنشاء أرشيف tar.lz بسرعة في تطبيقاتك.

## إجابات سريعة
- **ما المكتبة التي يجب أن أستخدمها؟** Aspose.Zip for .NET.
- **كم من الوقت تستغرق التنفيذ؟** حوالي 5‑10 دقائق لمثال أساسي.
- **هل أحتاج إلى ترخيص؟** نسخة تجريبية مجانية تكفي للاختبار؛ يلزم ترخيص تجاري للإنتاج.
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7.
- **هل يمكنني ضغط عدة ملفات في آن واحد؟** نعم – فقط أضف المزيد من الإدخالات قبل الحفظ.

## ما هو ضغط tar.lz؟
`tar.lz` هو أرشيف TAR تم ضغطه باستخدام خوارزمية LZMA (غالبًا ما يُشار إليها ببساطة بـ **LZ**). يجمع بين بساطة تجميع الملفات في TAR ونسبة الضغط العالية لـ LZ، مما يجعله مثاليًا لملفات النسخ الاحتياطي، توزيع الحزم، أو أي سيناريو حيث تكون سعة النطاق مهمًا.

## لماذا استخدام Aspose.Zip لـ .NET؟
Aspose.Zip يخفف تفاصيل المستوى المنخفض لإنشاء الأرشيفات، ويقدم لك واجهة برمجة تطبيقات كائنية نظيفة. ستحصل على:
* **عدم وجود تبعيات خارجية** – تنفيذ نقي بلغة .NET.  
* **دعم متعدد المنصات** – يعمل على Windows وLinux وmacOS.  
* **ضغط LZ مدمج** – لا حاجة لتثبيت أدوات أصلية إضافية.  
* **معالجة أخطاء قوية** – الاستثناءات توضح السبب، مما يسهل عملية التصحيح.

## المتطلبات المسبقة
قبل البدء، تأكد من وجود ما يلي:

- مجلد يحتوي على الملفات التي تريد أرشفتها. سيتم تخزين مسار هذا المجلد في المتغير `dataDir` (ستقوم بتعيينه في الخطوة 3).

## استيراد مساحات الأسماء
أضف مساحات الأسماء المطلوبة حتى يعرف المترجم مكان العثور على الفئات التي سنستخدمها.

```csharp
using System;
using Aspose.Zip.Tar;
```

## كيفية إنشاء أرشيف tar.lz – دليل خطوة بخطوة

### الخطوة 1: ضغط ملف واحد
المثال الأول يوضح أبسط سيناريو – إضافة ملف واحد إلى أرشيف TAR ثم حفظه كملف **tar.lz**.

```csharp
//ExStart: CompressSingleFile
using (TarArchive archive = new TarArchive())
{
    archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
    archive.SaveLzipped(dataDir + "archive.tar.lz");
}
```

**شرح**
- `new TarArchive()` ينشئ حاوية TAR فارغة.  
- `CreateEntry` يضيف الملف `alice29.txt` من مجلد `dataDir` الخاص بك.  
- `SaveLzipped` يكتب الأرشيف إلى القرص ويطبق ضغط LZ، منتجًا `archive.tar.lz`.

### الخطوة 2: ضغط عدة ملفات في أرشيف واحد
غالبًا ما تحتاج إلى تجميع عدة ملفات معًا. ما عليك سوى استدعاء `CreateEntry` لكل ملف قبل الحفظ.

```csharp
//ExStart: CompressMultipleFiles
using (TarArchive archive = new TarArchive())
{
    archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
    archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
    archive.SaveLzipped(dataDir + "archive.tar.lz");
}
```

**شرح**
- يتبع الكود نفس النمط كما في الخطوة 1، لكنه يضيف إدخالًا ثانيًا (`lcet10.txt`).  
- يمكنك تكرار `CreateEntry` عددًا لا نهائيًا حسب الحاجة؛ المكتبة تدير بنية TAR الداخلية تلقائيًا.

### الخطوة 3: تحديد دليل المستندات الخاص بك
استبدل العنصر النائب بالمسار الفعلي حيث توجد ملفات المصدر الخاصة بك. يُستخدم هذا المسار في الأمثلة أعلاه.

```csharp
string dataDir = "Your Document Directory";
```

**شرح**
- عيّن `dataDir` إلى مسار مجلد مؤهل بالكامل، مثل `@"C:\MyFiles\"`.  
- حفظ الدليل في متغير يجعل الكود قابلًا لإعادة الاستخدام وأسهل في الصيانة.
