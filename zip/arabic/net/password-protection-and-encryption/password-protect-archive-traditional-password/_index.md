# إنشاء ملف ZIP محمي بكلمة مرور باستخدام Aspose.Zip لـ .NET

في عالم تطوير .NET، تعلم كيفية **إنشاء ملف zip محمي بكلمة مرور** يُعد جانبًا حيويًا من تصميم التطبيقات. يوفر Aspose.Zip لـ .NET حلاً قويًا لـ **إضافة كلمة مرور إلى ملفات zip** باستخدام تشفير كلمة مرور تقليدي. سيوجهك هذا الدليل خطوة بخطوة خلال العملية، مع ضمان بقاء البيانات المؤرشفة سرية ومأمونة.

## إجابات سريعة
- **ماذا يعني “إنشاء ملف zip محمي بكلمة مرور”?** يعني إنشاء أرشيف ZIP تكون محتوياته مشفرة ولا يمكن فتحه إلا باستخدام كلمة المرور الصحيحة.  
- **ما المكتبة التي يمكنني استخدامها؟** يوفر Aspose.Zip لـ .NET دعمًا مدمجًا لحماية كلمة المرور التقليدية.  
- **هل أحتاج إلى ترخيص؟** تتوفر نسخة تجريبية مجانية، لكن يلزم الحصول على ترخيص تجاري للاستخدام في الإنتاج.  
- **هل يمكنني استخدامه مع .NET Core؟** نعم، تعمل المكتبة مع .NET Framework و .NET Core و .NET 5/6+.  
- **كم من الوقت تستغرق عملية التنفيذ؟** عادةً أقل من 10 دقائق لإنشاء أرشيف محمي بكلمة مرور أساسية.

## ما هو “إنشاء ملف zip محمي بكلمة مرور”؟
إنشاء ملف zip محمي بكلمة مرور يعني ضغط ملف أو أكثر في حاوية ZIP وتشفير الحاوية بكلمة مرور. يمكن مشاركة أو تخزين الأرشيف الناتج بأمان لأن محتوياته غير قابلة للقراءة دون كلمة المرور الصحيحة.

## لماذا تستخدم Aspose.Zip لحماية كلمة مرور أرشيف ZIP؟
- **تكامل كامل مع .NET** – يعمل بسلاسة مع مشاريع C# و VB.NET.  
- **تشفير تقليدي** – متوافق مع معظم أدوات ZIP التي تدعم حماية كلمة المرور.  
- **بدون تبعيات خارجية** – تتعامل المكتبة مع الضغط والتشفير والحفظ في واجهة برمجة تطبيقات واحدة.  
- **محسّن للأداء** – مناسب للملفات الصغيرة مثل السجلات وكذلك حزم المستندات الكبيرة.

## استيراد مساحات الأسماء
أولاً، استورد مساحات الأسماء التي تمنحك الوصول إلى فئات الضغط والتشفير.

```csharp
using Aspose.Zip;
using System.IO;
using Aspose.Zip.Saving;
```

## الخطوة 1: فتح دليل الموارد
حدد الدليل الذي يحتوي على الملفات التي تريد أرشفتها. سيُستخدم هذا المسار عند إنشاء تدفق ZIP.

## الخطوة 2: إنشاء الأرشيف مع كلمة مرور تقليدية
الآن سنقوم بإنشاء الأرشيف و**إضافة كلمة مرور إلى zip** باستخدام `TraditionalEncryptionSettings`. كلمة المرور `"p@s$"` هي مجرد مثال؛ استبدلها بسر قوي من اختيارك.

```csharp
//ExStart: PasswordProtectArchiveWithTraditionalPassword
using (FileStream zipFile = File.Open(dataDir + "CompressWithTraditionalEncryption_out.zip", FileMode.Create))
{
    using (FileStream source1 = File.Open(dataDir + "alice29.txt", FileMode.Open, FileAccess.Read))
    {
        var archive = new Archive(new ArchiveEntrySettings(null, new TraditionalEncryptionSettings("p@s$")));
        archive.CreateEntry("alice29.txt", source1);
        archive.Save(zipFile);
    }
}
//ExEnd: PasswordProtectArchiveWithTraditionalPassword 
```

> **نصيحة احترافية:** احفظ كلمة المرور بأمان (مثلاً في Azure Key Vault) بدلاً من تضمينها مباشرة في الشيفرة.

## الخطوة 3: حفظ الأرشيف
استدعاء `archive.Save(zipFile);` يكتب عملية **حفظ zip مع كلمة مرور** إلى القرص. بعد هذه الخطوة، يصبح الملف `CompressWithTraditionalEncryption_out.zip` أرشيف ZIP محمي بالكامل بكلمة مرور وجاهز للتوزيع.

## كيفية إضافة كلمة مرور إلى ملفات zip باستخدام Aspose.Zip .NET
عند استدعاء `TraditionalEncryptionSettings` فإنك تخبر Aspose.Zip باستخدام خوارزمية تشفير ZIP القديمة. هذه الطريقة سريعة، تعمل على جميع المنصات، وهي أبسط طريقة لـ **حفظ zip مع كلمة مرور** عندما لا تحتاج إلى تشفير AES الأقوى.

## كيفية ضغط الملفات مع كلمة مرور
إذا كنت بحاجة لحماية ملفات متعددة، ما عليك سوى تكرار استدعاء `CreateEntry` لكل ملف داخل نفس كتلة `using`. سيورث كل إدخال نفس كلمة المرور، مما يتيح لك **ضغط الملفات مع كلمة مرور** في أرشيف واحد.

## كيفية تشفير أرشيفات zip (تقليدي مقابل AES)
التشفير التقليدي مدعوم على نطاق واسع، ولكن للبيانات الحساسة للغاية يُنصح باستخدام خيار AES‑256 المقدم من Aspose.Zip. نمط الشيفرة هو نفسه؛ كل ما عليك هو استبدال `TraditionalEncryptionSettings` بـ `AesEncryptionSettings`.
