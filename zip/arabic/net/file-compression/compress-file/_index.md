## ما هو ضغط الملفات ولماذا هو مهم؟

ضغط الملفات يقلل من حجم البيانات عن طريق إزالة التكرار، مما يوفر مساحة على القرص ويقصر أوقات نقل الشبكة. عندما تحتاج إلى أرشفة السجلات، حزم الموارد للنشر، أو ببساطة الحفاظ على النسخ الاحتياطية مرتبة، يصبح معرفة **كيفية ضغط الملفات** برمجيًا مهارة قيمة.

## استيراد مساحات الأسماء

لبدء العمل، تحتاج إلى استيراد مساحات الأسماء الضرورية. في كود C# الخاص بك، أضف مساحات الأسماء التالية:

```csharp
using System;
using Aspose.Zip.Cpio;
```

الآن، دعنا نقسم مثال الكود إلى خطوات متعددة.

## الخطوة 1: تعيين دليل المستندات الخاص بك

قبل ضغط الملفات، عيّن الدليل الذي تُخزن فيه مستنداتك. استبدل `"Your Document Directory"` بالمسار الفعلي لدليل المستندات الخاص بك.

```csharp
string dataDir = "Your Document Directory";
```

## الخطوة 2: ضغط ملف

الآن، دعنا نتعمق في الكود الخاص بضغط ملف. يوضح هذا المثال كيفية ضغط الملفات باستخدام الفئة `CpioArchive`.

```csharp
//ExStart: CompressFile
using (CpioArchive archive = new CpioArchive())
{
    archive.CreateEntries(dataDir);
    archive.Save(dataDir + "archive.cpio");
}
//ExEnd: CompressFile
Console.WriteLine("Successfully Compressed Files");
```

### الشرح

- **فئة `CpioArchive`** – تمثل أرشيف Cpio وتوفر طرقًا لإنشاء وتعديل إدخالات الأرشيف.  
- **طريقة `CreateEntries`** – تفحص الدليل المحدد وتضيف كل ملف إلى الأرشيف (مثالية لـ *c# file compression* للمجلدات الكاملة).  
- **طريقة `Save`** – تكتب الأرشيف إلى القرص؛ في هذا المثال يُحفظ الملف باسم `archive.cpio`.  
- **رسالة النجاح** – تؤكد أن عملية الضغط انتهت دون أخطاء.

## الأسئلة الشائعة

**س: ماذا يحدث إذا كان الدليل المصدر يحتوي على مجلدات فرعية؟**  
ج: `CreateEntries` يفحص المجلدات الفرعية بشكل متكرر، ويضيف ملفاتها إلى الأرشيف تلقائيًا.

**س: كيف يمكنني التحقق من سلامة أرشيف Cpio الذي تم إنشاؤه؟**
