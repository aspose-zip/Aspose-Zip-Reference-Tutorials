# إضافة ملف إلى ملف Zip باستخدام Aspose.Zip لـ .NET

## مقدمة

في تطوير .NET الحديث، **إضافة ملف إلى zip** يمكن أن تقدر من تكاليف التخزين ويحسن أوقات التحميل بشكل كبير. يقدم Aspose.Zip لـ .NET واجهة برمجة تطبيقات نظيفة التأثير حتى الآن لك **ضغط ملف .NET** المشاريع ببضع أسطر من الشيف فقط. في هذا البرنامج التعليمي، سنستعرض مثالًا عمليًا كاملًا يوضح لك كيفية ** إنشاء أرشيف zip ** الجملون C#، باستخدام التكنولوجيا الاعتماد على `FileStream`.

## إجابات سريعة
- **ما المكتبة التي يجب استخدامها؟** Aspose.Zip for .NET
- **هل يمكنني إضافة ملف إلى zip بسطر واحد من الشيفرة؟** نعم – `archive.CreateEntry(...)` يقوم بعمل
- **هل أحتاج إلى ترخيص للتطوير؟** نسخة مجانية كافية للاختبار؛ مطلوب للإنتاج
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6/7
- **هل هو آمن للملفات الكبيرة؟** نعم، مؤسسة بث البيانات، لذا تبقى الذاكرة منخفضة

## ما هو "إضافة ملف إلى ملف مضغوط" في Aspose.Zip؟

إضافة ملف إلى أرشيف ZIP يعني أخذ الملف الموجود على القرص (أو في الذاكرة) وكتابته داخل حاوية مضغوطة تتبع مواصفات الملف ZIP. تقوم Aspose.Zip بتجريد التفاصيل ذات المستوى المنخفض، مما يتيح لك التركيز على الأعمال التجارية المتخصصة من الحسابات الاختبارية أو الخوارزميات.

## لماذا نستخدم Aspose.Zip لـ .NET؟

- **تحسين للأداء**: يبث البيانات، متجنبًا المخازن المؤقتة.
- **مجموعة المهام المتنوعة**: يدعم التشفير، الأرشيفات المقسمة، وإعدادات الإدخالات المخصصة.
- **واجهة برمجة بسيطة**: إنشاء التنوع بسطر واحد (`CreateEntry`) حدد من الشيفرة المعرفة.
- **متعددة المنصات**: تعمل على أنظمة Windows وLinux وmacOS مع .NET Core/5+.

## المتطلبات الأساسية

- معرفة أساسيات ببرمجة C#.
- تثبيت Visual Studio (أو أي بيئة تطوير .NET مفضلة).
- مكتبة Aspose.Zip لـ .NET، والتي يمكنك تحميلها **[هنا](https://releases.aspose.com/zip/net/)**.

## استيراد مساحات الأسماء

أولاً، قم بتضمين مساحات الأسماء المطلوبة في ملف C# الخاص بك:

```csharp
using Aspose.Zip;
using System.IO;
using Aspose.Zip.Saving;
```

هذه الاستيرادات تمنحك الوصول إلى فئة `Archive`، وأدوات إدخال/إخراج الملفات، وخيارات الحفظ.

## الخطوة 1: إعداد مجلد المستندات

حدد المجلد الذي يحتوي على ملف المصدر الذي تريد ضغطه. استبدل النص التوضيحي بالمسار الفعلي على جهازك.

```csharp
string dataDir = "Your Document Directory";
```

> **نصيحة احترافية:** استخدم `Path.Combine` للمسارات المستقلة عن النظام الأساسي، مثل `Path.Combine(dataDir, "alice29.txt")`.

## الخطوة 2: إنشاء ملف ZIP باستخدام FileStream

افتح كائن FileStream يشير إلى ملف ZIP الناتج. يوضح هذا المثال تقنية **ضغط الملفات باستخدام FileStream**.

```csharp
using (FileStream zipFile = File.Open(dataDir + "CompressSingleFile_out.zip", FileMode.Create))
```

بيان `using` يضمن إغلاق التيار وتفريغ الملف بشكل صحيح.

## الخطوة 3: إضافة ملف إلى الأرشيف

الآن، افتح ملف المصدر (`alice29.txt`) وأضفه إلى الأرشيف. هذه هي الخطوة الأساسية في عملية **ضغط الملفات باستخدام C#**.

```csharp
using (FileStream source1 = File.Open(dataDir + "alice29.txt", FileMode.Open, FileAccess.Read))
{
    using (var archive = new Archive(new ArchiveEntrySettings()))
    {
        archive.CreateEntry("alice29.txt", source1);

        // Save the archive
        archive.Save(zipFile);
    }
}
```

### كيف يعمل الكود
- **إعداد FileStream** – يُنشئ اتصالاً بملف ZIP الناتج.
- **CreateEntry** – يأخذ التيار المصدر (`source1`) ويكتبها في الأرشيف تحت الاسم `"alice29.txt"`.
- **حفظ** – يحفظ البيانات الجديدة إلى `CompressSingleFile_out.zip`.

يمكنك الاتصال بـ `CreateEntry` لملفات إضافية، مما يحول هذا المقتطف إلى دليل كامل لإنشاء أرشيف zip بـ C#.

## المشكلات والحلول الشائعة

| مشكلة | السبب | الحل |
|-------|--------|-----|
| **الملف غير موجود** | مسار `dataDir` غير صحيح | تحقق من سلسلة الدليل أو استخدم `Path.GetFullPath` للتصحيح |
| **تم رفض الوصول** | الموافقات ملف غير كافية | شغّل Visual Studio كمسؤول أو امنح صلاحيات كتابة للجلد |
| **ملف مضغوط كامل** | `archive.Save` تم التأكد من خروج الكتلة `باستخدام` | تأكد من أن `archive.Save(zipFile);` داخل الكتلة `using` داخلي كما هو مكتوب |
