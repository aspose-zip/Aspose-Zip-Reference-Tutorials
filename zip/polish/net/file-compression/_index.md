## Frequently Asked Questions

**Q: Czy mogę używać Aspose.Zip do tworzenia zip archive .net na .NET Core?**  
A: Absolutnie. Aspose.Zip w pełni wspiera .NET Core 3.1 i nowsze, a także .NET 5/6/7.

**Q: Jak zmodyfikować istniejące archiwum bez jego najpierw rozpakowywania?**  
A: Użyj API „modify zip file c#” – możesz dodawać, usuwać lub zmieniać nazwy wpisów bezpośrednio w strumieniu archiwum.

**Q: Która metoda kompresji daje najlepszy współczynnik dla plików tekstowych?**  
A: Zazwyczaj Bzip2 lub LZMA zapewniają wyższą kompresję tekstu, ale są wolniejsze od Deflate. Przetestuj obie, aby znaleźć odpowiedni kompromis.

**Q: Czy istnieje sposób na zaszyfrowanie plików podczas tworzenia archiwum zip?**  
A: Tak, Aspose.Zip obsługuje szyfrowanie AES. Możesz ustawić hasło na obiekcie ZipArchive przed zapisaniem.

**Q: Czy muszę martwić się o kodowanie nazw plików?**  
A: Aspose.Zip automatycznie obsługuje nazwy plików Unicode, ale możesz ręcznie ustawić kodowanie, jeśli potrzebna jest kompatybilność ze starszymi systemami.
