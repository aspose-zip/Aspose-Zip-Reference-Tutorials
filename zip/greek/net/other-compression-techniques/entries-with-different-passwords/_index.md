Η κρυπτογράφηση ενός αρχείου ZIP (ή 7z) σημαίνει την ασφάλιση των καταχωρήσεων του ώστε να μην μπορούν να ανοιχτούν χωρίς τον σωστό κωδικό πρόσβασης. ## Βήμα 2: Δημιουργήστε Καταχωρήσεις με Διαφορετικούς Κωδικούς Πρόσβασης

Αυτή είναι η καρδιά του tutorial. Ανοίγουμε ένα νέο αρχείο 7z, δημιουργούμε τρία αντικείμενα `FileInfo` και προσθέτουμε το καθένα ως καταχώρηση με τον δικό του κωδικό AES.

```csharp
//ExStart: EntriesWithDifferentPasswords
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    FileInfo fi1 = new FileInfo("data1.bin");
    FileInfo fi2 = new FileInfo("data2.bin");
    FileInfo fi3 = new FileInfo("data3.bin");

    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("entry1.bin", fi1, false, new SevenZipEntrySettings(new SevenZipStoreCompressionSettings(), new SevenZipAESEncryptionSettings("test1")));
        archive.CreateEntry("entry2.bin", fi2, false, new SevenZipEntrySettings(new SevenZipStoreCompressionSettings(), new SevenZipAESEncryptionSettings("test2")));
        archive.CreateEntry("entry3.bin", fi3, false, new SevenZipEntrySettings(new SevenZipStoreCompressionSettings(), new SevenZipAESEncryptionSettings("test3")));
        
        archive.Save(sevenZipFile);
    }
}
//ExEnd: EntriesWithDifferentPasswords
```

### Πώς Λειτουργεί Αυτό

- `SevenZipArchive` είναι το δοχείο για ένα αρχείο 7‑z.  
- `CreateEntry` δέχεται το όνομα της καταχώρησης, το αρχείο προέλευσης, μια σημαία για αντικατάσταση και ένα αντικείμενο `SevenZipEntrySettings`.  
- Μέσα στο `SevenZipEntrySettings` παρέχουμε δύο αντικείμενα ρυθμίσεων: ένα για συμπίεση (`SevenZipStoreCompressionSettings`) και ένα για κρυπτογράφηση (`SevenZipAESEncryptionSettings`).  
- Κάθε κλήση παρέχει **διαφορετικό κωδικό** (`"test1"`, `"test2"`, `"test3"`), επιτυγχάνοντας προστασία ανά καταχώρηση.

## Κρυπτογράφηση καταχωρήσεων zip με κωδικό zip ανά αρχείο – βέλτιστες πρακτικές

Όταν **κρυπτογραφείτε καταχωρήσεις zip** χρησιμοποιώντας κωδικό ανά αρχείο, λάβετε υπόψη τις παρακάτω συμβουλές:

1. **Χρησιμοποιήστε ισχυρούς, μοναδικούς κωδικούς** – αποφύγετε κοινές λέξεις και επαναχρησιμοποίηση.  
2. **Αποθηκεύστε τους κωδικούς με ασφάλεια** – σκεφτείτε έναν διαχειριστή κωδικών ή ένα ασφαλές vault αν πρέπει να τους διανείμετε.  
3. **Δοκιμάστε με πολλαπλά εργαλεία** – βεβαιωθείτε ότι τόσο το 7‑Zip όσο και το WinRAR μπορούν να διαβάσουν το αρχείο, καθώς ορισμένα παλαιότερα εργαλεία μπορεί να μην υποστηρίζουν AES‑256.

## Προστασία κωδικού ZIP αρχείου – συνηθισμένα προβλήματα

| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| **Σφάλμα λανθασμένου κωδικού** | Η συμβολοσειρά κωδικού περιέχει περιττά κενά ή αόρατους χαρακτήρες. | Κόψτε τα κενά των συμβολοσειρών κωδικού (`new SevenZipAESEncryptionSettings(password.Trim())`). |
| **Το αρχείο δεν ανοίγει σε παλαιότερα εργαλεία** | Ορισμένα παλιά εργαλεία ZIP δεν υποστηρίζουν την κρυπτογράφηση AES‑256 που χρησιμοποιείται από το 7z. | Χρησιμοποιήστε έναν σύγχρονο αποσυμπιεστή (7‑Zip 19.00+). |
| **Το αρχείο δεν προστέθηκε στο αρχείο** | Η διαδρομή του αρχείου προέλευσης είναι λανθασμένη ή το αρχείο δεν υπάρχει. | Επαληθεύστε το `dataDir` και τα ονόματα αρχείων, ή χρησιμοποιήστε `Path.Combine(dataDir, "data1.bin")`. |
