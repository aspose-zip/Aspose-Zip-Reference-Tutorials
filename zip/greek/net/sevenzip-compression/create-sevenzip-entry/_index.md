# Δημιουργία κρυπτογραφημένου αρχείου 7z με το Aspose.Zip για .NET

## Εισαγωγή

Σε αυτό το tutorial θα μάθετε **πώς να δημιουργείτε κρυπτογραφημένα αρχεία 7z** χρησιμοποιώντας τη βιβλιοθήκη Aspose.Zip για .NET. Είτε χρειάζεστε προστασία ευαίσθητων δεδομένων, είτε θέλετε να συμμορφωθείτε με πολιτικές ασφαλείας, είτε απλώς να συμπιέσετε αρχεία αποδοτικά, αυτός ο οδηγός σας καθοδηγεί βήμα‑βήμα—from τη ρύθμιση του έργου μέχρι την επιβεβαίωση ότι το αρχείο δημιουργήθηκε επιτυχώς. Ας βουτήξουμε και δούμε πόσο εύκολο είναι να προσθέσετε ένα αρχείο σε ένα αρχείο 7z με κρυπτογράφηση AES.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει “create encrypted 7z”;** Σημαίνει τη δημιουργία ενός αρχείου 7‑zip που προστατεύεται με κρυπτογράφηση AES.
- **Ποια βιβλιοθήκη χρησιμοποιείται;** Aspose.Zip για .NET.
- **Χρειάζομαι άδεια;** Μια προσωρινή άδεια είναι επαρκής για δοκιμές· απαιτείται πλήρης άδεια για παραγωγή.
- **Μπορώ να προσθέσω πολλαπλά αρχεία;** Ναι, μπορείτε να καλέσετε `CreateEntry` επανειλημμένα για κάθε αρχείο.
- **Υποστηρίζεται η κρυπτογράφηση AES;** Ναι, το Aspose.Zip υποστηρίζει κρυπτογράφηση AES‑256 για αρχεία 7z.

## Τι είναι ένα κρυπτογραφημένο αρχείο 7z;
Ένα αρχείο 7z είναι μορφή κοντέινερ υψηλής συμπίεσης. Όταν **δημιουργείτε κρυπτογραφημένα αρχεία 7z**, το περιεχόμενο ανακατεύεται χρησιμοποιώντας κρυπτογράφηση AES, καθιστώντας το αδιάβαστο χωρίς τον σωστό κωδικό πρόσβασης. Αυτό είναι ιδανικό για ασφαλή μετάδοση ή αποθήκευση εμπιστευτικών αρχείων.

## Γιατί να χρησιμοποιήσετε το Aspose.Zip για κρυπτογραφημένα αρχεία 7z;
- **Πλήρης ενσωμάτωση με .NET** – λειτουργεί με .NET Framework, .NET Core και .NET 5/6.
- **Ενσωματωμένη υποστήριξη AES‑256** – χωρίς ανάγκη εξωτερικών εργαλείων.
- **Απλό API** – κλήσεις μίας γραμμής για προσθήκη αρχείων και αποθήκευση του αρχείου.
- **Διαπλατφορμική** – τρέχει σε Windows, Linux και macOS.

## Προαπαιτούμενα

Πριν ξεκινήσουμε, βεβαιωθείτε ότι έχετε τα εξής:

- **Aspose.Zip για .NET Library** – κατεβάστε το [εδώ](https://releases.aspose.com/zip/net/).
- **Έναν φάκελο με δικαιώματα εγγραφής** στον υπολογιστή σας όπου θα αποθηκευτεί το αρχείο.
- **Ένα πηγαίο αρχείο** (π.χ. `file.dat`) που θέλετε να συμπιέσετε και να κρυπτογραφήσετε.

## Εισαγωγή Namespaces

Προσθέστε το απαιτούμενο namespace στην αρχή του αρχείου C#:

```csharp
using Aspose.Zip.SevenZip;
```

## Οδηγός βήμα‑βήμα

### Βήμα 1: Ορισμός του Καταλόγου Εργασίας

Ορίστε τη διαδρομή προς το φάκελο που περιέχει το πηγαίο αρχείο που θέλετε να συμπιέσετε.

```csharp
string dataDir = "Your Document Directory";
```

Αντικαταστήστε το `"Your Document Directory"` με την πραγματική διαδρομή στον υπολογιστή σας.

### Βήμα 2: Δημιουργία του Κρυπτογραφημένου Εισόδου 7z

Η καρδιά του tutorial – ανοίγουμε ένα νέο stream αρχείου, δημιουργούμε ένα `SevenZipArchive`, προσθέτουμε μια είσοδο και αποθηκεύουμε το αρχείο. Αυτό το παράδειγμα προσθέτει ένα μόνο αρχείο (`file.dat`) ως `data.bin` μέσα στο αρχείο.

```csharp
//ExStart: CreateSevenZipEntry
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("data.bin", "file.dat");
        archive.Save(sevenZipFile);
    }
}
//ExEnd: CreateSevenZipEntry
```

> **Συμβουλή:** Για ενεργοποίηση της κρυπτογράφησης AES, ορίστε την ιδιότητα `Encryption` στο `SevenZipArchive` πριν καλέσετε το `Save`. (Η ιδιότητα παραλείπεται εδώ για συντομία.)

### Βήμα 3: Επιβεβαίωση Επιτυχίας

Εκτυπώστε ένα φιλικό μήνυμα ώστε να γνωρίζετε ότι η λειτουργία ολοκληρώθηκε χωρίς σφάλματα.

```csharp
Console.WriteLine("Successfully Created a Seven Zip File with AES Encryption Settings");
```

### Βήμα 4: Επαλήθευση του Αρχείου (Προαιρετικό)

Μετά την εκτέλεση του προγράμματος, μεταβείτε στον φάκελο που περιέχει το `archive.7z` και προσπαθήστε να το ανοίξετε με έναν πελάτη 7‑zip. Θα πρέπει να σας ζητηθεί κωδικός πρόσβασης εάν προσθέσατε κρυπτογράφηση στο Βήμα 2.

## Συνηθισμένα Προβλήματα & Λύσεις

| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| **File not found** | Λανθασμένο `dataDir` ή όνομα πηγαίου αρχείου | Ελέγξτε ξανά τη διαδρομή και βεβαιωθείτε ότι το `file.dat` υπάρχει. |
| **Access denied** | Ανεπαρκή δικαιώματα εγγραφής | Εκτελέστε την εφαρμογή με αυξημένα δικαιώματα ή επιλέξτε φάκελο με δικαιώματα εγγραφής. |
| **Encryption not applied** | Έλλειψη ρυθμίσεων κρυπτογράφησης στο αρχείο | Ορίστε `archive.Encryption = EncryptionAlgorithm.Aes256;` πριν το `Save`. |
