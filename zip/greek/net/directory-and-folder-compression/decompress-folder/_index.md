# Συμπίεση Καταλόγου σε Zip & Αποσυμπίεση – Aspose.Zip για .NET

Αν χρειάζεστε **συμπίεση καταλόγου σε zip** και στη συνέχεια εξαγωγή του αρχείου σε μια εφαρμογή .NET, βρίσκεστε στο σωστό μέρος. Σε αυτό το tutorial θα περάσουμε από όλη τη ροή εργασίας — ξεκινώντας με τη δημιουργία ενός zip αρχείου, και στη συνέχεια θα σας δείξουμε μια καθαρή, βήμα‑βήμα αποσυμπίεση χρησιμοποιώντας το Aspose.Zip για .NET. Στο τέλος θα έχετε ένα επαναχρησιμοποιήσιμο μοτίβο για zip συμπίεση σε έργα .NET και μια σταθερή κατανόηση του πώς να αποσυμπιέζετε με στυλ .NET.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει “compress directory to zip”;** Σημαίνει τη μετατροπή των περιεχομένων ενός φακέλου σε ένα ενιαίο αρχείο .zip.  
- **Πώς εξάγω zip σε φάκελο;** Χρησιμοποιήστε τη μέθοδο `Archive.ExtractToDirectory` όπως φαίνεται στον οδηγό.  
- **Ποιες εκδόσεις .NET υποστηρίζονται;** Όλες οι σύγχρονες εκδόσεις .NET Framework, .NET Core και .NET 5/6+.  
- **Απαιτείται άδεια για παραγωγή;** Ναι, απαιτείται εμπορική άδεια Aspose.Zip για χρήση εκτός δοκιμής.  
- **Μπορώ να το αυτοματοποιήσω σε CI/CD pipelines;** Απόλυτα — απλώς προσθέστε τον ίδιο κώδικα στα scripts κατασκευής.

## Τι είναι το “compress directory to zip”;
Η συμπίεση ενός καταλόγου σε zip ενώνει κάθε αρχείο και υπο‑φάκελο σε ένα ενιαίο συμπιεσμένο αρχείο. Αυτό μειώνει το μέγεθος αποθήκευσης, απλοποιεί τη μεταφορά και αποτελεί τυπικό τρόπο πακέτωσης πόρων για ανάπτυξη.

## Γιατί να χρησιμοποιήσετε το Aspose.Zip για .NET;
Το Aspose.Zip προσφέρει ένα **pure‑managed** API που λειτουργεί χωρίς εγγενείς εξαρτήσεις, υποστηρίζει μεγάλα αρχεία και παρέχει υψηλής απόδοσης δυνατότητες zip συμπίεσης .NET. Επιπλέον διαχειρίζεται περιπτώσεις όπως αρχεία με προστασία κωδικού και ονόματα Unicode από την αρχή.

## Προαπαιτούμενα
- Βιβλιοθήκη **Aspose.Zip για .NET** εγκατεστημένη (κατεβάστε την [εδώ](https://releases.aspose.com/zip/net/)).  
- Ένας φάκελος στο δίσκο που θέλετε να συμπιέσετε — ορίστε τη διαδρομή του στη μεταβλητή `dataDir`.  
- Περιβάλλον ανάπτυξης .NET (Visual Studio, VS Code ή οποιοδήποτε IDE προτιμάτε).

## Εισαγωγή Namespaces
Πρώτα, φέρετε τα απαιτούμενα namespaces στο scope:

```csharp
using Aspose.Zip;
using System.IO;
```

## Βήμα 1: Συμπίεση Καταλόγου σε Zip
Θα δημιουργήσουμε ένα zip αρχείο από τον φάκελο που σκοπεύετε να αποσυμπιέσετε αργότερα. Η βοηθητική μέθοδος `CompressDirectory.Run()` κάνει το σκληρό κομμάτι.

```csharp
string dataDir = "Your Document Directory";
CompressDirectory.Run();
```

> **Συμβουλή:** Το δείγμα `CompressDirectory` πακετάρει κάθε αρχείο στο `dataDir` στο `CompressDirectory_out.zip`. Μπορείτε να μετονομάσετε το αρχείο εξόδου ώστε να ταιριάζει με τις συμβάσεις ονοματοδοσίας σας.

## Βήμα 2: Αποσυμπίεση του Φακέλου (Πώς να unzip .NET)

### Βήμα 2.1: Άνοιγμα του Zip Αρχείου
Ανοίξτε το παραγόμενο αρχείο με ένα `FileStream`. Αυτό προετοιμάζει το αρχείο για ανάγνωση.

```csharp
using (FileStream zipFile = File.Open(dataDir + "CompressDirectory_out.zip", FileMode.Open))
{
```

### Βήμα 2.2: Δημιουργία Instance του Archive
Δημιουργήστε το αντικείμενο `Archive`, το οποίο αντιπροσωπεύει το zip container.

```csharp
    using (var archive = new Archive(zipFile))
    {
```

### Βήμα 2.3: Εξαγωγή σε Φάκελο
Τέλος, εξάγετε τα περιεχόμενα σε έναν νέο φάκελο. Αυτό είναι το βήμα **extract zip to directory**.

```csharp
        archive.ExtractToDirectory(dataDir + "DecompressFolder_out");
    }
}
```

Συγχαρητήρια! Έχετε συμπιέσει με επιτυχία έναν κατάλογο σε zip και στη συνέχεια έχετε εξάγει το zip σε φάκελο χρησιμοποιώντας το Aspose.Zip για .NET. Αυτή η προσέγγιση εγγυάται την ακεραιότητα των δεδομένων ενώ διατηρεί τον κώδικα σύντομο και ευανάγνωστο.

## Συχνά Προβλήματα & Λύσεις
| Συμπτωμα | Πιθανή Αιτία | Διόρθωση |
|----------|--------------|----------|
| `UnauthorizedAccessException` κατά την εξαγωγή | Ο φάκελος προορισμού είναι μόνο για ανάγνωση ή σε χρήση | Βεβαιωθείτε ότι η διαδρομή προορισμού είναι εγγράψιμη και δεν είναι κλειδωμένη |
| Κενός φάκελος εξόδου μετά την εξαγωγή | Λάθος διαδρομή πηγής zip | Ελέγξτε ξανά ότι το `dataDir + "CompressDirectory_out.zip"` δείχνει στο σωστό αρχείο |
| Μεγάλα αρχεία προκαλούν OutOfMemoryException | Χρήση προεπιλεγμένου μεγέθους buffer σε πολύ μεγάλα αρχεία | Χρησιμοποιήστε `ArchiveOptions` για αύξηση του buffer ή ροή αρχείων σε τμήματα |
