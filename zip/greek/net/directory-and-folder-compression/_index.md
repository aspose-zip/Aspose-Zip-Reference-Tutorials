## Common Pitfalls & Tips

- **Large files:** Όταν εργάζεστε με αρχεία μεγαλύτερα από 2 GB, ενεργοποιήστε τη λειτουργία **Zip64** για να αποφύγετε περιορισμούς μεγέθους.  
- **Path length issues:** Χρησιμοποιήστε την ιδιότητα `UseLongFileNames` εάν η ιεραρχία του φακέλου υπερβαίνει το όριο των 260 χαρακτήρων των Windows.  
- **Performance tip:** Ορίστε `CompressionLevel` σε `Fast` για γρήγορες κατασκευές, ή σε `Maximum` όταν χρειάζεστε το μικρότερο δυνατό αρχείο.

## Συχνές Ερωτήσεις

**Q: Μπορώ να δημιουργήσω ένα zip archive με προστασία κωδικού πρόσβασης χρησιμοποιώντας το Aspose.Zip;**  
A: Ναι. Κατά την αποθήκευση του αρχείου, μπορείτε να παρέχετε ένα `ZipPassword` και να επιλέξετε έναν αλγόριθμο κρυπτογράφησης όπως AES‑256.

**Q: Υποστηρίζει το Aspose.Zip τη ροή (streaming) μεγάλων αρχείων χωρίς να τα φορτώνει ολόκληρα στη μνήμη;**  
A: Απόλυτα. Μπορείτε να εργαστείτε με αντικείμενα `FileStream`, επιτρέποντας τη συμπίεση ή εξαγωγή αρχείων οποιουδήποτε μεγέθους αποδοτικά.

**Q: Τι γίνεται αν χρειαστεί να χωρίσω ένα μεγάλο archive σε πολλαπλά μέρη;**  
A: Χρησιμοποιήστε τη μέθοδο `SplitArchive` για να ορίσετε το μέγιστο μέγεθος κάθε μέρους· το Aspose.Zip θα δημιουργήσει αυτόματα διαδοχικά χωριστά αρχεία.

**Q: Είναι δυνατόν να προσθέσω αρχεία σε ένα υπάρχον zip archive;**  
A: Ναι. Ανοίξτε το archive σε λειτουργία `Update` και καλέστε `AddFile` ή `AddFolder` για να προσαρτήσετε νέο περιεχόμενο.

**Q: Ποιες .NET runtime εκδόσεις υποστηρίζονται επίσημα;**  
A: Το Aspose.Zip for .NET υποστηρίζει .NET Framework 4.5+, .NET Core 3.1+, και .NET 5/6/7+.
