# Προστασία με κωδικό πρόσβασης αρχείου Zip με Aspose.Zip .NET

## Εισαγωγή

Καλώς ήρθατε σε αυτό το βήμα‑βήμα tutorial σχετικά με **πώς να προστατεύετε με κωδικό πρόσβασης αρχεία zip** με παραδοσιακή κρυπτογράφηση χρησιμοποιώντας το Aspose.Zip για .NET. Το Aspose.Zip είναι μια ισχυρή βιβλιοθήκη που επιτρέπει στους προγραμματιστές να δημιουργούν, να διαβάζουν και να διαχειρίζονται αρχεία zip χωρίς κόπο. Σε αυτόν τον οδηγό, θα σας δείξουμε πώς να συμπιέζετε πολλαπλά αρχεία, να τα προσθέτετε σε ένα zip και να ασφαλίζετε το αρχείο με κωδικό πρόσβασης — όλα διατηρώντας τον κώδικα καθαρό και συντηρήσιμο.

## Γρήγορες Απαντήσεις
- **Τι σημαίνει “προστασία με κωδικό πρόσβασης αρχείου zip”;** Σημαίνει κρυπτογράφηση ενός αρχείου zip ώστε τα περιεχόμενά του να μπορούν να ανοιχτούν μόνο μετά την εισαγωγή του σωστού κωδικού πρόσβασης.  
- **Ποια βιβλιοθήκη το διαχειρίζεται στο .NET;** Το Aspose.Zip για .NET παρέχει ενσωματωμένη υποστήριξη για παραδοσιακή κρυπτογράφηση.  
- **Χρειάζεται άδεια για παραγωγή;** Ναι, απαιτείται εμπορική άδεια για χρήση σε παραγωγή· διατίθεται δωρεάν δοκιμή.  
- **Μπορώ να το τρέξω σε Linux;** Απόλυτα—το Aspose.Zip είναι cross‑platform και λειτουργεί σε Windows, Linux και macOS.  
- **Πόσα αρχεία μπορώ να προσθέσω;** Μπορείτε να προσθέσετε όποιον αριθμό αρχείων θέλετε· το παράδειγμα δείχνει τρία, αλλά η προσέγγιση κλιμακώνεται.

## Τι είναι η “προστασία με κωδικό πρόσβασης αρχείου zip”;

Ένα zip αρχείο προστατευμένο με κωδικό πρόσβασης χρησιμοποιεί κρυπτογράφηση (στην περίπτωση αυτή, παραδοσιακή κρυπτογράφηση PKZIP) για να «ανακατεύει» τα δεδομένα των αρχείων μέσα στο zip. Όταν ένας χρήστης προσπαθήσει να εξάγει το αρχείο, πρέπει να εισάγει τον σωστό κωδικό πρόσβασης για να αποκρυπτογραφήσει τα περιεχόμενα.

## Γιατί να χρησιμοποιήσετε το Aspose.Zip για αυτήν την εργασία;

- **Απλό API** – Μία γραμμή κώδικα για ενεργοποίηση κρυπτογράφησης.  
- **Χωρίς εξωτερικές εξαρτήσεις** – Καθαρά .NET, χωρίς native DLLs.  
- **Cross‑platform** – Λειτουργεί με .NET Framework, .NET Core και .NET 5/6+.  
- **Βελτιστοποιημένη απόδοση** – Διαχειρίζεται μεγάλα αρχεία αποδοτικά.

## Πώς να κρυπτογραφήσετε zip με παραδοσιακή κρυπτογράφηση

### Βήμα 1: Ρύθμιση του αρχείου Zip (Προστασία με κωδικό πρόσβασης)

Δημιουργήστε το αρχείο zip και διαμορφώστε τις ρυθμίσεις παραδοσιακής κρυπτογράφησης. Ο κωδικός `"p@s$"` είναι μόνο παράδειγμα—αντικαταστήστε τον με έναν ισχυρό κωδικό για πραγματικά έργα.

```csharp
//ExStart: CompressMultipleFilesWithTraditionalEncryption
using (FileStream zipFile = File.Open(".\\CompressMultipleFilesWithTraditionalEncryption_out.zip", FileMode.Create))
{
    // Create archive with traditional encryption settings
    using (var archive = new Archive(new ArchiveEntrySettings(null, new TraditionalEncryptionSettings("p@s$"))))
    {
        // Continue to the next step...
    }
}
//ExEnd: CompressMultipleFilesWithTraditionalEncryption
```

### Προσθήκη αρχείων στο zip

Τώρα, προσθέστε κάθε αρχείο που θέλετε να συμπεριλάβετε. Η μέθοδος `CreateEntry` δέχεται το όνομα του αρχείου μέσα στο zip και ένα stream (`source1`, `source2`, `source3`) που δείχνει στα πραγματικά δεδομένα του αρχείου.

```csharp
archive.CreateEntry("alice29.txt", source1);
archive.CreateEntry("asyoulik.txt", source2);
archive.CreateEntry("fields.c", source3);
```

### Αποθήκευση του αρχείου Zip (συμπίεση αρχείων με κωδικό)

Τέλος, αποθηκεύστε το αρχείο στο δίσκο. Αυτό το βήμα γράφει το κρυπτογραφημένο zip στο προαναφερθένποσό.

```csharp
archive.Save(zipFile);
```

> **Συμβουλή:** Πάντα κλείνετε τα streams (`using` statements) για να απελευθερώνετε άμεσα τους χειριστές αρχείων, ειδικά όταν εργάζεστε με μεγάλα αρχεία.

## Συχνά Προβλήματα και Λύσεις

| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| **Σφάλμα λανθασμένου κωδικού** | Ασυμφωνία κωδικού ή τυπογραφικό λάθος στο `TraditionalEncryptionSettings` | Επαληθεύστε τη συμβολοσειρά κωδικού και βεβαιωθείτε ότι χρησιμοποιείται ο ίδιος κωδικός κατά την εξαγωγή. |
| **Το αρχείο δεν προστέθηκε** | Το stream πηγής (`sourceX`) είναι null ή έχει κλείσει | Ανοίξτε τα streams πηγής με `File.OpenRead` πριν τα περάσετε στο `CreateEntry`. |
| **Μείωση απόδοσης σε μεγάλα αρχεία** | Χρήση προεπιλεγμένου επιπέδου συμπίεσης με πολλά μεγάλα entries | Σκεφτείτε να ορίσετε `CompressionLevel` στο `ArchiveEntrySettings` για ταχύτερη επεξεργασία. |
