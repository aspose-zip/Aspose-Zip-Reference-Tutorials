# Δημιουργία Αρχείων ZIP με Προστασία Κωδικού στην .NET με το Aspose.Zip

Αναθέτοντας έναν μοναδικό κωδικό σε κάθε καταχώρηση, μπορείτε να διατηρήσετε ασφαλή τα ευαίσθητα έγγραφα ενώ απολαμβάνετε την ευκολία της συμπίεσης ZIP. Σε αυτό το tutorial θα μάθετε πώς να συμπιέζετε αρχεία με ατομικούς κωδικούς, να επιλέγετε διαφορετικές μεθόδους κρυπτογράφησης και να ορίζετε κωδικό σε καταχώρηση zip — όλα με σαφή, έτοιμο‑για‑παραγωγή κώδικα.

## Βήμα 2: Συμπίεση Αρχείων με Ατομικούς Κωδικούς

Τώρα, ας συμπιέσουμε αρχεία με ατομικούς κωδικούς. Θα χρησιμοποιήσουμε τρία δείγμα αρχεία (`alice29.txt`, `asyoulik.txt` και `fields.c`) με διαφορετικούς κωδικούς για το καθένα. Αυτό δείχνει πώς να **συμπιέζετε αρχεία με κωδικούς** και επίσης πώς να **ορίζετε κωδικό σε καταχώρηση zip** χρησιμοποιώντας διαφορετικά σχήματα κρυπτογράφησης, συμπεριλαμβανομένου ενός **zip archive with aes**.

```csharp
using (FileStream zipFile = File.Open(dataDir + "CompressFilesWithIndividualPasswords_out.zip", FileMode.Create))
{
    FileInfo source1 = new FileInfo(dataDir + "alice29.txt");
    FileInfo source2 = new FileInfo(dataDir + "asyoulik.txt");
    FileInfo source3 = new FileInfo(dataDir + "fields.c");

    using (var archive = new Archive())
    {
        // Compress each file with an individual password
        archive.CreateEntry("alice29.txt", source1, true, new ArchiveEntrySettings(new DeflateCompressionSettings(), new TraditionalEncryptionSettings("pass1")));
        archive.CreateEntry("asyoulik.txt", source2, true, new ArchiveEntrySettings(new DeflateCompressionSettings(), new AesEcryptionSettings("pass2", EncryptionMethod.AES128)));
        archive.CreateEntry("fields.c", source3, true, new ArchiveEntrySettings(new DeflateCompressionSettings(), new AesEcryptionSettings("pass3", EncryptionMethod.AES256)));
        
        // Save the compressed files
        archive.Save(zipFile);
    }
}
```

### Πώς Λειτουργεί Αυτό
- **TraditionalEncryptionSettings** χρησιμοποιεί τον παλαιότερο αλγόριθμο ZipCrypto (συμβατό με τα περισσότερα εργαλεία ZIP).  
- **AesEcryptionSettings** σας επιτρέπει να επιλέξετε AES‑128 ή AES‑256 για ισχυρότερη ασφάλεια.  
- Κάθε κλήση `CreateEntry` λαμβάνει ένα αντικείμενο `ArchiveEntrySettings` όπου καθορίζουμε τόσο τη μέθοδο συμπίεσης όσο και τις ρυθμίσεις κρυπτογράφησης, προστατεύοντας έτσι **password protect zip archive** τις καταχωρήσεις.

## Κοινά Προβλήματα και Λύσεις

| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| “Invalid password” κατά το άνοιγμα του ZIP | Ασυμφωνία μεταξύ του κωδικού που χρησιμοποιείται στον κώδικα και του κωδικού που εισάγεται στον αποκωδικοποιητή | Επαληθεύστε το ακριβές string που περνάτε στο `TraditionalEncryptionSettings` ή στο `AesEcryptionSettings`. |
| Τα παλαιότερα εργαλεία εξαγωγής δεν μπορούν να ανοίξουν καταχωρήσεις κρυπτογραφημένες με AES | Δεν υποστηρίζουν όλοι οι πελάτες ZIP την κρυπτογράφηση AES | Χρησιμοποιήστε τη μέθοδο ZipCrypto για μέγιστη συμβατότητα ή συμβουλέψτε τους χρήστες να χρησιμοποιήσουν ένα σύγχρονο εργαλείο (π.χ., 7‑Zip). |
| Μεγάλα αρχεία προκαλούν `OutOfMemoryException` | Φόρτωση τεράστιων αρχείων στη μνήμη πριν τη συμπίεση | Μεταφέρετε το αρχείο απευθείας με `FileStream` χωρίς να το φορτώσετε ολόκληρο σε αντικείμενο `FileInfo`. |
