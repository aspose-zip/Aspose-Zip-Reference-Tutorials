# Δημιουργία Αρχείου ZIP με Προστασία Κωδικού Πρόσβασης σε .NET με το Aspose.Zip

## Εισαγωγή

Όταν χρειάζεται να μοιραστείτε ευαίσθητα δεδομένα, ένα **αρχείο zip με προστασία κωδικού πρόσβασης** είναι ένας από τους πιο απλούς αλλά και αποτελεσματικούς τρόπους για να διατηρήσετε τις πληροφορίες ασφαλείς. Σε εφαρμογές .NET, το Aspose.Zip καθιστά εύκολη τη **συμπίεση αρχείων με κωδικούς πρόσβασης**, δίνοντας σε κάθε αρχείο το δικό του κλειδί κρυπτογράφησης. Σε αυτό το tutorial θα μάθετε ακριβώς πώς να δημιουργήσετε ένα τέτοιο αρχείο, γιατί είναι σημαντικό και πού μπορείτε να το εφαρμόσετε σε πραγματικά έργα.

## Γρήγορες Απαντήσεις
- **Ποια βιβλιοθήκη πρέπει να χρησιμοποιήσω;** Το Aspose.Zip για .NET παρέχει ενσωματωμένη υποστήριξη για κωδικούς πρόσβασης ανά αρχείο και πολλαπλές μεθόδους κρυπτογράφησης.  
- **Πόσες γραμμές κώδικα;** Περίπου 30 γραμμές, συμπεριλαμβανομένης της ρύθμισης και της αποθήκευσης του αρχείου.  
- **Μπορεί κάθε αρχείο να έχει διαφορετικό κωδικό πρόσβασης;** Ναι – μπορείτε να αναθέσετε έναν μοναδικό κωδικό πρόσβασης σε κάθε καταχώρηση.  
- **Ποιες μέθοδοι κρυπτογράφησης είναι διαθέσιμες;** Παραδοσιακό ZipCrypto, AES‑128 και AES‑256.  
- **Χρειάζομαι άδεια για παραγωγή;** Απαιτείται εμπορική άδεια για χρήση σε παραγωγή· διατίθεται δωρεάν δοκιμή.

## Τι είναι ένα αρχείο zip με προστασία κωδικού πρόσβασης;
Ένα αρχείο zip με προστασία κωδικού πρόσβασης είναι ένα συμπιεσμένο αρχείο (ZIP) που απαιτεί κωδικό πρόσβασης για την εξαγωγή των περιεχομένων του. Ο κωδικός μπορεί να προστατεύει ολόκληρο το αρχείο ή μεμονωμένες καταχωρήσεις, και σύγχρονοι αλγόριθμοι όπως το AES‑128/256 παρέχουν ισχυρή κρυπτογράφηση.

## Γιατί να συμπιέζετε αρχεία με κωδικούς πρόσβασης χρησιμοποιώντας το Aspose.Zip;
- **Αναλυτική ασφάλεια** – αναθέστε διαφορετικό κωδικό πρόσβασης ανά αρχείο, ιδανικό για σενάρια multi‑tenant.  
- **Πολλαπλά πρότυπα κρυπτογράφησης** – επιλέξτε μεταξύ του παλαιού ZipCrypto και του ισχυρού AES.  
- **Χωρίς εξωτερικά εργαλεία** – όλα διαχειρίζονται προγραμματιστικά μέσα στη βάση κώδικα .NET.  
- **Απόδοση** – γρήγορη συμπίεση με Deflate ενώ διατηρεί το μέγεθος του αρχείου μικρό.

## Βήμα 2: Συμπίεση Αρχείων με Ατομικούς Κωδικούς Πρόσβασης

Τώρα θα δημιουργήσουμε ένα **αρχείο zip με προστασία κωδικού πρόσβασης** όπου κάθε αρχείο χρησιμοποιεί τον δικό του κωδικό και μέθοδο κρυπτογράφησης. Το παράδειγμα χρησιμοποιεί τρία δείγμα αρχεία (`alice29.txt`, `asyoulik.txt` και `fields.c`) με διαφορετικούς κωδικούς.

```csharp
using (FileStream zipFile = File.Open(dataDir + "CompressFilesWithIndividualPasswords_out.zip", FileMode.Create))
{
    FileInfo source1 = new FileInfo(dataDir + "alice29.txt");
    FileInfo source2 = new FileInfo(dataDir + "asyoulik.txt");
    FileInfo source3 = new FileInfo(dataDir + "fields.c");

    using (var archive = new Archive())
    {
        // Compress each file with an individual password
        archive.CreateEntry("alice29.txt", source1, true, new ArchiveEntrySettings(new DeflateCompressionSettings(), new TraditionalEncryptionSettings("pass1")));
        archive.CreateEntry("asyoulik.txt", source2, true, new ArchiveEntrySettings(new DeflateCompressionSettings(), new AesEcryptionSettings("pass2", EncryptionMethod.AES128)));
        archive.CreateEntry("fields.c", source3, true, new ArchiveEntrySettings(new DeflateCompressionSettings(), new AesEcryptionSettings("pass3", EncryptionMethod.AES256)));
        
        // Save the compressed files
        archive.Save(zipFile);
    }
}
```

### Πώς λειτουργεί
- **CreateEntry** προσθέτει ένα αρχείο στο αρχείο.  
- Το τέταρτο όρισμα (`ArchiveEntrySettings`) σας επιτρέπει να καθορίσετε τόσο τη συμπίεση (`DeflateCompressionSettings`) όσο και την κρυπτογράφηση (`TraditionalEncryptionSettings` ή `AesEcryptionSettings`).  
- Με τη μεταβίβαση διαφορετικού κωδικού για κάθε καταχώρηση, δημιουργείτε ένα **αρχείο zip με προστασία κωδικού πρόσβασης** όπου κάθε αρχείο ξεκλειδώνεται μόνο με το δικό του κλειδί.
