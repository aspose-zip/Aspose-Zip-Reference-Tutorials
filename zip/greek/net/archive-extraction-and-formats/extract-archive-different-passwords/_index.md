# Πώς να εξάγετε ZIP με κωδικό πρόσβασης χρησιμοποιώντας το Aspose.Zip για .NET

Σε σύγχρονες εφαρμογές .NET, η προστασία ευαίσθητων δεδομένων μέσα σε αρχεία ZIP αποτελεί συνηθισμένη απαίτηση. Αυτό το tutorial δείχνει **πώς να εξάγετε zip με κωδικό πρόσβασης** όταν κάθε καταχώρηση χρησιμοποιεί διαφορετικό κωδικό, παρέχοντάς σας λεπτομερή έλεγχο ασφαλείας ενώ διατηρεί τη διαδικασία εξαγωγής απλή.

## Γρήγορες Απαντήσεις
- **Ποια βιβλιοθήκη πρέπει να χρησιμοποιήσω;** Aspose.Zip για .NET.  
- **Μπορώ να εξάγω καταχωρήσεις που έχουν διαφορετικούς κωδικούς;** Ναι—κάθε καταχώρηση μπορεί να ανοιχθεί με τον δικό της κωδικό.  
- **Χρειάζεται άδεια για παραγωγή;** Απαιτείται εμπορική άδεια· διατίθεται δωρεάν δοκιμαστική έκδοση.  
- **Υποστηριζόμενες πλατφόρμες;** .NET Framework, .NET Core, .NET 5/6+.  
- **Τυπικός χρόνος υλοποίησης;** Περίπου 10 λεπτά για ένα βασικό σενάριο.

## Προαπαιτούμενα

Πριν ξεκινήσετε, βεβαιωθείτε ότι έχετε:

- **Aspose.Zip για .NET** εγκατεστημένο στο έργο σας. Μπορείτε να βρείτε την επίσημη τεκμηρίωση [εδώ](https://reference.aspose.com/zip/net/).  
- Περιβάλλον ανάπτυξης .NET (Visual Studio, Rider ή VS Code) που στοχεύει .NET 5 ή νεότερο.  
- Ένα αρχείο ZIP που περιέχει καταχωρήσεις κρυπτογραφημένες με **διαφορετικούς κωδικούς** (το δείγμα που χρησιμοποιείται εδώ είναι `different_password.zip`).

## Εισαγωγή Namespaces

Πρώτα, εισάγετε τα namespaces που απαιτούνται για εργασία με αρχεία:

```csharp
using Aspose.Zip;
using System.IO;
```

Αυτές οι δύο δηλώσεις `using` σας δίνουν πρόσβαση στην κλάση `Archive` και στα τυπικά εργαλεία I/O.

## Βήμα 1: Ορισμός Καταλόγου Εργασίας

Ορίστε το φάκελο όπου βρίσκεται το αρχείο ZIP και όπου θα γραφτούν τα εξαγόμενα αρχεία:

```csharp
string dataDir = "Your Document Directory";
```

> **Συμβουλή:** Χρησιμοποιήστε `Path.Combine` για δημιουργία διαδρομών που λειτουργούν σε πολλαπλές πλατφόρμες, εάν χρειάζεται υποστήριξη Linux/macOS.

## Βήμα 2: Εξαγωγή Καταχωρήσεων Αρχείου με Διαφορετικούς Κωδικούς

Παρακάτω περιγράφονται τα ακριβή βήματα για το άνοιγμα του αρχείου και την εξαγωγή κάθε καταχώρησης με τον δικό της κωδικό.

### Βήμα 2.1: Άνοιγμα του Αρχείου Zip

```csharp
using (FileStream zipFile = File.Open(dataDir + "\\different_password.zip", FileMode.Open))
{
    using (Archive archive = new Archive(zipFile))
    {
        // Continue to the next steps...
    }
}
```

Το αντικείμενο `Archive` αντιπροσωπεύει το κοντέινερ ZIP. Η διατήρηση του `FileStream` και του `Archive` μέσα σε μπλοκ `using` εξασφαλίζει ότι όλοι οι πόροι θα απελευθερωθούν άμεσα.

### Βήμα 2.2: Εξαγωγή της Πρώτης Καταχώρησης (Κωδικός = “first_pass”)

```csharp
archive.Entries[0].Extract(dataDir + "alice29_extracted_pass_out.txt", "first_pass");
```

Εδώ **εξάγουμε πολλαπλές καταχωρήσεις zip** προσπερνώντας τη συλλογή `Entries`. Η πρώτη καταχώρηση αποκρυπτογραφείται με τον κωδικό `"first_pass"`.

### Βήμα 2.3: Εξαγωγή της Δεύτερης Καταχώρησης (Κωδικός = “second_pass”)

```csharp
archive.Entries[1].Extract(dataDir + "asyoulik_extracted_pass_out.txt", "second_pass");
```

Η δεύτερη καταχώρηση χρησιμοποιεί διαφορετικό κωδικό, επιδεικνύοντας **εξαγωγή zip με κωδικό πρόσβασης** για κάθε μεμονωμένο αρχείο.

## Γιατί Είναι Σημαντική Αυτή η Προσέγγιση

- **Ακριβής ασφάλεια:** Κάθε αρχείο μπορεί να έχει τον δικό του κωδικό, μειώνοντας τον κίνδυνο σε περίπτωση παραβίασης ενός μόνο κωδικού.  
- **Ευελιξία:** Μπορείτε προγραμματιστικά να αποφασίζετε ποιος κωδικός θα εφαρμοστεί βάσει επιχειρηματικής λογικής (π.χ., ρόλοι χρηστών).  
- **Απόδοση:** Το Aspose.Zip επεξεργάζεται τις καταχωρήσεις στη μνήμη, αποφεύγοντας την ανάγκη αποσυμπίεσης ολόκληρου του αρχείου πρώτα.

## Συνηθισμένα Προβλήματα & Λύσεις

| Πρόβλημα | Αιτία | Διόρθωση |
|----------|-------|----------|
| *Εξαίρεση “Invalid password”* | Λάθος κωδικός ή η καταχώρηση δεν είναι κρυπτογραφημένη. | Επαληθεύστε τη συμβολοσειρά κωδικού και βεβαιωθείτε ότι η καταχώρηση είναι προστατευμένη με κωδικό. |
| *Αρχείο δεν βρέθηκε* | Λάθος διαδρομή `dataDir`. | Χρησιμοποιήστε `Path.Combine(dataDir, "different_password.zip")` και ελέγξτε ξανά το φάκελο. |
| *Μεγάλα αρχεία προκαλούν υψηλή χρήση μνήμης* | Όλες οι καταχωρήσεις φορτώνονται στη μνήμη εξ' ορισμού. | Διαβάστε κάθε καταχώρηση ξεχωριστά ή χρησιμοποιήστε `Archive.ExtractToDirectory` με callback κωδικού (εάν υποστηρίζεται). |
