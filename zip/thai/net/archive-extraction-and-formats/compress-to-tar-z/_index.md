# เพิ่มไฟล์ลงใน tar และบีบอัดเป็น TarZ ด้วย Aspose.Zip สำหรับ .NET

## บทนำ

หากคุณต้องการ **add files to tar** แล้วบีบอัดไฟล์เก็บเป็นรูปแบบ TarZ, Aspose.Zip สำหรับ .NET ทำให้กระบวนการทั้งหมดเป็นเรื่องง่าย ในบทแนะนำนี้เราจะเดินผ่านทุกขั้นตอน—ตั้งแต่การตั้งค่าโปรเจกต์ของคุณ การสร้าง tar archive การเพิ่มไฟล์ และสุดท้ายการบันทึกไฟล์ .tar.z ที่บีบอัดแล้ว เมื่อเสร็จคุณจะได้โค้ดสั้นที่นำกลับมาใช้ใหม่ได้และสามารถใส่ลงในแอปพลิเคชัน .NET ใดก็ได้

## คำตอบสั้น
- **ไลบรารีที่จัดการการสร้าง tar คืออะไร?** Aspose.Zip for .NET  
- **จำนวนบรรทัดของโค้ดเท่าไหร่?** ประมาณ 15 บรรทัด (ไม่รวมคอมเมนต์)  
- **ต้องการไลเซนส์สำหรับการทดสอบหรือไม่?** มีรุ่นทดลองฟรี; จำเป็นต้องมีไลเซนส์สำหรับการใช้งานจริง  
- **เวอร์ชัน .NET ที่รองรับ?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+  
- **สามารถบีบอัดโฟลเดอร์ได้หรือไม่, ไม่ใช่แค่ไฟล์?** ได้ – คุณสามารถเพิ่มไดเรกทอรีทั้งหมดด้วยลูป  

## add files to tar คืออะไร?
การเพิ่มไฟล์ลงใน tar archive จะรวมไฟล์เหล่านั้นเป็นคอนเทนเนอร์เดียวที่ไม่ได้บีบอัด ซึ่งรักษาโครงสร้างไดเรกทอรีและเมตาดาต้าไฟล์ไว้ Tar เป็นฟอร์แมตคลาสสิกของ Unix และเป็นพื้นฐานของหลายกระบวนการบีบอัด รวมถึงรูปแบบ TarZ ที่ใช้ในคู่มือนี้

## ทำไมต้องเพิ่มไฟล์ลงใน tar ก่อนบีบอัดเป็น TarZ?
- **พกพาได้** – tar archive ทำงานข้ามแพลตฟอร์มโดยไม่ต้องกังวลเกี่ยวกับการจัดการไฟล์แต่ละไฟล์  
- **ความเร็ว** – การสร้าง tar container ทำได้เร็ว; การบีบอัด Z‑compression ต่อมาจะมุ่งเน้นเพียงการลดขนาด  
- **ความเข้ากันได้** – เครื่องมือเก่าหลายตัวคาดหวังไฟล์ `.tar` ก่อนทำการบีบอัดแบบ gzip ซึ่งตรงกับที่ `.tar.z` ให้  

## ข้อกำหนดเบื้องต้น

ก่อนที่เราจะลงลึกในโค้ด, โปรดตรวจสอบว่าคุณมี:

- **Aspose.Zip for .NET** ติดตั้งแล้ว ดาวน์โหลดจากเว็บไซต์อย่างเป็นทางการ [ที่นี่](https://releases.aspose.com/zip/net/).  
- โฟลเดอร์บนเครื่องของคุณที่มีไฟล์ที่ต้องการเก็บเป็น archive. แทนที่พาธตัวอย่างด้วยไดเรกทอรีจริงของคุณ

## นำเข้า Namespaces

เพิ่ม `using` statements ที่จำเป็นที่ด้านบนของไฟล์ C# ของคุณ:

```csharp
using System;
using Aspose.Zip.Tar;
```

## คู่มือขั้นตอนต่อขั้นตอน

### ขั้นตอนที่ 1: กำหนดไดเรกทอรีเอกสารของคุณ

```csharp
string dataDir = "Your Document Directory";
```

> **เคล็ดลับ:** ใช้ `Path.Combine` หากคุณต้องสร้างพาธแบบไดนามิก; จะช่วยหลีกเลี่ยงการขาดตัวคั่นพาธบน OS ต่าง ๆ

### ขั้นตอนที่ 2: สร้าง Tar Archive และเพิ่มไฟล์

#### 2.1: สร้างอินสแตนซ์ Tar archive

```csharp
using (TarArchive archive = new TarArchive())
{
    // Your code for creating the Tar archive goes here
}
```

#### 2.2: เพิ่มไฟล์ลงใน archive  

ภายในบล็อก `using`, เพิ่มไฟล์แต่ละไฟล์ที่ต้องการรวม:

```csharp
archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
```

คุณสามารถเรียก `CreateEntry` ซ้ำได้ตามจำนวนไฟล์ที่ต้องการ, หรือใช้ลูปผ่านไดเรกทอรีเพื่อเพิ่มไฟล์โดยอัตโนมัติ

#### 2.3: บันทึกไฟล์ TarZ ที่บีบอัด  

หลังจากเพิ่มรายการทั้งหมดแล้ว, บีบอัด tar archive ให้เป็นรูปแบบ `.tar.z`:

```csharp
archive.SaveZCompressed(dataDir + "archive.tar.z");
```

ไฟล์ `archive.tar.z` ที่ได้จะอยู่ในโฟลเดอร์เดียวกับที่คุณระบุใน `dataDir`

## ปัญหาและวิธีแก้ไขทั่วไป

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|-----|
| **ไฟล์ไม่พบ** | พาธผิดหรือขาดส่วนขยายไฟล์ | ตรวจสอบว่า `dataDir` ลงท้ายด้วยตัวคั่นพาธและชื่อไฟล์ถูกต้อง |
| **การเข้าถึงถูกปฏิเสธ** | สิทธิ์ไม่เพียงพอในโฟลเดอร์เป้าหมาย | รันแอปพลิเคชันด้วยสิทธิ์ที่เหมาะสมหรือเลือกโฟลเดอร์ที่สามารถเขียนได้ |
| **ไฟล์ที่บีบอัดมีขนาดใหญ่กว่าที่คาด** | ไฟล์ต้นฉบับถูกบีบอัดแล้ว (เช่น รูปภาพ, วิดีโอ) | TarZ ทำงานดีที่สุดกับไฟล์ข้อความหรือไฟล์บันทึก; พิจารณาให้ไฟล์ที่บีบอัดแล้วอยู่ในสภาพเดิม |
