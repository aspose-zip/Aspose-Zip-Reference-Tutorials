# ดึงข้อมูลจากไฟล์อาร์ไคฟ์ที่มีรหัสผ่านและบีบอัดเป็น TarBz2 (.NET)

ในคู่มือนี้คุณจะได้เรียนรู้ **วิธีดึงข้อมูลจากไฟล์อาร์ไคฟ์ที่มีรหัสผ่าน** และยังสามารถ **บีบอัดไฟล์เป็น TarBz2** ด้วย Aspose.Zip for .NET ไม่ว่าคุณจะกำลังสร้างบริการสำรองข้อมูล, ไคลเอนต์คลาวด์สตอเรจ, หรือไพป์ไลน์การประมวลผลข้อมูล เทคนิคเหล่านี้จะช่วยลดขนาดการจัดเก็บ, เร่งความเร็วการถ่ายโอน, และปกป้องข้อมูลที่สำคัญ

## คำตอบสั้น
- **TarBz2 คืออะไร?** เป็นไฟล์อาร์ไคฟ์ที่บีบอัดโดยรวมการบรรจุ TAR กับการบีบอัด BZIP2 เพื่อให้ได้อัตราการบีบอัดสูง  
- **ทำไมต้องเลือก Aspose.Zip for .NET?** ให้ API เดียวที่เป็น fluent สำหรับหลายรูปแบบอาร์ไคฟ์โดยไม่ต้องพึ่งพาไลบรารีเนทีฟ  
- **ฉันสามารถสร้างไฟล์ TarGz ได้หรือไม่?** ได้ – Aspose.Zip รองรับ TarGz, TarLz, TarXz, TarZ และอื่น ๆ  
- **จะดึงไฟล์อาร์ไคฟ์ที่มีรหัสผ่านอย่างไร?** ตั้งค่า property `Password` บนแต่ละ `ArchiveEntry` ก่อนทำการดึงข้อมูล  
- **ต้องมีลิขสิทธิ์สำหรับการใช้งานในโปรดักชันหรือไม่?** จำเป็นต้องมีลิขสิทธิ์เชิงพาณิชย์สำหรับการใช้งานในโปรดักชัน; มีรุ่นทดลองฟรีสำหรับการประเมิน

## วิธีดึงไฟล์อาร์ไคฟ์ที่มีรหัสผ่านโดยใช้ Aspose.Zip for .NET
การดึงไฟล์อาร์ไคฟ์ที่ **มีการป้องกันด้วยรหัสผ่าน** ทำได้อย่างง่ายดาย เพียงเปิดไฟล์อาร์ไคฟ์, ค้นหา entry ที่ต้องการ, กำหนดรหัสผ่าน, แล้วเรียก `Extract` วิธีนี้ใช้ได้กับ TarBz2, TarGz และรูปแบบที่รองรับอื่น ๆ

## “บีบอัดไฟล์เป็น TarBz2” หมายความว่าอย่างไร?
การบีบอัดไฟล์เป็น TarBz2 หมายถึงการรวมไฟล์และโฟลเดอร์หลาย ๆ ไฟล์เข้าด้วยกันในคอนเทนเนอร์ **TAR** แล้วจึงใช้การบีบอัด **BZIP2** ผลลัพธ์คือไฟล์ `.tar.bz2` ที่ง่ายต่อการขนส่งและบีบอัดสูง

## ทำไมต้องใช้ Aspose.Zip for .NET เพื่อจัดการรูปแบบเหล่านี้?
- **API แบบรวมศูนย์** – ไลบรารีเดียว รองรับหลายรูปแบบ (TarBz2, TarGz, TarLz, TarXz, TarZ)  
- **ไม่มีการพึ่งพาไลบรารีเนทีฟ** – ทำงานได้บน Windows, Linux, macOS ตั้งแต่แรก  
- **รองรับรหัสผ่าน** – ปกป้องและดึงไฟล์อาร์ไคฟ์ด้วยรหัสผ่านต่อ entry ได้อย่างปลอดภัย  
- **เน้นประสิทธิภาพ** – การประมวลผลแบบ stream ลดการใช้หน่วยความจำ

## ข้อกำหนดเบื้องต้น
- .NET 6.0 หรือใหม่กว่า (หรือ .NET Core 3.1+ / .NET Framework 4.5+)  
- ติดตั้งแพคเกจ NuGet Aspose.Zip for .NET (`Install-Package Aspose.Zip`)  
- มีความรู้พื้นฐานเกี่ยวกับ C# และการทำ I/O ไฟล์

## คู่มือแบบขั้นตอน

### ขั้นตอนที่ 1: เลือกรูปแบบอาร์ไคฟ์ที่ต้องการ
ตัดสินใจว่าจะใช้ **TarBz2**, **TarGz**, **TarLz**, **TarXz**, หรือ **TarZ** ตามอัตราการบีบอัดและความเข้ากันได้ที่ต้องการ  
- **TarBz2** – บีบอัดสูงสุด, ประมวลผลช้ากว่า  
- **TarGz** – สมดุลระหว่างความเร็วและขนาด (ครอบคลุมคีย์เวิร์ดรอง *compress files to targz*)  
- **TarZ** – รูปแบบเก่าสำหรับเครื่องมือ Unix รุ่นเก่า

### ขั้นตอนที่ 2: สร้างอินสแตนซ์ `Archive` ใหม่
สร้างอ็อบเจกต์ `Archive` และระบุพาธไฟล์ผลลัพธ์ อ็อบเจกต์นี้จะจัดการกระบวนการบรรจุและบีบอัด

### ขั้นตอนที่ 3: เพิ่มไฟล์และโฟลเดอร์
ใช้เมธอด `AddAll` หรือ `AddFile` เพื่อใส่ไฟล์ที่ต้องการบีบอัด รักษาโครงสร้างไดเรกทอรีโดยการเพิ่มโฟลเดอร์ฐาน

### ขั้นตอนที่ 4: ตั้งค่าชนิดการบีบอัดที่ต้องการ
ระบุอัลกอริทึมบีบอัด (`CompressionType.BZip2`, `CompressionType.GZip` ฯลฯ) เมื่อทำการบันทึกอาร์ไคฟ์ ที่นี่คุณจะทำ **compress files to tarbz2** หรือรูปแบบอื่นจริง ๆ

### ขั้นตอนที่ 5: บันทึกอาร์ไคฟ์
เรียก `Save` พร้อมระบุ enum ของรูปแบบที่ต้องการ (`ArchiveFormat.TarBz2`, `ArchiveFormat.TarGz` ฯลฯ) ไลบรารีจะเขียนคอนเทนเนอร์ TAR แล้วทำการบีบอัดตามที่กำหนดในขั้นตอนเดียว

### ขั้นตอนที่ 6: ดึงไฟล์อาร์ไคฟ์ที่มีรหัสผ่าน
หากต้อง **ดึง entry ของอาร์ไคฟ์ที่มีรหัสผ่านต่างกัน** (คีย์เวิร์ดรอง *how to extract password protected archive*), เปิดอาร์ไคฟ์, ค้นหา entry แต่ละอัน, กำหนดรหัสผ่าน, แล้วทำการดึงข้อมูล

### ขั้นตอนที่ 7: ตรวจสอบผลลัพธ์
หลังจากดึงไฟล์แล้ว ให้เปรียบเทียบขนาดไฟล์และ checksum เพื่อยืนยันว่าอาร์ไคฟ์ถูกสร้างและแตกออกอย่างถูกต้อง

## กรณีการใช้งานทั่วไป
- **ยูทิลิตี้สำรองข้อมูล** – เก็บสำรองประจำวันเป็น `.tar.bz2` เพื่อลดค่าใช้จ่ายการจัดเก็บ  
- **การแลกเปลี่ยนข้อมูลข้ามแพลตฟอร์ม** – รูปแบบที่อิง TAR เป็นที่เข้าใจทั่วโลกบน Linux, macOS, และ Windows  
- **การแจกจ่ายอย่างปลอดภัย** – ปกป้อง entry แต่ละอันด้วยรหัสผ่านสำหรับสภาพแวดล้อมที่ต้องปฏิบัติตามข้อกำหนด

## การแก้ไขปัญหาและเคล็ดลับ
- **อาร์ไคฟ์ขนาดใหญ่** – ใช้ API แบบสตรีม (`Archive.CreateEntryFromFile`) เพื่อหลีกเลี่ยงการโหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำ  
- **รหัสผ่านไม่ตรงกัน** – ตรวจสอบให้แน่ใจว่ารหัสผ่านที่ตั้งบนแต่ละ `ArchiveEntry` ตรงกับรหัสที่ใช้ขณะดึง; มิฉะนั้นจะเกิด `InvalidPasswordException`  
- **ระดับการบีบอัดที่ไม่รองรับ** – BZIP2 ไม่รองรับการกำหนดระดับบีบอัดเอง; หากต้องการควบคุมละเอียดกว่า ให้พิจารณา TarLz หรือ TarXz  
- **เคล็ดลับประสิทธิภาพ** – เมื่อสร้าง **create tarbz2 archive**, เปิดใช้งานการบัฟเฟอร์บนสตรีมพื้นฐานเพื่อเพิ่มความเร็วในการเขียน

## คำถามที่พบบ่อย

**Q: จะสร้างไฟล์ TarGz อย่างไร?**  
A: ตั้งค่า `CompressionType.GZip` และ `ArchiveFormat.TarGz` ขณะเรียก `Save`

**Q: สามารถดึงไฟล์อาร์ไคฟ์ที่ป้องกันด้วยรหัสผ่านโดยไม่รู้รหัสได้หรือไม่?**  
A: ไม่ได้; ต้องระบุรหัสผ่านที่ถูกต้องสำหรับแต่ละ entry มิฉะนั้นการดึงจะล้มเหลว

**Q: Aspose.Zip รองรับการดึงไฟล์อาร์ไคฟ์ที่มีรหัสผ่านต่างกันต่อ entry หรือไม่?**  
A: รองรับ สามารถกำหนดรหัสผ่านให้กับ `ArchiveEntry` แต่ละอันก่อนดึงได้

**Q: รูปแบบใดให้การบีบอัดที่ดีที่สุด?**  
A: TarBz2 มักให้สัดส่วนบีบอัดสูงสุด ตามด้วย TarLz และ TarXz. TarGz ให้สมดุลที่ดีระหว่างความเร็วและขนาด

**Q: มีขีดจำกัดจำนวนไฟล์ที่สามารถเพิ่มลงในอาร์ไคฟ์ TAR หรือไม่?**  
A: โดยหลักการไม่มีข้อจำกัด, แต่ไฟล์อาร์ไคฟ์ขนาดใหญ่มากอาจต้องแบ่งเป็นหลายส่วนเพื่อความสะดวกในการจัดการ
