# เพิ่มไฟล์ลงใน tar และสร้างไฟล์ tarxz ด้วย Aspose.Zip

## การแนะนำ

**เพิ่มไฟล์ลงใน tar** แล้ว **สร้างไฟล์เก็บถาวร tarxz .net** Aspose.Zip สำหรับ .NET ที่เป็นความจริงที่เชื่อถือได้ เทคโนโลยีการบรรจุ log, ประสิทธิภาพหรือทรัพยากรอื่นๆ ที่สามารถนำมาใช้สำหรับส่งต่อสาธารณะ ของ TarXz จะให้ส่วนแบ่งของของเหลวสูงพร้อมคงโครงสร้าง tar ที่ใช้งานในครั้งแรกนี้เราจะดำเนินการขั้นตอนพร้อมตัวอย่างโค้ดเพื่อให้คุณทราบอีกครั้งก่อนเปิดตัว tarxz .NET .NET ของคุณอย่างมั่นใจ

## คำตอบด่วน
- **คลาสหลักคืออะไร** `TarArchive` จาก `Aspose.Zip.Tar`
- **วิธีบีบอัด tarxz?** ใช้ `SaveXzCompressed` หลังจากเพิ่มรายการ
- **รองรับเวอร์ชัน .NET?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6
- **จำเป็นต้องมีใบอนุญาตใช่ไหม** ใช่ ใบอนุญาต Aspose.Zip ที่ถูกต้องสำหรับการผลิต
- **ระยะเวลาดำเนินการโดยทั่วไป?** ~5-10 นาที

## ไฟล์เก็บถาวร TarXz คืออะไร

**TarXz archive** สามารถใช้เป็นพื้นที่แบบ Unix `tar` กับห้องน้ำ XZ ส่วน tar จะรวบรวมไฟล์หลายไฟล์เป็นสตรีมเดียวเท่านั้น XZ ช่วยให้พื้นที่เก็บข้อมูลเพื่อความอยู่รอดนี้นิยมในการรองรับสโค้ด, เก็บข้อมูล, และชุดข้อมูลขนาดใหญ่เพราะคงโครงสร้างที่ต้องใช้ไฟล์เป็นหลักมากกว่าการใช้ tar ธรรมดาหรือ zip

## ทำไมต้องสร้างไฟล์เก็บถาวร tarxz .net ด้วย Aspose.Zip

- **อัตราส่วนการบีบอัดสูง** – XZ มักจะได้น้อยกว่า gzip ธ 30‑50%
- **ความเข้ากันได้ข้ามแพลตฟอร์ม** – รองรับ TarXz สามารถเปิดได้บน Linux, macOS, และ Windows
- **Simple API** – Aspose.Zip การจัดการรายละเอียดระดับล่าง คุณต้องโฟกัสที่โลจิกเนื้อหา
- **ไม่มีเครื่องมือภายนอก** – สามารถทำงานภายใน .NET ของคุณตามความต้องการหรือไปป์ไลน์ CI

## ข้อกำหนดเบื้องต้น

และเริ่มต้นการดำเนินการให้คุณทราบ:

- **Aspose.Zip สำหรับ .NET** ดาวน์โหลดแล้ว (ดาวน์โหลดจาก **[เอกสารประกอบ Aspose.Zip](https://reference.aspose.com/zip/net/)**)
- ไดรฟ์ที่มีไฟล์ที่คุณต้องการในที่เก็บข้อมูลจะอ้างอิงด้วยเรื่องของ `dataDir`
- แล้ว Aspose.Zip ที่ถูกต้อง (ไม่บังคับสำหรับส่วนใหญ่แต่จำเป็นจริงๆ)

## นำเข้าเนมสเปซ

เริ่มการนำเข้าเนมสเปซที่ฟังก์ชันการทำงานของกฎหมาย TarXz

```csharp
using System;
using Aspose.Zip.Tar;
```

## วิธีเพิ่มไฟล์ลงในไฟล์ tar โดยใช้ Aspose.Zip

ต่อไปนี้เป็นคำแนะนำแบบขั้นตอนที่แสดงให้เห็นอย่างชัดเจนว่า **add files to tar** ก่อนบีบอัดอย่างไร

### ขั้นตอนที่ 1: สร้างไฟล์บีบอัดแบบ TarArchive

สร้างอินสแตนซ์ `TarArchive` ใหม่ที่จะเก็บไฟล์ที่คุณต้องการบีบอัด

```csharp
using (TarArchive archive = new TarArchive())
{
```

> **Pro tip:** คำสั่ง `using` ทำให้แน่ใจว่า archive ถูกทำลายอย่างถูกต้อง ปล่อยทรัพยากรที่ไม่ได้จัดการออกไป

### ขั้นตอนที่ 2: เพิ่มไฟล์ลงในไฟล์บีบอัด

เพิ่มไฟล์แต่ละไฟล์ที่ต้องการรวม ในตัวอย่างนี้เราจะเพิ่มไฟล์ข้อความสองไฟล์ แต่คุณสามารถเพิ่มรายการได้ตามต้องการ

```csharp
    archive.CreateEntry("alice29.txt", dataDir + "alice29.txt");
    archive.CreateEntry("lcet10.txt", dataDir + "lcet10.txt");
```

> **Why this matters:** การเพิ่มรายการก่อนบีบอัดทำให้ Aspose.Zip สร้างคอนเทนเนอร์ tar ก่อน แล้วค่อยใช้การบีบอัด XZ ในขั้นตอนเดียว

### ขั้นตอนที่ 3: บันทึกไฟล์บีบอัดด้วยการบีบอัดแบบ XZ

สุดท้าย ให้เขียน tar archive ลง **disk** ด้วยการบีบอัด XZ ไฟล์ที่ได้จะมีนามสกุล `.tar.xz`

```csharp
    archive.SaveXzCompressed(dataDir + "archive.tar.xz");
}
```

> **Result:** ตอนนี้คุณมีไฟล์ `archive.tar.xz` **ที่บีบอัดเต็มรูปแบบ** ซึ่งสามารถถ่ายโอน, เก็บรักษา, หรือแตกไฟล์บน **แพลตฟอร์ม** ใดก็ได้ที่รองรับ TarXz

## วิธีบีบอัดไฟล์ tarxz ด้วย Aspose.Zip

อีกครั้งคือ **วิธีบีบอัด tarxz**: คุณเพิ่มไฟล์ลงในภาชนะ tar (`เพิ่มไฟล์ลงใน tar`) จากนั้นเรียก `SaveXzCompressed` วิธีเรียกอย่างเดียวนี้ทำให้ไม่จำเป็นต้องบรรทัดคำสั่งภายนอกและทุกอย่างอยู่ในโค้ด .NET ของคุณ

## ปัญหาและแนวทางแก้ไขทั่วไป

| ปัญหา | สาเหตุ | แก้ไข |
|-------|-------|-----|
| **ข้อยกเว้น “ไม่พบไฟล์”** | เส้นทาง `dataDir` ไม่ถูกต้อง | ตรวจสอบเส้นทางไดเรกทอรีที่ลงท้ายด้วยแบ็กสแลช (`\`) หรือใช้ `Path.Combine` |
| **การใช้หน่วยความจำขนาดใหญ่** | ไฟล์ขนาดใหญ่มากถูกบีบอัดในหน่วยความจำ | ใช้ `TarArchive` ในโหมดสตรีมมิ่ง (โอเวอร์โหลด `SaveXzCompressed` ที่ยอมรับ `Stream`) |
| **ไม่มีใบอนุญาต** | ไฟล์ลิขสิทธิ์หายไป | โหลดใบอนุญาตเมื่อเริ่มต้นแอปพลิเคชัน: `new Aspose.Zip.License().SetLicense("Aspose.Zip.lic");` |
