# วิธีการแยกไฟล์ Zip ด้วยรหัสผ่านโดยใช้ Aspose.Zip สำหรับ .NET

ในแอปพลิเคชัน .NET สมัยใหม่ การปกป้องข้อมูลที่สำคัญภายในไฟล์ ZIP เป็นความต้องการที่พบทั่วไป บทแนะนำนี้แสดง **วิธีการแยกไฟล์ zip ด้วยรหัสผ่าน** เมื่อแต่ละรายการใช้รหัสผ่านที่แตกต่างกัน ให้คุณควบคุมความปลอดภัยได้อย่างละเอียดขณะยังคงกระบวนการแยกไฟล์ให้เรียบง่าย โดยทำตามตัวอย่าง Aspose.Zip นี้คุณจะเห็นวิธีทำการแยกไฟล์ zip ที่มีการป้องกันด้วยรหัสผ่านสำหรับแต่ละรายการอย่างชัดเจน

## คำตอบอย่างรวดเร็ว
- **ควรใช้ไลบรารีอะไร?** Aspose.Zip for .NET.  
- **ฉันสามารถแยกไฟล์ที่มีรหัสผ่านต่างกันได้หรือไม่?** ใช่—แต่ละรายการสามารถเปิดด้วยรหัสผ่านของมันเองได้.  
- **ฉันต้องการไลเซนส์สำหรับการใช้งานจริงหรือไม่?** จำเป็นต้องมีไลเซนส์เชิงพาณิชย์; มีการทดลองใช้ฟรี.  
- **แพลตฟอร์มที่รองรับ?** .NET Framework, .NET Core, .NET 5/6+.  
- **เวลาในการดำเนินการโดยทั่วไป?** ประมาณ 10 นาทีสำหรับสถานการณ์พื้นฐาน.

## “วิธีการแยกไฟล์ zip” คืออะไร?
การแยกไฟล์ ZIP หมายถึงการอ่านคอนเทนเนอร์ที่บีบอัดและเขียนเนื้อหาของมันลงในระบบไฟล์ เมื่อไฟล์ถูกป้องกันด้วยรหัสผ่าน คุณต้องระบุรหัสผ่านที่ถูกต้องสำหรับแต่ละรายการก่อนที่ข้อมูลจะถูกถอดรหัส

## ทำไมต้องใช้ Aspose.Zip สำหรับการแยกไฟล์ที่มีรหัสผ่าน?
- **ความปลอดภัยระดับละเอียด:** แต่ละไฟล์สามารถมีรหัสผ่านของตนเองได้ ลดความเสี่ยงหากรหัสผ่านเดียวถูกเปิดเผย.  
- **ความยืดหยุ่น:** คุณสามารถกำหนดโปรแกรมว่ารหัสผ่านใดจะใช้ตามตรรกะธุรกิจ (เช่น บทบาทผู้ใช้).  
- **ประสิทธิภาพ:** Aspose.Zip ประมวลผลรายการในหน่วยความจำ, ไม่จำเป็นต้องแยกไฟล์ทั้งหมดก่อน.  
- **การสนับสนุนข้ามแพลตฟอร์ม:** ทำงานบน Windows, Linux, และ macOS กับ .NET 5/6+.

## นำเข้า Namespaces

แรกเริ่มให้นำเข้า namespaces ที่จำเป็นสำหรับทำงานกับ archive:

```csharp
using Aspose.Zip;
using System.IO;
```

สอง `using` นี้ให้คุณเข้าถึงคลาส `Archive` และยูทิลิตี้ I/O มาตรฐาน

## ขั้นตอนที่ 1: กำหนดไดเรกทอรีทำงาน

กำหนดโฟลเดอร์ที่ไฟล์ ZIP อยู่และที่ไฟล์ที่แยกออกจะถูกเขียนไป:

```csharp
string dataDir = "Your Document Directory";
```

> **เคล็ดลับ:** ใช้ `Path.Combine` สำหรับการสร้างพาธแบบข้ามแพลตฟอร์ม หากคุณต้องการสนับสนุน Linux/macOS.

## วิธีการแยกไฟล์ Zip ด้วยรหัสผ่านโดยใช้ Aspose.Zip

ด้านล่างเราจะอธิบายขั้นตอนที่แม่นยำเพื่อเปิด archive และแยกรายการแต่ละรายการโดยใช้รหัสผ่านของมันเอง ส่วนนี้แสดง **การแยกไฟล์ zip ด้วยรหัสผ่าน** สำหรับทุกรายการ ซึ่งเป็นแกนของกระบวนการ “วิธีการแยกไฟล์ zip”

### ขั้นตอนที่ 2: แยกรายการใน Archive ด้วยรหัสผ่านที่ต่างกัน

#### ขั้นตอน 2.1: เปิดไฟล์ Zip

```csharp
using (FileStream zipFile = File.Open(dataDir + "\\different_password.zip", FileMode.Open))
{
    using (Archive archive = new Archive(zipFile))
    {
        // Continue to the next steps...
    }
}
```

อ็อบเจ็กต์ `Archive` แทนคอนเทนเนอร์ ZIP การเก็บ `FileStream` และ `Archive` ไว้ในบล็อก `using` ทำให้แน่ใจว่าทรัพยากรทั้งหมดจะถูกปล่อยอย่างทันท่วงที

#### ขั้นตอน 2.2: แยกรายการแรก (รหัสผ่าน = “first_pass”)

```csharp
archive.Entries[0].Extract(dataDir + "alice29_extracted_pass_out.txt", "first_pass");
```

ที่นี่เรา **แยกรายการ zip หลายรายการ** โดยอ้างอิงผ่านคอลเลกชัน `Entries` รายการแรกถูกถอดรหัสด้วยรหัสผ่าน `"first_pass"`

#### ขั้นตอน 2.3: แยกรายการที่สอง (รหัสผ่าน = “second_pass”)

```csharp
archive.Entries[1].Extract(dataDir + "asyoulik_extracted_pass_out.txt", "second_pass");
```

รายการที่สองใช้รหัสผ่านที่แตกต่างกัน แสดงการจัดการ **extract zip entry password** สำหรับไฟล์แต่ละไฟล์

#### ขั้นตอน 2.4: (ทางเลือก) วนลูปผ่านทุกรายการ

หากคุณต้องการ **แยกรายการ zip หลายรายการ** โดยไม่ต้องกำหนดดัชนีแบบคงที่ คุณสามารถวนลูปผ่าน `archive.Entries` และให้รหัสผ่านที่เหมาะสมกับแต่ละรายการตามตรรกะการค้นหาของคุณ วิธีนี้ขยายได้ดีเมื่อจัดการกับ archive ขนาดใหญ่

## ปัญหาที่พบบ่อยและวิธีแก้

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|-----|
| *“Invalid password” exception* | รหัสผ่านที่ให้ไม่ถูกต้องหรือรายการไม่ได้ถูกเข้ารหัสจริง. | ตรวจสอบสตริงรหัสผ่านและยืนยันว่ารายการนั้นถูกป้องกันด้วยรหัสผ่าน. |
| *File not found* | พาธ `dataDir` ไม่ถูกต้อง. | ใช้ `Path.Combine(dataDir, "different_password.zip")` และตรวจสอบโฟลเดอร์อีกครั้ง. |
| *Large archives cause high memory usage* | รายการทั้งหมดถูกโหลดเข้าสู่หน่วยความจำโดยค่าเริ่มต้น. | สตรีมแต่ละรายการแยกกันหรือใช้ `Archive.ExtractToDirectory` พร้อม callback รหัสผ่าน (หากสนับสนุน). |
