# วิธีเข้ารหัสไฟล์ 7z ด้วย Aspose.Zip สำหรับ .NET

## คำแนะนำ

ในบทเรียนนี้คุณจะได้เรียนรู้ **วิธีเข้ารหัสไฟล์ 7z** ด้วยไลบรารี Aspose.Zip สำหรับ .NET ไม่ว่าคุณจะต้องการปกป้องข้อมูลที่สำคัญ ปฏิบัติตามนโยบายความปลอดภัย หรือเพียงแค่บีบอัดไฟล์อย่างมีประสิทธิภาพ คู่มือนี้จะพาคุณผ่านทุกขั้นตอน ตั้งแต่การตั้งค่าโปรเจกต์จนถึงการยืนยันว่าอาร์ไคฟ์เวอร์ถูกสร้างสำเร็จแล้ว ไปดูกันว่าการ **เพิ่มไฟล์ลงใน 7z** ด้วยการเข้ารหัส AES ทำได้ง่ายแค่ไหน

## คำตอบสั้น ๆ
- **“สร้าง 7z ที่เข้ารหัส” หมายถึงอะไร?** หมายถึงการสร้างอาร์ไคฟ์เวอร์ 7‑zip ที่ได้รับการปกป้องด้วยการเข้ารหัส AES  
- **ใช้ไลบรารีอะไร?** Aspose.Zip สำหรับ .NET  
- **ต้องมีลิขสิทธิ์หรือไม่?** ใบอนุญาตชั่วคราวเพียงพอสำหรับการทดสอบ; ต้องมีลิขสิทธิ์เต็มสำหรับการใช้งานจริง  
- **สามารถเพิ่มหลายไฟล์ได้หรือไม่?** ได้ — เรียก `CreateEntry` ซ้ำเพื่อ **เพิ่มหลายไฟล์ 7z**  
- **รองรับการเข้ารหัส AES หรือไม่?** รองรับ, Aspose.Zip รองรับ **วิธีตั้งค่า AES**‑256 สำหรับอาร์ไคฟ์เวอร์ 7z  

## 7z ที่เข้ารหัสคืออะไร?
อาร์ไคฟ์เวอร์ 7z เป็นรูปแบบคอนเทนเนอร์ที่บีบอัดสูง เมื่อคุณ **สร้าง 7z ที่เข้ารหัส** เนื้อหาจะถูกสับด้วยการเข้ารหัส AES ทำให้ไม่สามารถอ่านได้หากไม่มีรหัสผ่านที่ถูกต้อง เหมาะสำหรับการส่งหรือเก็บไฟล์ที่เป็นความลับอย่างปลอดภัย

## ทำไมต้องใช้ Aspose.Zip สำหรับไฟล์ 7z ที่เข้ารหัส?
- **บูรณาการเต็มรูปแบบกับ .NET** – ทำงานกับ .NET Framework, .NET Core, และ .NET 5/6  
- **รองรับ AES‑256 ในตัว** – ไม่ต้องใช้เครื่องมือภายนอก; คุณสามารถเรียนรู้ **วิธีตั้งค่า AES** ได้อย่างง่ายดาย  
- **API ง่าย** – เรียกใช้บรรทัดเดียวเพื่อ **เพิ่มไฟล์ลงใน 7z** และบันทึกอาร์ไคฟ์เวอร์  
- **ข้ามแพลตฟอร์ม** – ทำงานบน Windows, Linux, และ macOS  

## ข้อกำหนดเบื้องต้น

ก่อนเริ่มทำตามขั้นตอนต่อไปนี้ให้แน่ใจว่าคุณมี:

- **Aspose.Zip สำหรับ .NET Library** – ดาวน์โหลดได้จาก [ที่นี่](https://releases.aspose.com/zip/net/)  
- **โฟลเดอร์ที่สามารถเขียนได้** บนเครื่องของคุณเพื่อบันทึกอาร์ไคฟ์เวอร์  
- **ไฟล์ต้นฉบับ** (เช่น `file.dat`) ที่ต้องการบีบอัดและเข้ารหัส  

## นำเข้า Namespaces

เพิ่ม namespace ที่จำเป็นที่ส่วนบนของไฟล์ C# ของคุณ:

```csharp
using Aspose.Zip.SevenZip;
```

## คู่มือแบบขั้นตอน

### ขั้นตอนที่ 1: กำหนดไดเรกทอรีทำงาน

ตั้งค่าพาธไปยังโฟลเดอร์ที่มีไฟล์ต้นฉบับที่ต้องการบีบอัด

```csharp
string dataDir = "Your Document Directory";
```

แทนที่ `"Your Document Directory"` ด้วยพาธจริงบนเครื่องของคุณ

### ขั้นตอนที่ 2: สร้างรายการ 7z ที่เข้ารหัส

ส่วนสำคัญของบทเรียน – เราเปิด stream ใหม่, สร้าง `SevenZipArchive`, เพิ่ม entry, และบันทึกอาร์ไคฟ์เวอร์ ตัวอย่างนี้เพิ่มไฟล์เดียว (`file.dat`) เป็น `data.bin` ภายในอาร์ไคฟ์เวอร์

```csharp
//ExStart: CreateSevenZipEntry
using (FileStream sevenZipFile = File.Open("archive.7z", FileMode.Create))
{
    using (var archive = new SevenZipArchive())
    {
        archive.CreateEntry("data.bin", "file.dat");
        archive.Save(sevenZipFile);
    }
}
//ExEnd: CreateSevenZipEntry
```

> **เคล็ดลับ:** เพื่อเปิดใช้งานการเข้ารหัส AES ให้ตั้งค่า `Encryption` ของ `SevenZipArchive` ก่อนเรียก `Save` (คุณสมบัตินี้ไม่ได้แสดงในตัวอย่างเพื่อความกระชับ)

### ขั้นตอนที่ 3: ยืนยันความสำเร็จ

พิมพ์ข้อความแจ้งเพื่อให้คุณทราบว่าการดำเนินการเสร็จสมบูรณ์โดยไม่มีข้อผิดพลาด

```csharp
Console.WriteLine("Successfully Created a Seven Zip File with AES Encryption Settings");
```

### ขั้นตอนที่ 4: ตรวจสอบอาร์ไคฟ์เวอร์ (ทางเลือก)

หลังจากโปรแกรมทำงานเสร็จ ให้ไปที่โฟลเดอร์ที่มี `archive.7z` แล้วลองเปิดด้วยโปรแกรม 7‑zip คุณควรจะถูกขอรหัสผ่านหากได้ตั้งค่าการเข้ารหัสในขั้นตอน 2 ขั้นตอนนี้ยังช่วยให้คุณ **ตรวจสอบรหัสผ่าน 7z** ได้อีกด้วย

## ปัญหาที่พบบ่อยและวิธีแก้

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|-------|--------|
| **ไม่พบไฟล์** | พาธ `dataDir` หรือชื่อไฟล์ต้นฉบับไม่ถูกต้อง | ตรวจสอบพาธอีกครั้งและยืนยันว่า `file.dat` มีอยู่ |
| **การเข้าถึงถูกปฏิเสธ** | สิทธิ์การเขียนไม่เพียงพอ | รันแอปพลิเคชันด้วยสิทธิ์ผู้ดูแลหรือเลือกโฟลเดอร์ที่เขียนได้ |
| **ไม่ได้เข้ารหัส** | ไม่ได้ตั้งค่าการเข้ารหัสบนอาร์ไคฟ์เวอร์ | ตั้งค่า `archive.Encryption = EncryptionAlgorithm.Aes256;` ก่อน `Save` |

## คำถามที่พบบ่อย

**ถาม: สามารถเพิ่มไฟล์มากกว่าหนึ่งไฟล์ในอาร์ไคฟ์เวอร์ 7z เดียวได้หรือไม่?**  
ตอบ: แน่นอน เรียก `archive.CreateEntry` สำหรับแต่ละไฟล์ที่ต้องการ **เพิ่มไฟล์ลงใน 7z** หรือ **เพิ่มหลายไฟล์ 7z**  

**ถาม: จะกำหนดรหัสผ่านสำหรับการเข้ารหัส AES อย่างไร?**  
ตอบ: ตั้งค่า `Password` ของ `SevenZipArchive` ก่อนบันทึก เช่น `archive.Password = "YourStrongPassword";` ซึ่งจะทำให้คุณสามารถ **ตรวจสอบรหัสผ่าน 7z** ได้เมื่อทำการแตกไฟล์  

**ถาม: Aspose.Zip รองรับรูปแบบอาร์ไคฟ์เวอร์อื่นหรือไม่?**  
ตอบ: Aspose.Zip มุ่งเน้นที่รูปแบบ ZIP และ 7z หากต้องการรูปแบบอื่นให้พิจารณาไลบรารีเฉพาะ  

**ถาม: ต้องมีลิขสิทธิ์สำหรับการใช้งานในผลิตภัณฑ์หรือไม่?**
