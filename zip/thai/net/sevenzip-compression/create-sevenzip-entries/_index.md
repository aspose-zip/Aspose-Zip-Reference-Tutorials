## c# สร้างอาร์ไคฟ์ 7z – คู่มือขั้นตอนโดยละเอียด

### ขั้นตอนที่ 1: ตั้งค่าเส้นทางไดเรกทอรีทรัพยากร

ก่อนสร้าง SevenZip entries ให้ตั้งค่าเส้นทางไปยังไดเรกทอรีทรัพยากรของคุณ แทนที่ `"Your Document Directory"` ในตัวแปร `dataDir` ด้วยเส้นทางที่แท้จริง

```csharp
string dataDir = "Your Document Directory";
```

> **เคล็ดลับ:** การใช้เส้นทางแบบเต็มช่วยลดความสับสนเมื่อแอปพลิเคชันทำงานจากไดเรกทอรีทำงานที่ต่างออกไป.

### ขั้นตอนที่ 2: สร้าง SevenZip Entries (บีบอัดไดเรกทอรีเป็น 7z)

ต่อไปเราจะลงลึกในกระบวนการหลัก – การสร้าง SevenZip entries ขั้นตอนนี้ **บีบอัดไดเรกทอรีเป็นรูปแบบ 7z**.

```csharp
//ExStart: CreateSevenZipEntries
using (SevenZipArchive archive = new SevenZipArchive())
{
    archive.CreateEntries(dataDir);
    archive.Save("SevenZip.7z");
}
//ExEnd: CreateSevenZipEntries
```

โค้ดด้านบนทำการเริ่มต้น `SevenZipArchive` เพิ่มไฟล์ทุกไฟล์จากโฟลเดอร์ที่ระบุ และเขียนอาร์ไคฟ์บีบอัดไปยัง **SevenZip.7z**.

### ขั้นตอนที่ 3: แสดงข้อความสำเร็จ

เพื่อให้แน่ใจว่าทุกอย่างทำงานเรียบร้อย ให้แสดงข้อความสำเร็จ:

```csharp
Console.WriteLine("Successfully Created a Seven Zip File");
```

ตอนนี้คุณมี **อาร์ไคฟ์ 7z** ที่พร้อมแชร์ สามารถโอนย้าย เก็บรักษา หรือประมวลผลต่อได้

## ปัญหาทั่วไปและวิธีแก้

| ปัญหา | วิธีแก้ |
|-------|----------|
| **อาร์ไคฟ์ว่าง** | ตรวจสอบว่า `dataDir` ชี้ไปยังโฟลเดอร์ที่มีไฟล์. ใช้ `Directory.Exists` เพื่อยืนยัน. |
| **ข้อผิดพลาดการเข้าถึงถูกปฏิเสธ** | ตรวจสอบว่าแอปพลิเคชันมีสิทธิ์อ่านโฟลเดอร์ต้นทางและมีสิทธิ์เขียนที่เส้นทางผลลัพธ์. |
| **ไฟล์ขนาดใหญ่ทำให้เกิด OutOfMemoryException** | ใช้ `SevenZipArchive` พร้อมตัวเลือกการสตรีมมิ่งหรือแยกอาร์ไคฟ์เป็นหลายส่วน. |
